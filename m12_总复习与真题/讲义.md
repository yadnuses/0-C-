## 一、必背考点速查表 ⭐（按主题分类，每条都是高频）

> 这一节是全课程考点的"压缩包"。每一行都对应一个考过/常考的点。看着眼熟就跳过，看着发懵就回对应模块。

### 1. 输入输出（M1）⭐

| 考点 | 要记的死规矩 | 易错 |
|------|------------|------|
| 占位符配对 | `%d`→int，`%f`→打印double，`%c`→char，`%s`→字符串，`%ld`→long | 配错出乱码/怪数字 |
| scanf 加 `&` | `scanf("%d", &n);` 普通变量前**必须加 `&`** | ⚠️ 漏 `&` 是头号送命题 |
| scanf 读 double | 读 double 用 **`%lf`**，不是 `%f` | ⚠️ 唯一不对称的坑 |
| 数组/字符串名 | 数组名、字符串名本身就是地址，scanf 时**不加 `&`** | `scanf("%s", str)` 不加 & |
| 整数除法 | `5/2=2`（砍小数）；要小数得 `5.0/2=2.5` | ⚠️ 选择/改错常客 |
| 取余 `%` | 只能用于整数，`x%2==0` 判偶数 | 小数取余直接报错 |
| `\n` 换行 | 想换行就在字符串末尾加 `\n` | 不写就挤成一坨 |

### 2. 数据类型（M5 重灾区）⭐⚠️

| 考点 | 要记的死规矩 | 例子 |
|------|------------|------|
| char 范围 | `char` 1字节，`-128 ~ 127`；`unsigned char` `0 ~ 255` | — |
| 溢出回绕 | 超出范围会"绕回去" | `char 127+1 = -128` |
| 强制转换=截断 | `(int)` 直接砍小数，**不四舍五入** | `(int)3.99 = 3` |
| `i++` vs `++i` | `i++` 先用旧值再加；`++i` 先加再用新值 | 见下方陷阱题 |
| 字符运算 | 字符本质是 ASCII 数字，能直接加减 | `'9'-'0'=9`，`'A'+32='a'`，`'A'+2='C'` |
| 大小写换算 | 小写比大写 ASCII 大 32 | `'a'-'A'=32` |
| float 判等 | 浮点数不能用 `==` 直接比，要看差的绝对值 | `fabs(a-b)<1e-6` |

### 3. 分支循环（M2 / M3）⭐⚠️

| 考点 | 要记的死规矩 | 易错 |
|------|------------|------|
| 判等 vs 赋值 | 判断相等用 `==`，`=` 是赋值 | ⚠️ `if(x=5)` 永远为真，改错高频 |
| switch 贯穿 | 每个 case 末尾要 `break`，否则**继续往下执行** | ⚠️ 漏 break 是经典坑 |
| switch 条件 | switch 和 case 的值**必须是整型/字符**，不能是小数/字符串 | — |
| while vs do-while | `while` 先判断（可能 0 次）；`do-while` 先执行（至少 1 次） | — |
| break vs continue | `break` 跳出整个循环；`continue` 跳过本轮剩余、进下一轮 | — |
| 死循环 | `while(1)` / `for(;;)` 是死循环，靠 break 退出 | 忘了改循环变量也会死循环 |

### 4. 函数（M4 / M9）⭐

| 考点 | 要记的死规矩 | 易错 |
|------|------------|------|
| 声明 vs 定义 | **声明有分号**（原型），定义没分号（带 `{}`） | `int f(int);` vs `int f(int x){...}` |
| 传值 | 默认**传值**，函数里改形参，**改不动外面的实参** | ⚠️ 想改外面要传地址/指针 |
| 局部变量 | 函数内的变量出了函数就**失效**（生命周期结束） | 不能返回局部数组的地址 |
| 返回值类型 | `void` 表示不返回；返回类型要和 `return` 的值一致 | — |
| 递归 | 必须有**终止条件**，否则栈溢出 | 阶乘/斐波那契常考 |

### 5. 指针（M7 / M10 命门）⭐⚠️

| 考点 | 要记的死规矩 | 例子 |
|------|------------|------|
| `&` 和 `*` | `&` 取地址，`*` 取内容（解引用） | `p=&a; *p` 就是 a |
| 数组等价式 | `a[i]` ⟺ `*(a+i)`；`&a[i]` ⟺ `a+i` | `*(a+2)` 就是 `a[2]` |
| 数组名 | 数组名 = **首元素地址**（常量，不能改） | `a` ⟺ `&a[0]` |
| 指针运算 | `p+1` 跳过一个**元素**（不是1字节），看类型 | int* 加1跳4字节 |
| `.` vs `->` | 结构体**变量**用 `.`，结构体**指针**用 `->` | `s.age` vs `p->age` |
| 空指针 | `NULL` 不能解引用，否则崩溃 | 用前先判 `if(p!=NULL)` |

### 6. 结构体 / 链表（M8 / M10）⭐

| 考点 | 要记的死规矩 | 例子 |
|------|------------|------|
| 结构体定义 | `struct 名 { 成员; };` 末尾**有分号** | ⚠️ 漏分号常错 |
| 访问成员 | 变量 `.`，指针 `->` | `stu.score` / `p->score` |
| 节点结构 | 链表节点 = **数据 + 指向下一个的 next 指针** | `struct Node{int data; struct Node *next;};` |
| 遍历套路 | `while(cur){ ...; cur=cur->next; }` | 背下来 |
| malloc 配 free | `malloc` 申请的内存要 `free` 释放 | 头文件 `<stdlib.h>` |
| malloc 返回值 | 返回 `void*`，要判是否为 NULL | `if(p==NULL)` |

### 7. 文件（M11）⭐

| 考点 | 要记的死规矩 | 易错 |
|------|------------|------|
| fopen 判 NULL | 打开后**必须判** `if(fp==NULL)`，失败返回 NULL | ⚠️ 不判直接用会崩 |
| 打开模式 | `"r"`读、`"w"`写(**清空原文件**)、`"a"`追加 | `"w"` 会清空已有内容 |
| 读取结束判断 | `fscanf(...) == 项数` 或 `feof(fp)` 判到末尾 | — |
| 用完关闭 | `fclose(fp);` 收尾 | 忘关可能丢数据 |
| 头文件 | 文件操作要 `#include <stdio.h>` | — |

## 二、最爱考的"输出预测"陷阱 🎯（重中之重，多刷几遍）

> 输出预测题是选择题/填空题的主力。套路就那么几个，**全在下面**。每题先盖住答案自己算，再看解析。算错的标★，反复看。

### 陷阱 1：整数除法砍小数 🎯
```c
printf("%d\n", 7 / 2);        // 答案：3
printf("%d\n", 7 / 2 * 2);    // 答案：6（不是7！）
```
**为什么**：两个整数相除，结果取整。`7/2=3`（砍掉.5），`3*2=6`。从左到右算，先除后乘，小数一旦被砍就回不来了。

### 陷阱 2：自增组合 i++ / ++i 🎯⚠️
```c
int i = 1;
printf("%d\n", i++ + ++i);    // 答案：4
```
**为什么**：`i++` 用的是**旧值** 1（用完 i 变 2）；`++i` 先把 i 加成 3 再用，用的是 **3**。所以 `1 + 3 = 4`。
```c
int i = 2;
printf("%d\n", i++ + i++);    // 答案：5
```
**为什么**：第一个 `i++` 用旧值 2（i 变 3），第二个 `i++` 用此时的值 3（i 变 4）。`2 + 3 = 5`。
> 口诀：**`i++` 先用后加，`++i` 先加后用**。草稿纸上一步步标 i 的值，别凭感觉。

### 陷阱 3：switch 缺 break 贯穿 🎯
```c
int x = 1;
switch (x) {
    case 1: printf("A");
    case 2: printf("B");
    case 3: printf("C"); break;
    default: printf("D");
}
// 答案：ABC
```
**为什么**：匹配到 `case 1` 后，因为没有 break，会**一路往下贯穿**执行 B、C，直到遇到 break 才停。所以打印 ABC，不是 A。

### 陷阱 4：字符运算（字符本质是数字）🎯
```c
printf("%c\n", 'A' + 2);      // 答案：C（'A'是65，65+2=67→'C'）
printf("%c\n", 'Z' - 1);      // 答案：Y
printf("%d\n", 'A' + 2);      // 答案：67（用%d就打数字）
printf("%c\n", 'a' - 32);     // 答案：A（小写转大写）
printf("%d\n", '9' - '0');    // 答案：9（字符数字转真实数字的经典写法）
```
**为什么**：字符在内存里就是 ASCII 码。`'A'=65`。用 `%c` 打就显示对应字符，用 `%d` 打就显示数字。

### 陷阱 5：指针/数组等价 🎯
```c
int a[3] = {1, 2, 3};
printf("%d\n", *(a + 2));         // 答案：3（等价于 a[2]）
printf("%d\n", *(a + 1) + a[2]);  // 答案：5（2 + 3）
printf("%d\n", *a);               // 答案：1（等价于 a[0]）
```
**为什么**：`*(a+i)` 完全等价于 `a[i]`。数组名 a 是首地址，加 i 跳到第 i 个元素，再 `*` 取出来。

### 陷阱 6：强制转换截断 🎯
```c
printf("%d\n", (int)(3.9 + 3.9));   // 答案：7
printf("%d\n", (int)3.9 + (int)3.9); // 答案：6
```
**为什么**：第一个**先算** `3.9+3.9=7.8`，再 `(int)` 截断成 7。第二个先各自截断成 3、3，再相加得 6。**括号位置决定一切**。

### 陷阱 7：运算符优先级 🎯
```c
printf("%d\n", 2 + 3 * 4);        // 答案：14（先乘后加）
printf("%d\n", 10 % 3 + 2);       // 答案：3（%和*/同级，先算10%3=1，再+2）
printf("%d\n", 1 + 2 > 2);        // 答案：1（先算1+2=3，再3>2为真=1）
```
**为什么**：`* / %` 高于 `+ -`，`+ -` 高于关系运算 `>`。拿不准就**加括号**。

### 陷阱 8：逻辑短路 🎯⚠️
```c
int a = 0, b = 0;
int r = (a++ && b++);             // 答案：r=0，a=1，b=0（&&左边为假，右边不执行！）
printf("a=%d b=%d r=%d\n", a, b, r);

int x = 1, y = 0;
int s = (x++ || y++);             // x=2 y=0（||左边为真，右边不执行）
printf("x=%d y=%d s=%d\n", x, y, s);
```
**为什么**：`&&` 左边为假，整体必假，**右边根本不算**（b++ 没执行）；`||` 左边为真，整体必真，右边不算（y++ 没执行）。这就是"短路"。

### 陷阱 9：float 判等的坑 🎯
```c
double a = 0.1 + 0.2;
printf("%d\n", a == 0.3);         // 答案：0（不是1！）
```
**为什么**：浮点数在内存里是**近似存储**，`0.1+0.2` 不精确等于 `0.3`。所以判等要用 `fabs(a-0.3) < 1e-6` 这种"差值够小就算相等"的写法。

### 陷阱 10：数组越界 / 下标从0 🎯
```c
int a[3] = {10, 20, 30};
printf("%d\n", a[3]);             // 越界！a只有a[0]~a[2]，a[3]是垃圾值（不报错但结果不可控）
printf("%d\n", a[0]);             // 答案：10（下标从0开始，第1个是a[0]）
```
**为什么**：长度为 3 的数组合法下标是 **0、1、2**。`a[3]` 越界，编译器不拦，运行时读到的是随机内存——这正是 C 危险又爱考的点。

### 陷阱 11：赋值表达式本身有值 🎯
```c
int a, b;
a = b = 5;                        // b=5，a也=5（赋值从右往左）
printf("%d\n", (a = 3));          // 答案：3（赋值表达式的值=被赋的值）
```

### 陷阱 12：sizeof 🎯
```c
printf("%lu\n", sizeof(int));     // 答案：4（int通常4字节）
printf("%lu\n", sizeof(char));    // 答案：1
printf("%lu\n", sizeof(double));  // 答案：8
int a[10];
printf("%lu\n", sizeof(a));       // 答案：40（10个int = 10*4）
```
> ⚠️ `sizeof(数组)` 是**整个数组的字节数**；但数组名传进函数后会退化成指针，那时 `sizeof` 只得到指针大小（8）。这是高频陷阱。

## 三、改错题高频套路 ⚠️（盯死这几个点，每年都考）

> 改错题不用你创造，只要你**眼尖**。考来考去就那几处。下面每个给一个错例 + 改法，把这几招练成条件反射。

### 套路 1：`=` 写成 `==`（判断里）⚠️
```c
if (x = 5) { ... }      // ❌ 这是赋值！x被赋成5，条件永远为真
if (x == 5) { ... }     // ✅ 判断相等用 ==
```
> 排查信号：`if` / `while` 的括号里出现**单个** `=`，八成是该写 `==`。

### 套路 2：scanf 漏 `&` ⚠️
```c
scanf("%d", n);         // ❌ 漏了 &，运行崩溃
scanf("%d", &n);        // ✅ 普通变量前加 &
```
> 注意：数组名/字符串名**不加** `&`（`scanf("%s", str)` 是对的），普通变量一定要加。

### 套路 3：switch 漏 `break` ⚠️
```c
switch (x) {
    case 1: printf("one");        // ❌ 漏break，会贯穿到case 2
    case 2: printf("two"); break;
}
// 改法：每个case末尾补 break;
case 1: printf("one"); break;     // ✅
```

### 套路 4：语句漏分号 `;` ⚠️
```c
int a = 5                // ❌ 漏分号
printf("%d", a)          // ❌ 漏分号
// 改法：每条语句末尾补 ;
int a = 5;               // ✅
printf("%d", a);         // ✅
```
> 报错带 `expected ';'` 就是这个。去报错行号的**上一行**末尾补分号。

### 套路 5：`.` 写成 `->` 或反过来 ⚠️
```c
struct Student s, *p = &s;
s->age = 20;             // ❌ s是变量，该用 .
p.age = 20;              // ❌ p是指针，该用 ->
// 改法
s.age = 20;              // ✅ 变量用 .
p->age = 20;             // ✅ 指针用 ->
```
> 口诀：**变量点（.），指针箭头（->）**。

### 套路 6：fopen 没判 NULL ⚠️
```c
FILE *fp = fopen("a.txt", "r");
fscanf(fp, "%d", &n);    // ❌ 万一文件不存在，fp是NULL，这里崩溃
// 改法：打开后先判
FILE *fp = fopen("a.txt", "r");
if (fp == NULL) {        // ✅
    printf("打开失败\n");
    return 1;
}
```

### 套路 7：scanf 读 double 用错 `%f` ⚠️
```c
double x;
scanf("%f", &x);         // ❌ 读double必须用 %lf
scanf("%lf", &x);        // ✅
printf("%f\n", x);       // ✅ 但打印double用 %f（注意不对称！）
```

### 套路 8：数组下标越界 / 从1开始 ⚠️
```c
int a[5];
for (int i = 1; i <= 5; i++) a[i] = i;   // ❌ a[5]越界！合法是a[0]~a[4]
for (int i = 0; i < 5; i++)  a[i] = i;   // ✅ 下标0~4
```
> 记牢：长度 n 的数组，合法下标 `0 ~ n-1`，循环条件写 `i < n`。

### 套路 9：malloc 没 free / 没判 NULL ⚠️
```c
struct Node *p = (struct Node*)malloc(sizeof(struct Node));
// ... 用完没free → 内存泄漏
free(p);                 // ✅ 用完释放
```

### 套路 10：函数声明/定义写反分号 ⚠️
```c
int add(int a, int b) { return a+b; };   // ❌ 定义末尾不该有分号
int add(int a, int b);                    // ✅ 声明（原型）才有分号
int add(int a, int b) { return a+b; }     // ✅ 定义没分号
```

### 套路 11：变量没声明就用 / 类型不符 ⚠️
```c
n = 10;                  // ❌ n没声明过
int n = 10;              // ✅ 先声明类型
```
> 报错带 `undeclared` 就是它。检查是不是名字打错、或忘了写 `int xxx;`。

### 套路 12：中文标点混进代码 ⚠️
```c
printf("hello")；        // ❌ 那个分号是中文全角的！
printf("hello");         // ✅ 全部英文半角
```
> 报错很怪、看不懂时，**先怀疑中文标点**（`；（）""`）。

## 四、综合编程大题 🎯（呼应 练习.c，给框架思路）

> 编程大题别慌。诀窍是：**先把架子搭出来（main + 函数声明 + 结构体），能编译就有分；再一个函数一个函数填逻辑。** 下面三道是考研最常考的方向，每道给出"先写什么、再写什么"的拆解。

### ⭐⭐ 先记死：算法大题的"三段式答题规范"（比代码本身还重要）

> 408 的算法大题（真题里的 41、42 题）**从来不是"代码能跑就给满分"**。阅卷是**分段给分**的——一道 13 分的题，分成三块各给各的分。你哪段写到位就拿哪段的分，缺一段就整段丢。所以**还没学到数据结构、还不会写那些代码之前，先把这个"答题骨架"刻进脑子**，以后做大题照着这个格式套就行。

一道标准算法大题，答题纸上要写成**三段**：

**① 算法思想（先用人话讲清楚你打算怎么做）—— 占 4~6 分**

- 这一段**不写代码**，纯文字 + 必要的小例子，把"我用什么办法、分哪几步"说明白。
- ⭐⭐ 最关键的认知：**哪怕后面代码一个字都不会写，这一段写对了照样能拿 4~6 分！** 所以考场上就算卡在代码，也**一定先把思路写满**，这是白捡的半壁分。
- ⚠️ 别上来就闷头甩代码。很多人代码写错了，连带思想分一起丢——白白损失一半。先讲思路，稳。

**② 代码实现（C 或 C++ 都行，关键处给注释）—— 占 7~9 分**

- 套用我们一直练的办法：**先搭架子**（把题目给的函数原型照抄下来 + 需要的变量/结构体）→ **再一个一个填逻辑**。能编译就有结构分。
- ⚠️⚠️ 题目给你的**函数原型一个字都不能改**（参数名、返回类型照搬）。比如题目写 `int func(MGraph G)`，你就得原样写 `int func(MGraph G)`，不能自己改名。
- 关键步骤**加一行注释**，阅卷老师一眼看懂你在干嘛，更容易给分。

**③ 时间复杂度 + 空间复杂度（两个都要写）—— 占 2 分**

- 这 2 分是**白送的**，千万别漏写！很多人代码写得满头大汗，最后忘了这一行，平白丢 2 分。
- ⚠️ 哪怕你拿不准准确的复杂度——**也要写出来**。写了还有机会蒙对，不写就一定是 0 分。
- （怎么判断一段代码是 O(n) 还是 O(n²)，等你学到数据结构会专门讲。现在只要**记住这一格必须填、是送分的**就够了。）

**一句话口诀，背下来 ⭐**：
> **先说思路、再写代码、最后标复杂度——三段都占着，分就不会整段丢。**

> 💡 现在你还没学树、图这些数据结构，写不出真题里那些代码很正常，**完全不用慌**。这一节要你现在就记的，不是"怎么写代码"，而是"**答题纸该长什么样**"这个习惯。等以后学深了，往这个三段骨架里填内容就行。下面三道练手题，也请你顺手按这个格式答——思路、代码、复杂度，一段都别省。

### 大题 1：学生成绩管理（结构体数组 + 排序 + 求平均）🎯

考点：结构体定义、数组遍历、冒泡排序、求平均/最值。

**第一步：先搭架子（结构体 + main + 声明）**
```c
#include <stdio.h>

struct Student {
    char name[20];
    int score;
};

int main(void)
{
    struct Student stu[5] = {
        {"Tom", 85}, {"Amy", 92}, {"Bob", 78},
        {"Lily", 88}, {"Jack", 95}
    };
    int n = 5;
    // 下面逐步填：求平均 → 找最高 → 排序输出
    return 0;
}
```

**第二步：求平均分**（遍历累加，注意整数除法）
```c
int sum = 0;
for (int i = 0; i < n; i++) sum += stu[i].score;
double avg = (double)sum / n;       // ⚠️ 转double，否则整数除法砍小数
printf("平均分：%.2f\n", avg);
```

**第三步：找最高分学生**（打擂台）
```c
int maxIdx = 0;
for (int i = 1; i < n; i++)
    if (stu[i].score > stu[maxIdx].score) maxIdx = i;
printf("最高分：%s %d\n", stu[maxIdx].name, stu[maxIdx].score);
```

**第四步：冒泡排序（从高到低）+ 输出**
```c
for (int i = 0; i < n - 1; i++)
    for (int j = 0; j < n - 1 - i; j++)
        if (stu[j].score < stu[j+1].score) {   // < 是从高到低
            struct Student t = stu[j];          // 整个结构体可直接赋值交换
            stu[j] = stu[j+1];
            stu[j+1] = t;
        }
for (int i = 0; i < n; i++)
    printf("%s: %d\n", stu[i].name, stu[i].score);
```
> 关键记忆：冒泡两层循环 `i:0~n-2`、`j:0~n-2-i`；结构体可整体赋值交换；从高到低用 `<`，从低到高用 `>`。

### 大题 2：字符串处理（统计 + 反转）🎯

考点：字符数组、字符分类判断、双指针反转。

**第一步：架子**
```c
#include <stdio.h>
#include <string.h>

int main(void)
{
    char s[100];
    printf("输入字符串：");
    scanf("%s", s);          // 读一个不含空格的串（含空格用 fgets）
    // 逐步填：统计 → 反转
    return 0;
}
```

**第二步：统计字母/数字/其他**
```c
int letter = 0, digit = 0, other = 0;
for (int i = 0; s[i] != '\0'; i++) {     // 遍历到 '\0' 结束
    if ((s[i] >= 'a' && s[i] <= 'z') || (s[i] >= 'A' && s[i] <= 'Z'))
        letter++;
    else if (s[i] >= '0' && s[i] <= '9')
        digit++;
    else
        other++;
}
printf("字母%d 数字%d 其他%d\n", letter, digit, other);
```

**第三步：原地反转（双指针对撞）**
```c
int len = strlen(s);
for (int i = 0, j = len - 1; i < j; i++, j--) {
    char t = s[i]; s[i] = s[j]; s[j] = t;    // 头尾交换，向中间靠
}
printf("反转后：%s\n", s);
```
> 关键记忆：字符串以 `'\0'` 结尾，遍历条件 `s[i]!='\0'`；`strlen` 求长度要 `#include <string.h>`；反转用左右双指针对撞，`i<j` 停。

### 大题 3：链表综合（建表 + 插入 + 删除 + 遍历）🎯

考点：节点结构、malloc、头插法、遍历、按值删除。

**第一步：节点结构 + 架子**
```c
#include <stdio.h>
#include <stdlib.h>

struct Node {
    int data;
    struct Node *next;
};
```

**第二步：头插法建表（1~5）**
```c
struct Node* createList(void)
{
    struct Node *head = NULL;
    for (int i = 1; i <= 5; i++) {
        struct Node *p = (struct Node*)malloc(sizeof(struct Node));
        p->data = i;
        p->next = head;     // 新节点指向原来的头
        head = p;           // 头更新为新节点
    }
    return head;            // 结果是 5→4→3→2→1
}
```

**第三步：遍历打印（背下这个套路）**
```c
void printList(struct Node *head)
{
    struct Node *cur = head;
    while (cur != NULL) {
        printf("%d ", cur->data);
        cur = cur->next;    // ⚠️ 别忘了往后走，否则死循环
    }
    printf("\n");
}
```

**第四步：表头插入一个值**
```c
struct Node* insertHead(struct Node *head, int x)
{
    struct Node *p = (struct Node*)malloc(sizeof(struct Node));
    p->data = x;
    p->next = head;
    return p;               // 新头
}
```

**第五步：删除值为 x 的节点**
```c
struct Node* deleteVal(struct Node *head, int x)
{
    // 处理头节点就是要删的
    while (head != NULL && head->data == x) {
        struct Node *t = head;
        head = head->next;
        free(t);
    }
    struct Node *cur = head;
    while (cur != NULL && cur->next != NULL) {
        if (cur->next->data == x) {
            struct Node *t = cur->next;
            cur->next = t->next;     // 跨过要删的节点
            free(t);
        } else {
            cur = cur->next;
        }
    }
    return head;
}
```
> 关键记忆：遍历套路 `while(cur){...; cur=cur->next;}`；删除靠"前一个的 next 跨过当前"；malloc 配 free；改了头要 return 新头。

## 五、附录A 速查表格 📚（选择题送分，扫一眼有印象就行）

### 1. 运算符优先级表（从高到低，共15级）⭐

> 不用死背全部，**记住"乘除高于加减、算术高于关系、关系高于逻辑、逻辑高于赋值"这条主线**就够应付大多数题。拿不准一律加括号。

| 级别 | 运算符 | 说明 | 结合性 |
|------|--------|------|--------|
| 1（最高） | `()` `[]` `->` `.` | 初等运算符 | 左→右 |
| 2 | `!` `~` `++` `--` `+`(正) `-`(负) `*`(解引用) `&`(取址) `(类型)` `sizeof` | 单目 | **右→左** |
| 3 | `*` `/` `%` | 乘除取余 | 左→右 |
| 4 | `+` `-` | 加减 | 左→右 |
| 5 | `<<` `>>` | 移位 | 左→右 |
| 6 | `<` `<=` `>` `>=` | 关系 | 左→右 |
| 7 | `==` `!=` | 相等 | 左→右 |
| 8 | `&` | 按位与 | 左→右 |
| 9 | `^` | 按位异或 | 左→右 |
| 10 | `|` | 按位或 | 左→右 |
| 11 | `&&` | 逻辑与 | 左→右 |
| 12 | `||` | 逻辑或 | 左→右 |
| 13 | `?:` | 条件（三目） | **右→左** |
| 14 | `=` `+=` `-=` 等 | 赋值 | **右→左** |
| 15（最低） | `,` | 逗号 | 左→右 |

> ⚠️ 三个"从右往左"的：**单目、条件 `?:`、赋值**。所以 `a = b = 5` 是从右往左：先 b=5，再 a=5。

### 2. 转义字符表 ⭐

| 转义字符 | 含义 | 转义字符 | 含义 |
|---------|------|---------|------|
| `\n` | 换行 | `\\` | 一个反斜杠 `\` |
| `\t` | 水平制表（Tab） | `\'` | 单引号 `'` |
| `\0` | 空字符（字符串结束标志） | `\"` | 双引号 `"` |
| `\r` | 回车 | `\b` | 退格 |

> ⚠️ 想打印一个 `\`，要写 `\\`；想在双引号串里打印 `"`，要写 `\"`。`\0` 是字符串结尾标志，ASCII 值为 0。

### 3. 常用 ASCII 值（背几个关键的就行）⭐

| 字符 | ASCII | 记忆点 |
|------|-------|--------|
| `'0'` | 48 | 数字字符起点；`'9'-'0'=9` 转真实数字 |
| `'A'` | 65 | 大写起点 |
| `'a'` | 97 | 小写起点；**比大写大 32** |
| `' '`(空格) | 32 | `'A'+32='a'` 大转小 |
| `'\0'` | 0 | 字符串结束 |

> 核心三条：①数字字符 `-'0'` 得真实数字；②大小写差 **32**（小写大）；③字符能当数字直接加减比较。

### 4. 进制常量速记
- 十进制：`123`
- 八进制：`0` 开头，如 `0123`（= 十进制 83）
- 十六进制：`0x` 开头，如 `0x1A`（= 十进制 26）
- 后缀：`L` 表 long，`U` 表 unsigned

### 5. 关键字数量
- C 语言共 **32 个关键字**（选择题偶尔考数量）。
- 常见误区：`main` **不是**关键字（它是函数名）；`printf`/`scanf` 也不是关键字（是库函数）。

## 六、应试技巧 🎯（考场上照着做）

1. **输出预测题**：拿到题别心算，在草稿纸上**逐行标出每个变量的值变化**。尤其是 `i++`/`++i`、switch 贯穿、强制转换，一步步走完再写答案。
2. **改错题**：盯死 5 个高频点——`=`vs`==`、漏 `&`、漏 `break`、漏 `;`、`.`vs`->`。再加一个 fopen 没判 NULL。从头到尾扫一遍，每行问自己"这五处有没有问题"。
3. **编程大题**：**先写框架**（`#include` + 结构体 + `main` + 函数声明），保证能编译，**能编译就有结构分**。再一个函数一个函数填逻辑。哪怕逻辑没写完，框架对了也拿分。
4. **拿不准优先级**：一律**加括号**。`a + b > c` 拿不准就写 `(a + b) > c`，括号不要钱还防错。
5. **整数除法**：凡是涉及"平均数/比例/百分比"，先想"会不会被整数除法砍掉小数"，需要小数就 `(double)` 强转或写成 `5.0`。
6. **scanf 检查清单**：每个普通变量前有没有 `&`？读 double 是不是用了 `%lf`？占位符和变量类型对不对？
7. **数组/循环边界**：长度 n 的数组，下标 `0~n-1`，循环写 `i < n`。检查有没有 `<=` 越界。
8. **时间分配**：选择/填空（输出预测、概念）求快求稳；改错题逐行扫；编程大题留足时间，先搭框架。不会的先跳过，回头再啃。
