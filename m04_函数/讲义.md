## 〇、开始之前：函数到底是个啥？

到现在为止，你写的所有代码都堆在 `main` 那一个大括号里。程序一小还行，一大就乱成一锅粥。**函数**就是来救这个场的。

### 一句话类比：函数就是一台"机器" 🏭

想象一台榨汁机：
- 你往里**放水果**（这叫"输入"，对应函数的**参数**）。
- 机器**嗡嗡работа一阵**（这叫"处理"，对应函数体里的代码）。
- 出来一杯**果汁**（这叫"输出"，对应函数的**返回值**）。

你不用关心榨汁机内部的刀片怎么转——你只要会"放水果、接果汁"就行。函数也一样：**写好一次，到处调用，里面怎么实现的，用的时候不用管。**

### 另一个类比：函数就是"菜谱" 📖

菜谱写好了"西红柿炒蛋"的做法。以后你想吃，照着做一遍就行，不用每次重新发明。
- 菜谱的名字 = 函数名
- 菜谱要你准备的食材 = 参数
- 照着做完端出来的菜 = 返回值

记住这两个类比，下面所有内容都能往上套。

## 一、为什么要有函数？⭐

新手常问："我全写在 main 里也能跑，干嘛非要函数？"三个硬道理：

### ① 避免重复代码（最直接的好处）

假设你要算三个圆的面积，没有函数你得写三遍：

```c
double area1 = 3.14 * 2 * 2;
double area2 = 3.14 * 5 * 5;
double area3 = 3.14 * 8 * 8;
```

有了函数，算面积的公式只写一遍：

```c
double area(double r) {       // 算面积的"机器"，写一次
    return 3.14 * r * r;
}
// 用的时候：
double a1 = area(2);
double a2 = area(5);
double a3 = area(8);
```

⚠️ 更关键的是：万一公式要改（比如把 3.14 改成更精确的 3.14159），**有函数只改一处，没函数你得满世界找着改**，改漏一个就出 bug。

### ② 分而治之（把大问题拆成小问题）⭐

一个复杂程序，直接写会让你脑子炸。正确做法是：**把大任务切成一个个小任务，每个小任务写成一个函数**。比如做一个"学生成绩系统"：

- `inputScores()` —— 负责输入成绩
- `average()` —— 负责算平均分
- `printReport()` —— 负责打印报告

`main` 里就像个"总指挥"，按顺序调用这几个函数。每个函数只干一件事，写起来、改起来、查 bug 都轻松。这就是翁恺笔记里说的**"自顶向下，逐步细化"**（后面第十节细讲）。

### ③ 好读、好维护

函数名起得好（比如 `average`），别人一看名字就知道这段在干嘛，不用读里面的代码。代码就像文章分了段落，清清楚楚。

> 🎯 选择题常考"使用函数的好处"，答案围绕：**减少重复代码、便于模块化（分而治之）、提高可读性和可维护性**。

## 二、函数三要素：返回类型、函数名、参数列表 ⭐⚠️ 核心

一个函数的"长相"固定就是这个模子，**所有函数都套这个壳**：

```c
返回类型  函数名(参数列表)
{
    函数体;
    return 返回值;
}
```

我们用一个真实例子，逐字拆解：

```c
int add(int a, int b)
{
    return a + b;
}
```

### ① `int` —— 返回类型（机器吐出来的东西是什么类型）
- 表示这个函数**最后会还给你一个 int（整数）**。
- 类比：榨汁机吐出的是"果汁"，这里规定吐出的是"整数"。
- 如果函数算出来的是小数，这里就写 `double`；如果啥都不还，写 `void`（第五节讲）。

### ② `add` —— 函数名（机器的名字）
- 你自己起的名字，**起名规矩和变量名完全一样**：只能用字母数字下划线、不能数字开头、区分大小写、见名知意。
- 类比：给榨汁机贴个标签叫"add"，以后喊这个名字就能用它。

### ③ `(int a, int b)` —— 参数列表（要往机器里放的原料）
- 括号里列出这个函数**需要外面给它什么**。这里要两个整数，分别叫 `a` 和 `b`。
- `a`、`b` 叫**形参**（形式参数），它们就是函数内部的"临时盒子"，专门接外面传进来的值。
- ⚠️⚠️ **超高频考点**：每个形参前面的类型**必须各写各的**！

```c
int add(int a, int b)   // ✅ 对，a 和 b 各自写了 int
int add(int a, b)       // ❌ 错！b 前面漏了类型
int add(int a, int b)   // ✅
// ❌ 不能图省事写成 int add(int a, b) 或 (int a b)
```

> 翁恺原话："每个形参前的类型必须分别写明，不能合并写成 `double r, h`。" 记死它，改错题爱考。

### ④ `{ return a + b; }` —— 函数体（机器内部干的活）
- 大括号里就是这台机器具体怎么工作。这里很简单：把 `a` 和 `b` 加起来还回去。
- `return a + b;` 表示"算出 a+b，把结果还给调用我的人"。
- ⚠️ 函数定义的大括号后面**不写分号**（记住这点，第六节会和"声明"对比，这是考点）。

### 读一遍这个函数
> "定义一台机器叫 add，它要两个整数 a 和 b，工作内容是把它俩加起来，然后把和这个整数还出去。"

## 三、函数的调用与执行流程 ⭐⚠️ 重点

光定义了函数，它不会自己跑。你得**调用**它，它才工作。

### 怎么调用？

```c
int s = add(3, 5);    // 调用 add，把 3 和 5 送进去，接住还回来的结果
printf("%d\n", s);    // 输出 8
```

拆解：
- `add(3, 5)` —— 喊出函数名，括号里塞进去两个**实参** `3` 和 `5`。
- 函数算出 `3 + 5 = 8`，把 `8` 还回来。
- `int s = ...` —— 用盒子 s 接住这个 8。

也可以不接，直接用在别处：

```c
printf("%d\n", add(3, 5));   // 直接把 add 还回来的 8 拿去打印
printf("%d\n", add(1, 2) + add(10, 20));   // 函数还能参与运算：3 + 30 = 33
```

### 执行流程：跳过去，干完活，再跳回来 ⭐⚠️

这是个**必考考点**，一定要在脑子里建立这个画面：

```c
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int main(void) {
    printf("开始\n");           // ① 先执行这句
    int s = add(3, 5);          // ② 程序"跳"进 add 去执行
    printf("结果是 %d\n", s);   // ④ add 回来后，接着执行这句
    return 0;
}
```

电脑实际跑的顺序：
1. 执行 `printf("开始\n")`，打印"开始"。
2. 碰到 `add(3, 5)`，**main 暂停**，程序跳进 add 里。
3. add 把 a=3、b=5 加起来得 8，`return 8` 把结果带回 main 调用的地方。
4. **回到 main 暂停的位置继续往下**，s 拿到 8，打印"结果是 8"。

> 翁恺原话："遇到函数调用时，暂停 main 函数，转而执行被调函数；被调函数执行完毕后，带着返回值回到 main 函数调用处继续执行。"

⚠️ 关键理解：**main 在等 add 干完才往下走**，不是两个一起跑。就像你让朋友去买瓶水（调用），你站原地等他回来（暂停），他把水递给你（返回），你才继续。

### 实参可以是常量、变量、表达式 ⭐

调用时括号里填的"实参"很灵活：

```c
int x = 10, y = 20;
add(3, 5);          // 实参是常量
add(x, y);          // 实参是变量
add(x + 1, y * 2);  // 实参是表达式（先算出 11 和 40 再传进去）
```

⚠️ 规则：**实参和形参数量要相同，类型尽量一致，顺序一一对应**。第一个实参给第一个形参，第二个给第二个，不能乱。

## 四、return：函数把结果"还给你" ⭐⚠️ 重点

`return` 是函数里最关键的一个词，干两件事，翁恺原话说得很清楚：

> "`return 表达式;` 的作用：①结束函数运行；②将表达式的值返回给主调函数。"

逐个拆：

### ① return 让函数当场"收工"
程序一执行到 `return`，函数**立刻结束**，return 后面的代码一行都不再跑。

```c
int test(void) {
    printf("A\n");
    return 1;          // 执行到这里函数就结束了
    printf("B\n");     // ❌ 这句永远不会被执行（死代码）
}
```
输出只有：
```
A
```
⚠️ `return` 后面的语句叫"死代码"，永远跑不到。新手有时把 return 写早了，导致后面的代码白写。

### ② return 把值带回去
`return a + b;` 表示"把 a+b 算出来的结果，交还给调用我的人"。这个值会**顶替到调用的位置上**：

```c
int s = add(3, 5);
// add(3, 5) 算完 return 8，于是这行就相当于 int s = 8;
```

### ⚠️⚠️ return 一次只能还一个值（超高频考点）
翁恺特别强调：**`return` 语句只能返回一个值。**

```c
return a + b;       // ✅ 还一个值
return a, b;        // ❌ 不是"还两个值"！这其实只还了 b（逗号表达式，别用）
```

🎯 真题常考"想让函数返回多个结果怎么办？"——`return` 做不到，标准答案是**用全局变量**（本讲第八节）或**用指针/地址传出**（M7 指针细讲）。翁恺第 5.3 节算复数的和与积，要返回实部和虚部两个值，用的就是全局变量这招。

### 一个函数里可以有多个 return ⭐
不是只能写一个 return。根据不同情况，可以写好几个，**哪个先执行到就从哪个出去**：

```c
int even(int n) {       // 判断 n 是不是偶数（翁恺例5-2）
    if (n % 2 == 0)
        return 1;       // 是偶数，还 1
    else
        return 0;       // 是奇数，还 0
}
```
⚠️ 这里两个 return，但每次调用只会走其中一个。这很常见，别以为函数只能有一个 return。

### ⚠️ return 的值类型要和"返回类型"对得上
函数头写的返回类型是 `int`，return 后面就该还整数；写 `double` 就该还小数。对不上会被电脑偷偷转换，出怪结果：

```c
int half(int n) {
    return n / 2.0;     // ⚠️ 算出来是小数(double)，但返回类型是 int，小数部分被砍掉
}
// half(5) 你以为得 2.5，实际得 2（被强转成 int 了）
```
要得到 2.5，函数返回类型得写成 `double`。

## 五、void：不还东西 / 不要东西的函数 ⭐

不是每台机器都吐产品。有的函数只是"干件事"，比如打印一个图形，它**不需要还任何值**。这种函数返回类型写 `void`（意思是"空、无"）。

```c
void sayHello(void) {
    printf("你好！\n");
    printf("欢迎学C语言\n");
    // 没有 return 值，因为它只负责打印，不用还东西
}
```

### void 在两个位置出现，意思不一样 ⚠️ 别搞混
```c
void sayHello(void)
//↑①              ↑②
```
- **位置①（前面的 void）= 返回类型为空**：这函数不还任何值。
- **位置②（括号里的 void）= 参数列表为空**：这函数不需要外面给它任何原料。

两个 void 各管各的，可以单独出现：

```c
void pyramid(int n) { ... }   // 不还值(void)，但要一个参数 n
int getNumber(void) { ... }   // 要还一个 int，但不需要参数(void)
```

### void 函数怎么调用？⚠️ 考点
普通函数能接住返回值（`int s = add(3,5);`），但 **void 函数没东西给你接**，所以它**只能单独成一句**，不能放进表达式或赋值：

```c
sayHello();              // ✅ 对，单独一句
pyramid(5);              // ✅ 对

int x = sayHello();      // ❌ 错！sayHello 啥都不还，没东西给 x
printf("%d", pyramid(5));// ❌ 错！pyramid 不还值，没法打印
```

> 翁恺原话："（void 函数的）调用以独立语句形式出现，如 `pyramid(5);`，不能出现在表达式中。"

### void 函数里能写 return 吗？能，但不带值 ⭐
void 函数里也可以用 `return;`（光秃秃一个 return 加分号，后面**不跟任何值**），作用是"提前收工"：

```c
void check(int n) {
    if (n < 0) {
        printf("负数，不处理\n");
        return;          // ✅ 提前结束，后面不执行了。注意 return 后面没有值！
    }
    printf("处理 %d\n", n);
}
```
⚠️ 区别记牢：
- 有返回值的函数：`return 值;`（必须带值）
- void 函数：`return;`（不带值，且常常可以省略不写）

## 六、函数声明（原型）：先打个招呼 ⭐⚠️⚠️ 必考中的必考

这一节是 M4 **最容易考、新手最容易栽**的地方，慢慢看。

### 问题从哪来？electronics 电脑是"从上往下读代码"的死脑筋
电脑读你的程序是**从上到下**一行行读的。如果你把函数 `add` 定义在 `main` **后面**，那电脑读到 main 里的 `add(3,5)` 时，**还没见过 add 是谁**，就懵了，直接报错：

```c
#include <stdio.h>

int main(void) {
    int s = add(3, 5);    // ❌ 报错！电脑还不认识 add（它定义在下面）
    printf("%d\n", s);
    return 0;
}

int add(int a, int b) {   // add 的定义在 main 后面，电脑太晚才看到
    return a + b;
}
```

### 解决办法一：把函数定义挪到 main 前面（能用但不推荐）
把 `add` 整个搬到 main 上面，电脑先认识它就不报错了。但大程序里函数很多，全堆在 main 前面会很乱。

### 解决办法二：函数声明（推荐，也是考点）⭐
在 main **前面**先写一行"**函数声明**"（也叫**函数原型**），等于提前给电脑递个名片："有个叫 add 的函数，要两个 int，还一个 int，具体长啥样你往下看。"

```c
#include <stdio.h>

int add(int a, int b);    // ⭐ 函数声明（原型）——注意结尾有分号！

int main(void) {
    int s = add(3, 5);    // ✅ 电脑见过名片了，认识 add，不报错
    printf("%d\n", s);
    return 0;
}

int add(int a, int b) {   // 函数定义（真正干活的代码）——注意没分号！
    return a + b;
}
```

翁恺建议的标准顺序：**函数声明 → main（调用）→ 函数定义**。照这个顺序写就不会错。

### ⭐⚠️⚠️ 全场最大的考点：声明有分号，定义没分号！
这是改错题、选择题的"钉子户"，给我刻进 DNA：

```c
int add(int a, int b);    // 声明：函数首部 + 分号 ;   ← 有分号！
                          // 它没有大括号身体，就是"打个招呼"

int add(int a, int b)     // 定义：函数首部 + { 身体 }  ← 没分号！
{
    return a + b;
}
```
对比记忆：
| | 长啥样 | 结尾 | 有没有身体{} |
|---|---|---|---|
| **声明（原型）** | `int add(int a, int b);` | **有分号 `;`** | 没有 |
| **定义** | `int add(int a, int b){...}` | **没分号** | 有 `{ }` |

⚠️ 新手两大死法：
1. 在**定义**的大括号前面手贱加了分号 → `int add(int a,int b);{...}` ❌ 报错。
2. **声明**忘了加分号 → `int add(int a, int b)` 当成定义的开头，电脑等着找 `{` → 报错。

### 声明里可以省略形参名字 ⭐（了解）
声明只是告诉电脑"类型长啥样"，形参叫什么名字不重要，所以可以只写类型：

```c
int add(int, int);        // ✅ 也合法，只写类型，省了名字 a、b
int add(int a, int b);    // ✅ 也合法，带名字更清楚
```
⚠️ 但**定义**里必须写名字（不然函数体里没法用）。入门阶段建议声明也带上名字，和定义保持一致，不容易错。

## 七、形参 vs 实参，与"传值调用" ⭐⚠️⚠️ 重点中的重点

这一节理解了，你就懂了函数最核心的脾气。先把两个词分清：

### 形参和实参，到底谁是谁？
- **形参**（形式参数）：写在函数**定义**的括号里那些变量，比如 `int add(int a, int b)` 里的 `a`、`b`。它们是函数内部的"空盒子"，等着接东西。
- **实参**（实际参数）：函数**调用**时括号里真正塞进去的值，比如 `add(3, 5)` 里的 `3`、`5`。

类比：形参是榨汁机上贴的标签"水果口"，实参是你真正塞进去的那个苹果。

### 传值调用：函数拿到的是"复印件"，不是"原件" ⭐⚠️⚠️
这是 C 语言函数**最重要、最爱考、新手最想不通**的一条规则。翁恺原话：

> "调用时，实参的值依次**复制**给形参。这种传递是**单向**的，**形参值的变化不会影响实参**。"

翻译成人话：你调用函数时，电脑把实参的值**复印一份**塞给形参。函数里折腾的是那张**复印件**，**原件（外面的变量）一根毛都不会动。**

看这个经典的"想交换两个数却失败"的例子：

```c
#include <stdio.h>

void swap(int a, int b) {   // 想交换 a 和 b
    int t = a;
    a = b;
    b = t;
    printf("函数内: a=%d, b=%d\n", a, b);
}

int main(void) {
    int x = 3, y = 5;
    swap(x, y);
    printf("函数外: x=%d, y=%d\n", x, y);
    return 0;
}
```
实际输出：
```
函数内: a=5, b=3
函数外: x=3, y=5      ← ⚠️ 外面的 x、y 根本没变！
```

为什么外面没变？因为：
1. 调用 `swap(x, y)` 时，把 x 的值 3、y 的值 5 **复印**给了形参 a、b。
2. 函数里交换的是 a 和 b（复印件），它俩确实换了。
3. 但 x、y（原件）压根没参与，函数结束 a、b 就销毁了，**x、y 还是 3 和 5**。

🎯 这是真题"明星题"：给你一段 swap 函数问最后输出啥，答案就是**没换成功，外面变量不变**。

### ⚠️ 一句话口诀
> **传值调用 = 给函数一份复印件，函数改复印件，改不到你的原件。**

### 那怎么才能真的改外面的变量？预告 M7 🔮
答案是**传地址（指针）**：不传值的复印件，而是把变量的"家庭住址"告诉函数，函数顺着地址找上门去改原件。这就是为什么 `scanf("%d", &n)` 要加 `&`（M1 埋的坑，M7 揭晓）。现在你只要记住：
- 普通传值：函数改不动外面（本节）。
- 传地址（加 `&` / 用指针）：函数能改外面（M7 讲）。

### 形参就是局部变量 ⭐
翁恺点明："**形参属于局部变量**。"意思是形参只在函数内部存在，函数一结束它就消失了，外面完全访问不到。这也呼应了第八节要讲的"作用域"。

## 八、局部变量 vs 全局变量 / 作用域 ⭐⚠️ 重点

"作用域"听着吓人，其实就是问一句：**这个变量，在哪些地方能用？** 翁恺把变量分成两大类。

### 局部变量：只在"自己家"里好使 ⭐
**定义在函数内部（大括号里）的变量，就是局部变量。** 它的作用域（能用的范围）**只限于定义它的那个函数**。出了这个函数，谁也看不见它。

```c
void f1(void) {
    int x = 10;        // x 是 f1 的局部变量
    printf("%d\n", x);
}

void f2(void) {
    printf("%d\n", x); // ❌ 报错！x 是 f1 家的，f2 看不到
}
```

⚠️ 两个不同函数里可以有**同名的局部变量**，它们是**毫不相干的两个盒子**，互不影响：

```c
void a(void) { int n = 1; }   // 这个 n
void b(void) { int n = 99; }  // 和这个 n 完全是两回事，名字一样而已
```
就像张三家的"小明"和李四家的"小明"是两个人。这点想通了，前面 swap 为啥改不到外面也就更清楚了——形参 a、b 是 swap 家的局部变量。

> 翁恺：局部变量"保证函数独立性，避免函数间相互干扰"。这正是用函数的好处之一。

### 复合语句里也能定义局部变量 ⭐（了解）
不只是函数，任意一对大括号 `{ }`（比如 if、for 的身体）里定义的变量，作用域也只在那对大括号内：

```c
if (x > 0) {
    int temp = x * 2;   // temp 只在这个 if 的 { } 里活着
}
printf("%d", temp);     // ❌ 出了 if 的大括号，temp 就没了，报错
```

### 全局变量：定义在所有函数外面，谁都能用 ⭐
**定义在所有函数外面（通常在文件最上方）的变量，是全局变量。** 它的作用域是**从定义那一行开始，一直到文件结束**，中间所有函数都能读、能改它。

```c
#include <stdio.h>

int count = 0;          // ⭐ 全局变量，定义在 main 外面

void add1(void) {
    count = count + 1;  // ✅ 能直接用全局的 count
}

int main(void) {
    add1();
    add1();
    printf("%d\n", count);  // 输出 2，两个函数共享同一个 count
    return 0;
}
```

全局变量的**用途**：让多个函数**共享数据**。前面说过 `return` 只能还一个值，想从函数传出多个结果时，全局变量是一招（翁恺算复数和与积、记现金账就是这么干的）。

### ⚠️ 全局变量是把双刃剑，慎用！⭐ 考点
翁恺明确提醒：全局变量"过度使用会降低函数独立性，使程序难以理解和维护，应**谨慎使用**"。

为什么慎用？因为谁都能改它，出了 bug 你都不知道是哪个函数改坏的，像一间所有人都能进的公共储物间，乱得很。结构化程序设计原则也说：**模块间联系尽量简单，慎用全局变量**，优先用参数传递。

🎯 选择题常考"局部变量和全局变量的区别"，记这张表：

| | 局部变量 | 全局变量 |
|---|---|---|
| 定义位置 | 函数（或 `{}`）**内部** | 所有函数**外部** |
| 作用域 | 只在所在函数/复合语句内 | 从定义处到文件结尾 |
| 用途 | 函数内部临时使用 | 多个函数共享数据 |
| 建议 | 优先用它 | 谨慎用，能不用就不用 |

### ⚠️⚠️ 同名时：局部"屏蔽"全局（高频考点）
如果一个函数里的局部变量，和全局变量**重名**了，在这个函数内部，**局部变量优先，全局变量被"屏蔽"（暂时看不见）**：

```c
#include <stdio.h>

int x = 100;            // 全局 x

void f(void) {
    int x = 5;          // 局部 x，和全局重名
    printf("%d\n", x);  // 输出 5！用的是局部的，全局 x=100 被屏蔽了
}
```
翁恺原话："与局部变量同名时，在局部变量作用域内，全局变量被'屏蔽'。" 记法：**近的盖远的，里面的盖外面的。**

## 九、变量的"生存周期"与静态局部变量 ⭐⚠️

前面"作用域"问的是**在哪能用**；这一节"生存周期"问的是另一件事：**这个变量活多久？什么时候出生、什么时候死？**

### 普通局部变量：函数一调就生，函数一完就死 ⭐
翁恺：普通局部变量（也叫"**自动变量**"）"函数被调用时分配存储单元，**函数调用结束立即回收**"。

也就是说，局部变量是"用完即焚"的：

```c
void f(void) {
    int n = 0;          // 每次进 f，n 重新出生、重新变 0
    n = n + 1;
    printf("%d\n", n);  // 永远打印 1
}
// 调三次 f()，输出三次都是 1，因为每次 n 都是新的、都从 0 开始
```
⚠️ 别指望普通局部变量"记住上次的值"，它一出函数就被销毁了，下次进来是全新的。

> 小知识：普通局部变量前面其实可以加个 `auto`（如 `auto int n;`），表示"自动变量"，但平时都**省略不写**，知道有这词就行。

### 全局变量：从程序开张活到关门
全局变量"程序开始执行时分配，程序结束时回收"，生存周期是**整个程序**。所以它能跨函数保存数据。

### 静态局部变量 static：长在函数里，但"死不了" ⭐⚠️ 考点
有时候你想要个变量，**既只让一个函数用（作用域局部），又能记住上次的值（生存周期到底）**。这就是 `static` 静态局部变量。

```c
void counter(void) {
    static int n = 0;   // ⭐ static：n 只初始化一次，之后记住上次的值
    n = n + 1;
    printf("%d\n", n);
}

int main(void) {
    counter();          // 输出 1
    counter();          // 输出 2  ← 记住了上次的 1！
    counter();          // 输出 3
    return 0;
}
```
对比上面普通局部变量永远打印 1，加了 `static` 就会 1、2、3 累加。

翁恺总结 static 局部变量的特点，背下来：
- **生存周期贯穿整个程序**（不随函数结束而销毁，存在"静态存储区"，和全局变量同一块地方）。
- **作用域仍只限于定义它的函数内**（出了函数还是看不见，这点和普通局部变量一样）。
- **若没赋初值，系统自动赋 0**（普通局部变量不赋值是随机垃圾值，这是区别）。
- ⚠️⚠️ **赋初值只在第一次调用时执行一次**，后面再调用就跳过初始化，直接用上次留下的值。

🎯 真题最爱拿"`static int n = 0;` 被调用 N 次，问输出"来考你。记死：**static 的初始化只生效一次，值会一直累积。**

### static 局部变量 vs 全局变量（区分考点）⭐
两者都"长寿、住静态区"，区别在**作用域**：

| | 静态局部变量 | 全局变量 |
|---|---|---|
| 生存周期 | 整个程序（一样长） | 整个程序（一样长） |
| 存储位置 | 静态存储区（一样） | 静态存储区（一样） |
| **作用域** | **只限本函数内** | 从定义处到文件结尾 |

记法：**静态局部变量 = 全局变量的"寿命" + 局部变量的"地盘"。** 活得久，但只有它自己家用。

## 十、结构化程序设计思想（理论考点）📚⭐

这一节不用写代码，是翁恺第 5.2.3 节的**纯理论**，选择题/简答题爱考，理解着记。

### ① 自顶向下，逐步细化 ⭐
拿到一个大问题，别上来就埋头写代码。先**把大问题拆成几个小问题，小问题再拆成更小的**，形成一个"树形"层次。每个最小的任务，写成一个函数。这就是"分而治之"。

类比：盖楼不会一块砖一块砖瞎垒，而是先分"打地基、起框架、装修"几大块，每块再细分。

### ② 模块化设计 ⭐
- 一个**模块**对应一个**函数**，**一个模块只干一件明确的事**。
- 模块之间联系要简单：**通过参数传递**沟通，**单入口单出口**，**慎用全局变量**。
- 模块别太大，翁恺建议**一个函数不超过 50 行**，太长说明该再拆了。

### ③ 结构化编码原则（了解，简答可能用）
- 只用**顺序、选择、循环**三种基本控制结构（呼应 M1 讲的三种基本结构）。
- 标识符命名要**见名知义**（变量叫 `age` 别叫 `a`）。
- 加必要的**注释**，保持**缩进**整齐。
- **程序清晰第一，效率第二**（先写对、写清楚，再谈快不快）。
- 良好交互：输入要给提示，输出要有说明。

🎯 简答题"什么是结构化程序设计"答题要点：**自顶向下、逐步细化、模块化、三种基本结构**。

## 十一、递归入门：函数自己调用自己 ⭐（M9 细讲，这里先混个脸熟）

到目前为止，函数都是"main 调用别的函数"。还有一种神奇用法：**函数调用它自己**，这叫**递归**。

### 一句话类比：俄罗斯套娃 🪆
打开一个套娃，里面还有个一模一样小一号的套娃，再打开还有……直到最小那个打不开为止。递归就是这个味道：**自己包含一个更小的自己，直到缩到最简单的情况停下。**

### 一个例子：算 n 的阶乘（n! = 1×2×…×n）
数学上 `n! = n × (n-1)!`，比如 `5! = 5 × 4!`。这本身就是"用更小的自己定义自己"，天生适合递归：

```c
int fact(int n) {
    if (n == 1)            // ① 终止条件：缩到最简单情况，直接给答案
        return 1;
    else
        return n * fact(n - 1);  // ② 自己调用自己（更小的一号）
}
```
`fact(3)` 的执行过程：
```
fact(3) = 3 * fact(2)
              fact(2) = 2 * fact(1)
                            fact(1) = 1   ← 到底了，开始往回算
              fact(2) = 2 * 1 = 2
fact(3) = 3 * 2 = 6
```

### ⚠️⚠️ 递归必须有"终止条件"，否则死循环到崩溃
最关键的一条：递归一定要有个**能停下来的出口**（上面的 `if (n == 1) return 1;`）。没有出口，函数就无限调用自己，像两面镜子对着照，一直照到电脑内存爆掉（"栈溢出"崩溃）。

```c
int bad(int n) {
    return n * bad(n - 1);   // ❌ 没有终止条件，永远停不下来，程序崩溃
}
```

> 🔮 递归比较烧脑，是 M9（翁恺第 10 章，汉诺塔那些）的重头戏。现在你只要记住三件事就够了：
> 1. 递归 = **函数调用自己**。
> 2. **必须有终止条件**，否则死循环崩溃。
> 3. 每次递归要**朝着终止条件靠近**（比如 n 一次次变小）。

## 十二、把知识串起来：一个完整的函数程序

下面这个程序用上了本模块大部分知识点，先读懂每一行，再编译运行：

```c
#include <stdio.h>

// ① 函数声明（原型）——注意分号！放在 main 前面打招呼
int add(int a, int b);
int isEven(int n);
void printLine(void);

int main(void) {
    int x = 7, y = 3;

    printLine();                          // 调用 void 函数，单独成句
    printf("%d + %d = %d\n", x, y, add(x, y));   // 把返回值直接拿来打印

    if (isEven(x))                        // 用返回值当判断条件
        printf("%d 是偶数\n", x);
    else
        printf("%d 是奇数\n", x);
    printLine();

    return 0;
}

// ② 下面是三个函数的定义——注意都没分号，都有 { }
int add(int a, int b) {                   // 要两个int，还一个int
    return a + b;
}

int isEven(int n) {                       // 还 1 表示偶数，还 0 表示奇数
    if (n % 2 == 0)
        return 1;
    return 0;                             // 一个函数里两个 return
}

void printLine(void) {                    // 不还值(void)、不要参数(void)
    printf("------------------\n");
}
```
输出：
```
------------------
7 + 3 = 10
7 是奇数
------------------
```

```bash

```

## 十三、新手最常犯的 8 个错，照着排查 ⚠️

函数这块坑特别多，单独列出来，报错时对着查：

1. **函数定义在 main 后面，又没在前面声明** → 报错 `implicit declaration`（隐式声明）/ `undeclared`。补一行函数声明（带分号！）放 main 前面。
2. **声明忘了加分号** → `int add(int a,int b)` 没分号，电脑当成定义开头去找 `{`，报错。声明结尾**必须有 `;`**。
3. **定义的大括号前多加了分号** → `int add(int a,int b);{...}` ❌。定义首部后面**不能有分号**。
4. **形参类型没各写各的** → `int add(int a, b)` ❌，b 漏了类型。每个形参前都要带类型。
5. **忘了写 return，或 return 没带值** → 非 void 函数必须 return 一个值，忘了会拿到垃圾值。
6. **return 的类型和返回类型对不上** → 比如返回类型写 int，却 `return 小数`，小数部分被砍。
7. **以为传值能改外面的变量**（swap 经典坑）→ 传值改不动原件，要改外面得用指针（M7）。
8. **void 函数硬要接返回值** → `int x = printLine();` ❌，void 啥都不还，没东西接。

### 万能口诀
> **报错先看是不是"声明/定义"的分号问题；逻辑不对先怀疑"传值改不到外面"。**

实在看不懂报错，把**代码**和**完整报错**一起发我。

## 十四、本模块考点速记 ⭐

1. 用函数的好处：**减少重复代码、分而治之（模块化）、提高可读性可维护性**。
2. 函数三要素：**返回类型、函数名、参数列表**；函数体在 `{ }` 里。
3. ⚠️⚠️ **声明（原型）有分号，定义没分号**——M4 头号考点。
4. 函数声明放 main 前面"打招呼"；标准顺序：**声明 → main → 定义**。
5. 形参类型必须**各写各的**：`int add(int a, int b)`，不能 `(int a, b)`。
6. `return`：①结束函数 ②带回一个值；**一次只能返回一个值**；return 后的代码是死代码。
7. 一个函数可以有**多个 return**，走到哪个从哪个出去。
8. `void` = 无返回值；`(void)` = 无参数；void 函数只能单独成句，不能接值；可用 `return;`（不带值）提前结束。
9. ⚠️⚠️ **传值调用**：实参的值**复制**给形参，函数改的是复印件，**改不到外面的原件**（swap 经典题）。要改外面得用**地址/指针**（M7）。
10. **局部变量**只在所在函数内有效；**全局变量**定义在函数外，谁都能用但**慎用**。
11. 同名时**局部变量屏蔽全局变量**（近的盖远的）。
12. 普通局部变量函数结束即销毁，**记不住上次的值**。
13. ⚠️ **静态局部变量 `static`**：只初始化一次，**记住上次的值**，生存周期=整个程序，作用域=本函数；没赋初值自动为 0。
14. 静态局部变量 vs 全局变量：寿命/存储区相同，**作用域不同**（前者只本函数）。
15. **递归** = 函数调自己，**必须有终止条件**，否则死循环崩溃（M9 细讲）。
