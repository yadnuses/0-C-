## 〇、开始之前：把指针的底子捡回来

链表全程都在玩指针，所以先用一分钟把前面学的指针复习一下，不然后面会晕。

回忆 M7 学的三句话：
- **变量是个盒子**，里面装值，比如 `int n = 10;` 就是一个装着 10 的盒子。
- **每个盒子在内存里都有一个"门牌号"（地址）**，用 `&n` 取出来。
- **指针就是专门用来装"门牌号"的盒子**，比如 `int *p = &n;`，p 里装的是 n 的地址，我们说"p 指向 n"。

两个关键符号别搞混 ⭐：
- `&` —— **取地址**（"&n" = "n 住在哪"）。
- `*` —— **顺着地址去找那个盒子**（"*p" = "p 指向的那个盒子里的值"），也叫"解引用"。

```c
int n = 10;
int *p = &n;       // p 指向 n
printf("%d\n", *p); // 输出 10，*p 就是顺着 p 找到 n 的值
*p = 20;            // 通过 p 改 n，现在 n 变成 20
printf("%d\n", n);  // 输出 20
```

记住这个画面，下面动态内存和链表全靠它：

```
      p                 n
   [ 0x100 ] ───────→ [  20  ]
   (装着n的地址)        (地址是0x100)
```

> 还有一个上节学的小尾巴：结构体指针用 `->` 取成员。`p->data` 等价于 `(*p).data`。链表里你会写到手软，先混个眼熟。

## 一、动态内存分配：运行时才决定要多少 ⭐⚠️ 核心

### 1.1 为什么需要它？数组的"先天残疾"

先看普通数组的写法：

```c
int a[5];     // 我要 5 个 int 的空间
```

问题来了：这个 `5` 是**写代码的时候**就钉死的。编译完，这个数组永远就是 5 个，多一个少一个都不行。

可现实里经常是这样：
- 程序要等用户输入"班里有多少人"，**才知道**要开多大的数组。
- 这个人数是程序**跑起来**才晓得的，写代码的时候你根本不知道。

```c
int n;
scanf("%d", &n);
int a[n];        // ⚠️ 老 C 标准（C89）不允许这么写！数组大小必须是编译时常量
```

怎么办？**动态内存分配**就是来救场的：让你在**程序运行的时候**，按需要多少就申请多少。

一句话总结这对矛盾 ⭐：
> **普通数组：大小编译时定死，改不了。动态内存：运行时按需申请，要多少给多少。**

### 1.2 malloc：开口跟系统要内存

申请内存的命令叫 `malloc`（**m**emory **alloc**ate，内存分配），住在 `<stdlib.h>` 里：

```c
#include <stdlib.h>

int *p = (int *)malloc(sizeof(int) * 5);   // 申请能放 5 个 int 的空间
```

这一行是本章第一个硬骨头，我们**逐字**拆开嚼烂：

- `malloc( ... )` —— 命令本体，括号里填"我要多少**字节**"。注意：malloc 只认字节数，不认"几个 int"。
- `sizeof(int)` —— 量出一个 int 占几个字节（一般是 4）。`sizeof` 是个"尺子"，量类型/变量的大小。
- `sizeof(int) * 5` —— 一个 int 是 4 字节，5 个就是 `4 * 5 = 20` 字节。所以这里其实是跟系统要 20 字节。
  - ⚠️ **千万别自己写死 `malloc(20)`**！换台机器 int 可能不是 4 字节，你的程序就崩了。永远写 `sizeof(类型) * 个数`，让尺子自己量，最稳。
- `(int *)` —— **强制类型转换**。malloc 要回来的是一块"裸地址"，它不知道你要拿来装啥，给你的类型是 `void *`（万能指针，啥都能指但不能直接用）。你得用 `(int *)` 告诉它："我要把这块地当成一串 int 来用"。
- `int *p =` —— 把要回来的这块地的**起始地址**，存进指针 p。从此 p 就指向这块新地盘的开头。

申请成功后，这块内存你就能**当数组用**：

```c
p[0] = 10;     // 和数组一模一样的下标写法
p[1] = 20;
p[2] = 30;
printf("%d\n", p[1]);   // 输出 20
```

画个图看 p 和这块内存的关系：

```
   p
[ 0x200 ]──→ [ 10 ][ 20 ][ 30 ][ ? ][ ? ]
              p[0]  p[1]  p[2] p[3] p[4]
              └──── 一共 20 字节，5 个 int ────┘
```

### 1.3 ⚠️⚠️ 申请完必须检查成功没有

系统内存不是无限的。万一内存不够，malloc 会**失败**，这时它返回一个特殊值 `NULL`（空指针，表示"啥也没指着"）。

如果不检查就直接用，程序当场崩溃。所以**专业的写法是申请完先查一下**：

```c
int *p = (int *)malloc(sizeof(int) * 5);
if (p == NULL) {
    printf("内存申请失败！\n");
    return 1;          // 申请失败就别往下走了
}
// 走到这里说明申请成功，可以放心用 p
```

⚠️ 考点：**malloc 可能返回 NULL，用之前要判断 `if (p == NULL)`**。考试里给你一段没检查的代码，问"有什么隐患"，答案就是这个。

### 1.4 ⚠️ malloc 不打扫卫生，里面是垃圾值

刚 malloc 回来的内存，里面装的是**上一个用户留下的垃圾值**，不是 0！

```c
int *p = (int *)malloc(sizeof(int) * 3);
printf("%d\n", p[0]);    // ❌ 输出一个莫名其妙的数（垃圾值），不是 0
```

如果你想要一块**自动清零**的内存，用 `calloc`（**c**lear + alloc）：

```c
int *p = (int *)calloc(5, sizeof(int));   // 申请 5 个 int，全部初始化成 0
printf("%d\n", p[0]);    // ✅ 输出 0
```

malloc 和 calloc 的区别，记这张表 ⭐：

| 函数 | 参数写法 | 申请的内存初始值 | 记忆 |
|------|---------|----------------|------|
| `malloc(总字节数)` | `malloc(sizeof(int)*5)` | **不清零**，是垃圾值 | 快，但脏 |
| `calloc(个数, 单个字节)` | `calloc(5, sizeof(int))` | **全部清零** | 慢一点，但干净 |

⚠️ 注意两个函数**参数形式不一样**：malloc 给一个总数，calloc 给"几个 × 每个多大"两个数，别记混。

### 1.5 ⚠️⚠️ 用完必须 free，否则内存泄漏

malloc 要来的内存，系统**不会自动收回**。你用完了，必须亲手还给系统，这个动作叫 `free`（释放）：

```c
free(p);      // 把 p 指向的那块内存还给系统
```

如果你 malloc 了却忘了 free，这块内存就**一直占着茅坑没人能用**，直到程序结束。程序跑得越久，占的越多，最后内存被吃光——这种病叫**内存泄漏（memory leak）**。

⭐ 黄金法则，背下来：
> **谁 malloc，谁负责 free。一个 malloc 配一个 free。**

### 1.6 好习惯：free 完把指针置 NULL

`free(p)` 只是把那块内存还回去了，但**指针 p 里还存着那块地的旧地址**。这时候 p 叫"野指针"（悬空指针），指着一块已经不属于你的地，再用它就会出事。

```c
free(p);       // 内存还回去了
p[0] = 5;      // ❌❌ 危险！p 还指着已释放的内存，这叫"使用已释放内存"，可能崩溃
```

好习惯是 free 完顺手把它置空：

```c
free(p);
p = NULL;      // ✅ 置空。以后误用它，因为是 NULL 程序会明确报错，比莫名其妙崩溃好查
```

### 1.7 完整范例：动态开一个用户指定大小的数组

```c
#include <stdio.h>
#include <stdlib.h>

int main(void)
{
    int n;
    printf("要存几个数？");
    scanf("%d", &n);                      // 运行时才知道 n 多大

    int *a = (int *)malloc(sizeof(int) * n);   // 按 n 申请
    if (a == NULL) { printf("申请失败\n"); return 1; }

    for (int i = 0; i < n; i++) a[i] = i * i;  // 当普通数组用
    for (int i = 0; i < n; i++) printf("%d ", a[i]);
    printf("\n");

    free(a);       // 用完释放
    a = NULL;      // 置空
    return 0;
}
```

⚠️ 这段就是动态内存的标准模板：**申请 → 检查 → 使用 → 释放 → 置空**，五步缺一不可。

### 动态内存考点速查 ⭐

| 考点 | 要点 |
|------|------|
| 头文件 | `malloc/calloc/free` 都在 `<stdlib.h>` |
| 返回类型 | malloc 返回 `void *`，要强制转换成你的指针类型 |
| 申请大小 | 用 `sizeof(类型) * 个数`，别写死字节数 |
| 失败 | 返回 `NULL`，用前必须判断 |
| 初始值 | malloc 不清零（垃圾值），calloc 清零 |
| 释放 | 用完 `free`，否则内存泄漏；free 后置 `NULL` |

## 二、二级指针：指向指针的指针 ⭐

### 2.1 别被名字吓到，就是"门牌号的门牌号"

我们已经知道：指针是"装地址的盒子"。那……**指针本身也是个盒子呀，它自己也有地址啊！**

于是就有了**二级指针**：专门用来装"指针的地址"的盒子。

打个比方：
- 普通变量 `n` 是一个**房间**，里面放着 10。
- 一级指针 `p` 记着"n 这个房间的门牌号"，p 指向 n。
- 二级指针 `pp` 记着"p 这个房间的门牌号"，pp 指向 p。

一层套一层而已，别慌。

### 2.2 怎么写？

```c
int n = 10;
int *p = &n;       // 一级指针 p 指向 n
int **pp = &p;     // 二级指针 pp 指向 p（注意是两个星号 **）
```

- `int *p` —— 一个星号，"p 是指向 int 的指针"。
- `int **pp` —— 两个星号，"pp 是指向（int 指针）的指针"。
- `pp = &p` —— 把 p 的地址装进 pp。

### 2.3 解引用：一层层剥开

二级指针取值要剥两层皮：

```c
printf("%d\n", n);     // 10  —— 直接看房间
printf("%d\n", *p);    // 10  —— 顺着 p 找到 n
printf("%d\n", **pp);  // 10  —— 顺着 pp 找到 p，再顺着 p 找到 n
```

画成箭头图，一眼看清三层关系 ⭐：

```
    pp              p               n
[ 0x300 ]──→ [ 0x200 ]──→ [   10   ]
 (装p的地址)   (装n的地址)    (真正的值)

 *pp  = p   （pp 剥一层 = p 里的值 = n 的地址）
 **pp = n   （pp 剥两层 = 最终的 10）
```

等价关系背一下（翁恺笔记原话）：
- `pp` ⇔ `&p`
- `*pp` ⇔ `p` ⇔ `&n`
- `**pp` ⇔ `*p` ⇔ `n`

### 2.4 二级指针到底有啥用？—— 在函数里修改"指针本身"

这是二级指针**唯一需要你真正理解**的用途，也是链表头插法的伏笔。

回忆 M7 学过的：想在函数里改一个变量的值，得传它的**地址**（传指针）。

```c
void addOne(int *x) { *x += 1; }   // 传 int 的地址，才能改 int

int n = 5;
addOne(&n);     // 传 &n
// 现在 n = 6
```

那如果我想在函数里改一个**指针**呢？同理——得传这个指针的**地址**，也就是**二级指针**：

```c
void makeIt(int **pp) {
    *pp = (int *)malloc(sizeof(int));  // 通过 pp 修改外面那个指针，让它指向新内存
    **pp = 100;
}

int *p = NULL;
makeIt(&p);     // 传 p 的地址（&p 是 int**）
printf("%d\n", *p);   // 100，p 在函数里被真正改了
```

⚠️ 核心理解：**想改 int 传 `int*`；想改"int 指针"就得传 `int**`。** 链表里如果你写一个函数要修改 head（head 是个指针），就得传 `&head`，也就是二级指针。后面头插法会再点一次。

## 三、链表 —— 本模块核心，考研大题命门 ⭐⚠️⚠️

> 这一节是整个 M10 的重头戏。链表几乎是数据结构入门必考，C 语言考研大题特别爱让你"手写插入/删除函数"。所以我们不背，我们**画图**。每一步都把节点和指针画出来，看懂了图，代码自然就会写。

### 3.1 为什么要链表？数组的痛点

假设有一排同学按个子排好队，存在数组里：

```
数组：[ 张三 ][ 李四 ][ 王五 ][ 赵六 ]
索引：   0       1       2       3
```

现在要在张三和李四中间**插一个新同学**。数组怎么办？后面所有人**都得往后挪一格**给他腾位置：

```
插入前：[ 张三 ][ 李四 ][ 王五 ][ 赵六 ]
插入后：[ 张三 ][ 新人 ][ 李四 ][ 王五 ][ 赵六 ]
                       ↑李四、王五、赵六全被迫后移
```

如果数组有 1000 个人，在开头插一个，999 个人都得挪——**慢死了**。删除同理，删掉一个，后面全得往前补位。

⭐ 数组的死穴：**插入/删除要移动大量元素，效率低 O(n)。**

链表换了个思路：每个同学**手里拉着下一个人的手**（用指针记住下一个人在哪），不要求大家挨着站。

```
head → [张三|·]→[李四|·]→[王五|·]→[赵六|NULL]
```

要在张三李四间插人？**只需改两只手**：让张三松开李四改拉新人，新人去拉李四。其他人纹丝不动：

```
head → [张三|·]→[新人|·]→[李四|·]→[王五|·]→[赵六|NULL]
```

⭐ 链表的优势：**插入/删除只改几根指针，不用搬动数据，O(1)。**

数组 vs 链表，背这张对比表 ⭐⚠️（选择题高频）：

| 对比项 | 数组 | 链表 |
|--------|------|------|
| 内存 | 连续存放 | **不必连续**，散落各处靠指针串 |
| 大小 | 编译时固定 | **动态可变**，随用随加 |
| 插入/删除 | 要移动大量元素，慢 | **只改指针，快** |
| 按下标随机访问 | `a[i]` 直接到，快 | **得从头一个个数**，慢 |
| 额外开销 | 无 | 每个节点多花一个指针的空间 |

> 一句话：**数组擅长"随便取第几个"，链表擅长"频繁插入删除"。** 各有所长。

### 3.2 节点定义：链表的"积木块" ⭐⚠️

链表是由一个个**节点（node）** 串起来的。每个节点必须装两样东西：
1. **数据**本身（要存的值）。
2. **下一个节点的地址**（指向后面那块积木的指针）。

```c
struct Node {
    int data;            // 数据域：存的值
    struct Node *next;   // 指针域：指向下一个节点
};
```

逐字看这个定义，尤其是第二行，它是链表的**灵魂**：

- `int data;` —— 数据域，这里存一个整数。（要存学生就换成 `int num; char name[20];` 等等。）
- `struct Node *next;` —— 指针域，这是个**指向 `struct Node` 类型的指针**。

⚠️⚠️ 注意一个看着很怪的地方：**`struct Node` 自己还没定义完，里面就用到了 `struct Node *`**。这不矛盾吗？

不矛盾。因为 `next` 是个**指针**，指针只是个"地址"，大小是固定的（不管指向多大的东西，指针就那么大）。编译器不需要知道 `Node` 完整长啥样，就能给 `next` 留位置。所以**节点里放一个指向自己同类型的指针，是完全合法的**——这正是链表能"自己串自己"的关键。

> 🎯 真题常考这个定义。让你"补全链表节点结构体定义"，记牢两个域：`数据` + `struct Node *next`。

### 3.3 节点长啥样？画出来

一个节点就是一块小盒子，分两格：

```
  ┌──────┬──────┐
  │ data │ next │
  │  1   │  ·───┼──→ 指向下一个节点
  └──────┴──────┘
```

为了画图方便，我们把节点简写成 `[1|·]`，竖线左边是数据，右边的 `·` 是 next 指针。如果 next 指向空（没有下一个了），就写 `[1|NULL]`。

一条有三个节点的链表，长这样：

```
head → [1|·]→[2|·]→[3|NULL]
```

读法：
- `head` 是**头指针**，记着第一个节点在哪。⚠️ head 本身**不是节点**，它只是一个"指路牌"，指向第一个节点。
- 顺着箭头：1 的 next 指向 2，2 的 next 指向 3。
- 最后一个节点 3 的 next 是 `NULL`，表示"到头了，后面没了"。⚠️ 这个 NULL 是遍历时判断"走到尽头"的信号，极其重要。

> ⭐ 两个关键术语：**头指针 head**（指向第一个节点）、**尾节点**（next 为 NULL 的那个）。

### 3.3.5 ⭐⭐ 考研最大辨析坑：带头结点 vs 不带头结点（必看）

> 你将来刷**王道、真题、408 大题**时，会发现链表代码分成两大流派，写法处处不同，**全看一个前提：这条链表有没有"头结点"。** 这是数据结构里最容易把人搞晕的一个点，提前在这讲清楚，等你学数据结构时就不会一脸懵。

#### ① 先分清两个长得像、其实完全不同的词 ⚠️

- **头指针（head pointer）**：你上面刚学的，就是那个"指路牌"，一个指向链表第一个节点的**指针**。**任何链表都有头指针**（不然找不到链表）。
- **头结点（head node）**：一个**真实存在的节点**，摆在第一个数据节点**前面**，但它的 `data` 域**不存有效数据**（空着或存链表长度），只有 `next` 域有用——指向真正的第一个数据节点。

⚠️ 一字之差：**头指"针"是指针，头结"点"是节点。** 王道书里这俩词满天飞，先咬清楚。

#### ② 两种链表长什么样（画出来对比）

```
【不带头结点】head 直接指向第一个数据节点
   head → [1|·]→[2|·]→[3|NULL]
          ↑第一个就是真数据

【带头结点】head 先指向一个"空壳"头结点，再往后才是数据
   head → [ |·]→[1|·]→[2|·]→[3|NULL]
          ↑这个空壳就是头结点，data 不存数据
```

#### ③ 写法处处不同，对照表背下来 ⭐

| | 不带头结点 | 带头结点 |
|---|---|---|
| 第一个数据节点 | `head` 本身 | `head->next` |
| 判断空表 | `head == NULL` | `head->next == NULL`（头结点恒在） |
| 遍历从哪开始 | `cur = head` | `cur = head->next`（跳过空壳头结点） |
| 在表头插入 | 要特殊处理、还要改 `head` 本身 | 和中间插入**一模一样**，不用特判 |
| 删除第一个数据节点 | 要特殊处理、改 `head` | 和删中间**一模一样** |

#### ④ 头结点到底图什么？—— 让插入/删除"不用特判头部" ⭐

回看本讲义前面 3.12 节那个麻烦："删除头节点是特殊情况，要单独更新 head"。**带头结点就是为了消灭这种特殊情况。**

因为有了那个永远不动的空壳头结点，**第一个数据节点也有"前驱"了**（前驱就是头结点）。于是插入第一个、删除第一个，都能套用"找到前驱 → 改前驱的 next"那套统一逻辑，再也不用单独写 `if(删的是头) head=head->next` 那个分支。代码清爽、不易错——**这就是真题大量用带头结点的原因。**

```c
// 带头结点：在第 i 个位置插入，开头一律先让 p 指向头结点，往后走 i-1 步找到前驱
LNode *p = L;                 // ⚠️ p 从【头结点】起步（不带头结点时是从首元素起步）
for (int j = 0; j < i - 1; j++) p = p->next;   // 走到第 i-1 个的位置
s->next = p->next;            // 先连
p->next = s;                  // 后断（第 i 个位置统一这么插，哪怕 i=1 也不用特判）
```

#### ⑤ 怎么不被坑？一句话判断 ⭐

> **看到遍历 / 判空是 `head->next`（先跳一格）→ 带头结点；是 `head` 直接上 → 不带头结点。**

⚠️ 拿到一道真题先看清它用哪种（题目通常会写明"带头结点的单链表 L"）。**两种的插入删除代码不能混抄**——把带头结点的代码套到不带头结点的链上，第一个元素必出错。

> 📌 一句话记住：**头结点 = 摆在最前面的空壳节点，专门给"第一个数据节点"当前驱，好让插入/删除不用为头部写特殊分支。** 它和你这节学的"不带头结点"链表，判空、遍历、插删的起手式都差一格 `->next`，看真题先认准是哪派。

### 3.4 创建一个节点：三步走 ⭐

每造一个节点，固定三个动作：**申请内存 → 填数据 → next 置空**。

```c
struct Node *p = (struct Node *)malloc(sizeof(struct Node));  // ① 申请一块节点大小的内存
p->data = 1;       // ② 填数据
p->next = NULL;    // ③ 先把 next 置空（暂时不知道接谁，先封口）
```

逐字拆：
- `malloc(sizeof(struct Node))` —— 申请"一个节点这么大"的内存。⚠️ 用 `sizeof(struct Node)` 自动量，别自己算。
- `(struct Node *)` —— 把 void* 转成节点指针。
- `p->data = 1;` —— `->` 是结构体指针取成员，等价于 `(*p).data = 1`。
- `p->next = NULL;` —— **新节点的 next 一定先置 NULL**！这是好习惯，防止 next 里是垃圾值乱指。

画出来：

```
  p
[ · ]──→ [1|NULL]
```

### 3.5 手动串三个节点：理解"串"是怎么发生的

不用任何技巧，纯手动把三个节点连起来，体会指针是怎么"拉手"的：

```c
struct Node *a = (struct Node *)malloc(sizeof(struct Node));
a->data = 1; a->next = NULL;

struct Node *b = (struct Node *)malloc(sizeof(struct Node));
b->data = 2; b->next = NULL;

struct Node *c = (struct Node *)malloc(sizeof(struct Node));
c->data = 3; c->next = NULL;

a->next = b;     // 让 1 拉住 2
b->next = c;     // 让 2 拉住 3
struct Node *head = a;   // 头指针指向第一个节点
```

每一步的图：

```
刚造好（各自独立）：   [1|NULL]   [2|NULL]   [3|NULL]
                       a          b          c

a->next = b 之后：     [1|·]→[2|NULL]   [3|NULL]
                       a      b          c

b->next = c 之后：     [1|·]→[2|·]→[3|NULL]
                       a      b     c

head = a：   head → [1|·]→[2|·]→[3|NULL]
```

看懂这个过程，你就懂了链表的本质：**所谓"链"，就是前一个节点的 next 装着后一个节点的地址。**

### 3.6 遍历链表：从头走到尾的固定套路 ⭐⚠️

有了链表，最常做的事就是**从头到尾把每个节点访问一遍**（打印、查找、求和、计数都靠它）。套路是死的，背下来：

```c
struct Node *cur = head;        // ① 从头开始，cur 是"当前走到哪了"
while (cur != NULL) {           // ② 只要还没走到尽头（NULL）就继续
    printf("%d ", cur->data);   // ③ 处理当前节点（这里是打印）
    cur = cur->next;            // ④ 关键！往后挪一步
}
```

⚠️⚠️ 最容易错的就是第④步。挪到下一个节点**必须写 `cur = cur->next`**，绝对不能写 `cur++`！
- 数组里元素挨着放，`cur++` 能到下一个。
- 链表节点**散落在内存各处**，下一个在哪只有 `cur->next` 知道。`cur++` 只会让指针指向旁边一块莫名其妙的内存，直接玩完。

用箭头图模拟 cur 是怎么"走"的（链表 `head → [1|·]→[2|·]→[3|NULL]`）：

```
第1轮：cur 指向节点1，打印 1，然后 cur = cur->next（指向节点2）
        head → [1|·]→[2|·]→[3|NULL]
               ↑cur

第2轮：cur 指向节点2，打印 2，然后 cur 指向节点3
        head → [1|·]→[2|·]→[3|NULL]
                     ↑cur

第3轮：cur 指向节点3，打印 3，然后 cur = cur->next（变成 NULL）
        head → [1|·]→[2|·]→[3|NULL]
                           ↑cur

第4轮：cur == NULL，while 条件不成立，循环结束。输出：1 2 3
        head → [1|·]→[2|·]→[3|NULL]   cur=NULL（走出去了，停）
```

⭐ 用 for 写也一样，很多教材爱用（翁恺笔记就是 for 版）：

```c
struct Node *ptr;
for (ptr = head; ptr != NULL; ptr = ptr->next) {
    printf("%d ", ptr->data);
}
```

### 3.7 求链表长度（遍历的应用）

数节点个数，就是遍历一遍数数：

```c
int length(struct Node *head) {
    int count = 0;
    struct Node *cur = head;
    while (cur != NULL) {
        count++;
        cur = cur->next;
    }
    return count;
}
```

⚠️ 空链表（head == NULL）时，循环一次都不进，直接返回 0，逻辑天然正确。

### 3.8 头插法建表：新节点永远塞最前面 ⭐⚠️

每次来一个新节点，都把它怼到链表**最前面**，成为新的头。这叫**头插法**。

核心两句，**顺序绝对不能反** ⚠️：

```c
newNode->next = head;    // ① 新节点先拉住原来的第一个
head = newNode;          // ② 头指针再改成指向新节点
```

为什么顺序是这样？我们画图模拟。假设现在链表是 `head → [1|·]→[2|NULL]`，要头插一个 `[9]`：

```
初始状态：
    head → [1|·]→[2|NULL]
    newNode → [9|?]   （刚造好，next 还没接）

① newNode->next = head;  让 9 先拉住现在的头（节点1）
    head → [1|·]→[2|NULL]
            ↑
    newNode → [9|·]──┘   （9 的 next 现在指向节点1）

② head = newNode;  头指针改指向 9
    head → [9|·]→[1|·]→[2|NULL]
    成功！9 成了新的头
```

⚠️⚠️ 如果顺序反了，先写 `head = newNode`，那么 head 立刻指向了 9，**原来的第一个节点 [1] 就找不着了（地址丢失）**，整条链断掉。所以铁律：**先让新节点接住旧头，再移动 head。**

头插法建表完整循环（读入若干数，建成链表）：

```c
struct Node *head = NULL;          // 空表起步，head 先是 NULL
int x;
while (scanf("%d", &x) == 1) {
    struct Node *newNode = (struct Node *)malloc(sizeof(struct Node));
    newNode->data = x;
    newNode->next = head;          // ① 接旧头
    head = newNode;                // ② 移动 head
}
```

⚠️ 注意头插法的"副作用"：**先输入的在最后，后输入的在最前**——也就是结果是**逆序**的。输入 1 2 3，链表会是 `3→2→1`。考试爱考这个细节。

> 💡 还记得二级指针吗？如果你想把头插法包成一个函数 `void insertHead(struct Node **head, int x)`，因为要修改 head 这个指针本身，参数就得用 `struct Node **`（二级指针），调用时传 `&head`。这就是 2.4 节埋的伏笔。

### 3.9 尾插法建表：新节点接在最后面（顺序版）⭐

头插法是逆序，想保持**输入顺序**，就用**尾插法**：新节点永远接在尾巴后面。

尾插法要额外记一个 `tail` 指针（始终指向当前最后一个节点），不然每次都得从头跑到尾，太慢。

```c
struct Node *head = NULL, *tail = NULL;
int x;
while (scanf("%d", &x) == 1) {
    struct Node *newNode = (struct Node *)malloc(sizeof(struct Node));
    newNode->data = x;
    newNode->next = NULL;          // 新节点永远是尾巴，next 置 NULL

    if (head == NULL) {            // 情况A：空表，新节点既是头也是尾
        head = newNode;
        tail = newNode;
    } else {                       // 情况B：非空，接到旧尾巴后面
        tail->next = newNode;      // 旧尾巴拉住新节点
        tail = newNode;            // tail 更新为新尾巴
    }
}
```

画图（已有 `head → [1|·]→[2|NULL]`，tail 指向 2，尾插 [9]）：

```
① tail->next = newNode;  旧尾巴(2)拉住新节点(9)
    head → [1|·]→[2|·]→[9|NULL]
                  ↑tail（还指着2）

② tail = newNode;  tail 挪到新尾巴
    head → [1|·]→[2|·]→[9|NULL]
                       ↑tail
```

头插 vs 尾插，对比记忆 ⭐：

| 建表法 | 核心代码 | 结果顺序 | 要不要 tail |
|--------|---------|---------|-----------|
| 头插法 | `new->next=head; head=new;` | **逆序**（后进的在前） | 不用 |
| 尾插法 | `tail->next=new; tail=new;` | **顺序**（和输入一致） | 要 |

### 3.10 插入节点（在中间插）：改两根指针，先连后断 ⭐⚠️🎯

这是**大题最爱考**的操作。要在某个节点 `prev` 后面插入新节点 `newNode`。

铁律口诀：**先连后断**（先让新节点接住后面的，再让前面的接新节点）。

```c
newNode->next = prev->next;   // ① 新节点先拉住 prev 原来的后继
prev->next = newNode;         // ② prev 改成拉住新节点
```

画图。假设要在 `[1]` 和 `[2]` 之间插 `[9]`，prev 指向 [1]：

```
初始：
    prev
   head → [1|·]→[2|NULL]
   newNode → [9|?]

① newNode->next = prev->next;   让 9 拉住 prev 的后继（节点2）
   head → [1|·]→[2|NULL]
            └──────↑
   newNode → [9|·]─┘   （9 现在也指向节点2）

② prev->next = newNode;   让 1 改拉 9
   head → [1|·]→[9|·]→[2|NULL]
   成功！9 插进了 1 和 2 中间
```

⚠️⚠️ 为什么顺序不能反？如果先写 `prev->next = newNode`（先断），那么 prev 立刻指向 9，**原来 prev->next 指的节点 [2] 就丢了**，找不回来，后半截链全断。所以必须**先用新节点把后继接住，再改 prev**。

> 🎯 真题常见问法："请在单链表中 p 所指节点之后插入 s 所指节点。" 标准答案就两句：
> ```c
> s->next = p->next;
> p->next = s;
> ```
> 顺序写反直接扣分。

### 3.11 删除节点：前驱跳过它，再 free ⭐⚠️🎯

删除也是大题常客。要删掉某个节点，关键是**让它的前驱直接跳过它**，然后**释放被删节点的内存**。

设 `prev` 是被删节点的前驱，`del` 是要删的节点（即 `prev->next`）：

```c
struct Node *del = prev->next;   // del 指向要删的节点
prev->next = del->next;          // ① 前驱跨过 del，直接拉住 del 的后继
free(del);                       // ② 释放被删节点的内存
```

画图。链表 `head → [1|·]→[9|·]→[2|NULL]`，要删中间的 [9]，prev 指向 [1]：

```
初始：
   prev          del
  head → [1|·]→[9|·]→[2|NULL]

① prev->next = del->next;   让 1 跨过 9，直接拉住 9 的后继（节点2）
  head → [1|·]────────→[2|NULL]
              [9|·]→（被甩出链外，但内存还占着）
              ↑del

② free(del);   把节点9的内存还给系统
  head → [1|·]→[2|NULL]
  删除完成，9 的内存也回收了
```

⚠️⚠️ 两个致命坑：
1. **顺序不能反**。必须先用 `prev->next = del->next` 把后继接好，**再 free**。如果先 `free(del)`，那 `del->next` 就读不到了（内存已还），后半截链丢失。
2. **别忘了 free**。只把节点甩出链外、不 free，就是内存泄漏。被删的节点必须释放。

> 🎯 真题："删除单链表中 p 所指节点的后继节点。" 标准三句：
> ```c
> struct Node *q = p->next;   // q 是要删的（p 的后继）
> p->next = q->next;          // p 跨过 q
> free(q);                    // 释放
> ```

### 3.12 删除头节点是特殊情况，要更新 head ⚠️

如果删的正好是**第一个节点**，它没有前驱，得特殊处理——直接移动 head：

```c
struct Node *del = head;
head = head->next;     // head 后移，跳过第一个节点
free(del);             // 释放原来的头
```

```
初始：head → [1|·]→[2|·]→[3|NULL]

head = head->next;   head 挪到第二个
       head → [2|·]→[3|NULL]
       [1|·]（甩出，待 free）

free(del);  回收节点1
       head → [2|·]→[3|NULL]
```

⚠️ 所以写删除函数时，**先判断要删的是不是头节点**：是头就动 head，不是头才用"前驱跳过"那套。这个分支考试要写全才不丢分。

### 3.13 链表操作三大考点小结 ⭐

| 操作 | 核心 | 铁律 |
|------|------|------|
| 遍历/查找 | `while(cur){...; cur=cur->next;}` | 到 NULL 停，挪步用 `cur=cur->next` 不是 `cur++` |
| 插入 | `new->next=p->next; p->next=new;` | **先连后断**，顺序反了断链 |
| 删除 | `p->next=del->next; free(del);` | **先接后删**，删头要单独更新 head |

### 3.14 串起来：一个能跑的完整链表程序

把建表、遍历、求长度、删除拼成一个完整程序，你现在应该能逐行看懂：

```c
#include <stdio.h>
#include <stdlib.h>

struct Node {
    int data;
    struct Node *next;
};

// 尾插法：在链表尾部加一个值为 x 的节点，返回新的 head
struct Node *append(struct Node *head, int x) {
    struct Node *newNode = (struct Node *)malloc(sizeof(struct Node));
    newNode->data = x;
    newNode->next = NULL;
    if (head == NULL) return newNode;        // 空表，新节点就是头
    struct Node *cur = head;
    while (cur->next != NULL) cur = cur->next; // 走到最后一个节点
    cur->next = newNode;                      // 接上
    return head;
}

// 遍历打印
void print(struct Node *head) {
    struct Node *cur = head;
    while (cur != NULL) {
        printf("%d ", cur->data);
        cur = cur->next;
    }
    printf("\n");
}

// 求长度
int length(struct Node *head) {
    int n = 0;
    for (struct Node *cur = head; cur != NULL; cur = cur->next) n++;
    return n;
}

// 删除第一个值为 x 的节点，返回新的 head
struct Node *deleteValue(struct Node *head, int x) {
    if (head == NULL) return NULL;
    if (head->data == x) {                   // 特殊情况：删头节点
        struct Node *del = head;
        head = head->next;
        free(del);
        return head;
    }
    struct Node *prev = head;                // 找前驱
    while (prev->next != NULL && prev->next->data != x)
        prev = prev->next;
    if (prev->next != NULL) {                // 找到了
        struct Node *del = prev->next;
        prev->next = del->next;              // 先接
        free(del);                           // 后删
    }
    return head;
}

// 用完释放整条链表（防内存泄漏）
void freeList(struct Node *head) {
    while (head != NULL) {
        struct Node *del = head;
        head = head->next;     // ⚠️ 先记住下一个，再 free 当前，否则丢链
        free(del);
    }
}

int main(void)
{
    struct Node *head = NULL;
    head = append(head, 1);
    head = append(head, 2);
    head = append(head, 3);
    print(head);                       // 1 2 3
    printf("长度=%d\n", length(head)); // 长度=3

    head = deleteValue(head, 2);       // 删掉 2
    print(head);                       // 1 3

    freeList(head);                    // 用完整条释放
    return 0;
}
```

⚠️ 特别看 `freeList` 里那句注释：**释放当前节点前，必须先用临时指针记住下一个节点**，否则 free 完就找不到后面了。这是新手写释放函数最常翻的车。

## 四、函数指针：指向函数的指针（扩展了解）⭐

> 这部分考研偶尔考个选择题，了解即可，不用像链表那样死磕。

### 4.1 函数也有地址

我们一直说"指针指向变量"。其实**函数在内存里也有地址**（函数代码也得存在内存某处嘛），所以也能有指针指向它。

```c
int add(int a, int b) { return a + b; }

int (*fp)(int, int);    // 定义一个函数指针 fp
fp = add;               // 让 fp 指向 add 函数（函数名就是它的地址）
printf("%d\n", fp(3, 5));      // 输出 8，通过指针调用
printf("%d\n", (*fp)(3, 5));   // 也行，输出 8
```

### 4.2 定义格式：括号别漏

```c
返回类型 (*指针名)(参数类型表);
```

例如 `int (*fp)(int, int);` —— "fp 是个指针，指向'返回 int、收两个 int 参数'的函数"。

⚠️⚠️ `(*fp)` 外面的括号**绝对不能省**！
- `int (*fp)(int, int);` —— fp 是**函数指针**（指向函数的指针）。✅
- `int *fp(int, int);` —— 这成了**函数声明**：fp 是个返回 `int*` 的函数。完全是两码事！

记法：**`*` 和指针名要用括号包住**，才是函数指针。

### 4.3 用途：把函数当参数传

函数指针最大的用处是**让一个函数变通用**——把"具体要做啥"通过函数指针参数传进去。

```c
// 通用计算器：具体怎么算由传进来的 op 函数决定
int calc(int a, int b, int (*op)(int, int)) {
    return op(a, b);
}

int add(int a, int b) { return a + b; }
int mul(int a, int b) { return a * b; }

calc(3, 5, add);   // 8，传 add 进去就做加法
calc(3, 5, mul);   // 15，传 mul 进去就做乘法
```

⚠️ 翁恺笔记里的经典例子是**通用数值积分函数** `calc`，被积函数通过函数指针传入，换个函数就能积分不同曲线。原理一样：函数指针让代码更灵活通用。

## 五、本模块易错坑大盘点 ⚠️

新手在链表上栽的跟头就那么几种，提前看一遍，考场上少丢分：

1. **挪步写成 `cur++`** —— 链表挪到下一个**只能** `cur = cur->next`，节点不连续，`cur++` 必崩。
2. **插入/删除顺序写反** —— 插入"先连后断"，删除"先接后删"。先动旧指针就断链。
3. **删除忘记 free** —— 节点甩出链外不释放 = 内存泄漏。
4. **free 后还用指针** —— free 完那块内存就不是你的了，再读写是"使用已释放内存"。好习惯 free 后置 NULL。
5. **新节点 next 忘了置 NULL** —— 尾节点 next 是垃圾值，遍历时永远到不了 NULL，死循环 / 崩溃。
6. **删头节点不更新 head** —— 头节点没前驱，必须单独 `head = head->next`。
7. **malloc 没检查 NULL** —— 申请可能失败，用前判断。
8. **释放整条链表时丢链** —— 必须先用临时指针存住 `head->next`，再 free 当前节点。
9. **二维指针/二级指针搞混星号个数** —— 改 int 用 `int*`，改"int 指针"用 `int**`。
10. **函数指针定义漏括号** —— `int (*fp)(int)` 才是函数指针，`int *fp(int)` 是返回指针的函数。

## 六、链表口诀（背下来，考场默写）

- **节点 = 数据 + 指向下一个的指针**（`struct Node *next` 指向同类型）。
- **head 是指路牌不是节点**，尾节点 next 为 NULL。
- **遍历**：`while(cur){ ...; cur=cur->next; }`，到 NULL 停，挪步别用 `cur++`。
- **头插法**：`new->next=head; head=new;`（逆序，先接旧头再移 head）。
- **尾插法**：`tail->next=new; tail=new;`（顺序，要记 tail）。
- **插入**：`new->next=p->next; p->next=new;`（**先连后断**）。
- **删除**：`p->next=del->next; free(del);`（**先接后删**，删头单独更新 head）。
- **malloc 配 free，别泄漏**；free 完置 NULL。

## 七、本模块考点速记 ⭐

1. **动态内存三件套** `malloc / calloc / free` 在 `<stdlib.h>`；`malloc(sizeof(类型)*个数)`，返回 `void*` 要强转。
2. **malloc 不清零（垃圾值），calloc 清零**；malloc 给总字节数，calloc 给"个数, 单个大小"。
3. **malloc 可能返回 NULL**，用前必须 `if(p==NULL)` 检查。
4. **谁 malloc 谁 free**，不释放 = 内存泄漏；free 后置 `NULL` 防野指针。
5. **二级指针 `int **pp`**：`*pp` 是指针，`**pp` 是最终值；想在函数里改"指针本身"就传二级指针。
6. **链表节点**含数据域 + 指针域 `struct Node *next`（指向同类型）。
7. **数组 vs 链表**：数组连续、定长、随机访问快、插删慢；链表离散、变长、插删快、随机访问慢。
8. **遍历**用 `cur=cur->next` 到 NULL 停，**不能用 `cur++`**。
9. **头插法逆序**（`new->next=head; head=new;`），**尾插法顺序**（要 tail）。
10. **插入先连后断** `new->next=p->next; p->next=new;`，顺序反了断链。
11. **删除先接后删** `p->next=del->next; free(del);`，删头节点要单独更新 head。
12. **不能返回函数内局部变量的地址**（函数结束局部变量就没了，地址失效）——指针作返回值的铁律。
13. **函数指针** `返回类型 (*名)(参数表)`，括号不能省；用于把函数当参数传，实现通用。
