## 〇、开始之前：数据为什么会"消失"？

到现在为止，你写的所有程序，数据都装在**变量（盒子）**里。可你有没有发现一个问题：**程序一关，盒子里的东西全没了。**

比如你写个程序让用户输入了 100 个学生的成绩，算完平均分，程序一结束——这 100 个成绩就蒸发了。下次想再用，还得重新输一遍。为啥？因为变量住在**内存**里，内存是"临时的桌面"，电脑一断电、程序一退出，桌面就被清空。

那怎么让数据"留下来"？答案就是**文件**。文件存在**硬盘**上，硬盘是"永久的抽屉"，断电、关机、重启都不丢。把数据写进文件，就等于把东西从桌面收进抽屉，下次打开抽屉还在。

这一章就讲一件事：**怎么把数据写进文件（存进抽屉），又怎么从文件读出来（从抽屉拿回来）。**

记住这个核心类比，整章都用它：

> **文件操作 = 开抽屉 → 拿东西/放东西 → 关抽屉。**

## 一、为什么要有文件？文件到底是什么 ⭐

### 文件的作用（选择题爱考）📚

翁恺笔记里把文件的好处总结成三条，背下来就是送分题：

1. **持久化存储**——数据长久保存，程序结束、电脑关机都不丢。（这是最核心的）
2. **处理大数据量**——键盘一次输不了几万条数据，屏幕也显示不下，但文件能装。
3. **数据共享**——一个程序的输出（写进文件），可以当成另一个程序的输入（再读出来）。

### 文件分两类（了解）

- **程序文件**：你写的源代码 `.c`、编译出的 `.exe` 等，是"程序本身"。
- **数据文件**：程序运行时要读写的数据，比如 `成绩.txt`、`账本.dat`。**本章操作的都是数据文件。**

### 文本文件 vs 二进制文件 ⭐⚠️

这是个高频考点，必须分清：

| | 文本文件 | 二进制文件 |
|--|---------|-----------|
| 存的是什么 | 字符的 **ASCII 码** | 数据的**二进制原始形式** |
| 能不能用记事本看 | ✅ 能，是人能读的文字 | ❌ 不能，打开是乱码 |
| 整数 1234 怎么存 | 存 4 个字节：`'1' '2' '3' '4'`（ASCII 49 50 51 52） | 存 2 个字节：`0x04D2`（就是 1234 的二进制） |

一句话区分：**文本文件存"看得懂的字符"，二进制文件存"机器里的原始数据"。**记事本能打开看的就是文本文件，打开是乱码的（比如 `.exe`、图片、`.dat`）就是二进制文件。

> 💡 C 语言把任何文件都看成一串**字节序列（数据流）**，从头到尾一个字节挨一个字节排着。

### 缓冲文件系统（了解，可能考概念）📚

程序和硬盘之间，其实隔着一块**内存缓冲区**。

- **写文件**时：你的数据先进缓冲区，缓冲区**满了**（或你关文件）才一次性刷到硬盘。
- **读文件**时：硬盘数据先进缓冲区，程序再从缓冲区拿。

为啥要中转一道？因为**直接读写硬盘很慢**，攒一批再一次性处理，能大大提高效率。这就叫"缓冲文件系统"，ANSI C 标准规定就用这套。

⚠️ 这也解释了后面一个重点：**为什么必须 `fclose`**——不关文件，缓冲区里没满的那批数据可能还没写进硬盘，就丢了！

## 二、文件操作三步走：开抽屉 → 拿放 → 关抽屉 ⭐⚠️ 全章总纲

C 语言操作文件，**雷打不动就这三步（严格说是四步，加上"定义指针"）**：

```c
FILE *fp;                          // 0. 准备一个"文件指针"（抽屉的把手）
fp = fopen("data.txt", "r");       // 1. 打开文件（拉开抽屉）
if (fp == NULL) { /* 失败处理 */ } // ⚠️ 必须检查开没开成功
// 2. 读 / 写 ...（往抽屉里拿东西或放东西）
fclose(fp);                        // 3. 关闭文件（推回抽屉）
```

把这四步刻进脑子，**整章所有程序都是往这个框架里填东西**。下面一步步拆。

### 第 0 步：`FILE *fp;` 这个"文件指针"是什么？⭐⚠️ 核心概念

```c
FILE *fp;
```

这一行新手最懵：`FILE` 是啥？前面那个 `*` 又是啥？慢慢拆：

- **`FILE`** —— 是 `stdio.h` 里定义好的一种**类型**（它本质是个结构体，里面装着这个文件的缓冲区、读到哪了等一堆信息）。好消息是：**这些内部细节你完全不用管**，C 帮你封装好了。
- **`*`** —— 表示 `fp` 是一个**指针**（M7 学过：指针就是"存地址的盒子"）。
- **`fp`** —— 变量名，你自己起的（习惯叫 `fp`，file pointer 的缩写）。

合起来，`FILE *fp;` 的意思是：**造一个"文件指针"，以后用它来代表、操控我打开的那个文件。**

类比：`fp` 就是**抽屉的把手 / 遥控器**。你不需要知道抽屉内部怎么造的，只要攥住这个把手，就能对抽屉拿放、最后关上。后面所有读写函数，都要把 `fp` 递进去，告诉它"操作的是哪个文件"。

⚠️ 注意写法：是 `FILE`（**全大写**），不是 `file` 也不是 `File`。写错大小写直接报错。

> 🎯 真题常考概念：`FILE` 是在 `stdio.h` 中用 `typedef` 定义的**结构体类型**，程序员只需用 `FILE *` 指针，无需关心其内部成员。`typedef` 的作用是"给已有类型起个新别名"。

## 三、第 1 步：fopen 打开文件 + 必须判 NULL ⭐⚠️⚠️ 改错题最爱

### fopen 的样子

```c
FILE *fp = fopen("data.txt", "r");
```

逐字拆解这一行（一个字都不跳）：

- **`FILE *fp`** —— 造一个文件指针 `fp`（第 0 步和第 1 步合并写了，常见写法）。
- **`=`** —— 把右边 `fopen` 的结果（一个文件指针）装进 `fp`。
- **`fopen`** —— 函数名，**f**ile + **open**，打开文件。
- **第一个参数 `"data.txt"`** —— 要打开的**文件名**（双引号括起的字符串）。可以是 `"data.txt"` 这样的相对路径，也可以写完整路径。
- **第二个参数 `"r"`** —— **打开模式**，这里 `"r"` = read 只读（下一节专门讲模式表）。
- **`;`** —— 结束分号。

`fopen` 干的事：**找到这个文件，拉开抽屉，返回一个把手（文件指针）给你。**

⚠️ 路径里的反斜杠要写两个：Windows 路径 `C:\data.txt` 在代码里要写成 `"C:\\data.txt"`。因为单个 `\` 在字符串里有特殊含义（还记得 `\n` 吗），所以表示"一个真正的反斜杠"要写 `\\`。

### ⚠️⚠️ 重中之重：fopen 可能失败，必须判 NULL！

这是本章**改错题第一高频考点**，请把它纹在心里：

> **`fopen` 不一定成功！打开失败时它返回 `NULL`。你必须检查，否则程序崩溃。**

什么情况会失败？
- 用 `"r"` 模式打开一个**不存在**的文件（抽屉根本不存在，拉不开）。
- 文件没有权限（被锁住、是只读的还想写）。
- 磁盘满了、路径写错了……

如果 `fopen` 返回了 `NULL`，而你**没检查就直接拿 `fp` 去读写**，相当于攥着一个根本不存在的把手去开抽屉，程序当场崩溃（运行时错误）。所以**每次 fopen 之后，立刻判断**：

```c
FILE *fp = fopen("data.txt", "r");
if (fp == NULL) {                       // ⚠️ 雷打不动的检查
    printf("文件打开失败！\n");
    return 1;                           // 或 exit(1); 直接结束程序
}
// 走到这里，说明 fp 一定有效，可以放心读写
```

逐字看这个检查：
- `if (fp == NULL)` —— 问"`fp` 是不是 NULL（开失败了）？"。注意是 `==`（两个等号，比较），不是 `=`。
- 失败了就**打印提示 + 结束程序**（`return 1` 表示非正常退出，或者用 `exit(1)`，需要 `#include <stdlib.h>`）。
- 没失败，程序自然往下走，后面的读写就安全了。

### ❌ 错误 vs ✅ 正确 对照

```c
// ❌ 错误：不判 NULL，文件不存在时直接崩溃
FILE *fp = fopen("不存在.txt", "r");
fscanf(fp, "%d", &n);     // fp 是 NULL，这一步程序崩溃！

// ✅ 正确：先判断再使用
FILE *fp = fopen("不存在.txt", "r");
if (fp == NULL) {
    printf("打不开文件\n");
    return 1;
}
fscanf(fp, "%d", &n);     // 安全
```

🎯 **真题套路**：改错题给你一段文件读写代码，`fopen` 后面**没有 NULL 检查**，让你找错——这就是考点，补上 `if (fp == NULL)` 那段。看到 fopen 没判 NULL，几乎可以闭眼判它错。

> 口诀：**fopen 之后先问 NULL，不问就崩别犯傻。**

## 四、打开模式表：r / w / a / + / b ⭐⚠️ 必背

`fopen` 的第二个参数就是"打开模式"，它决定了你能对文件干啥（只读？只写？追加？）。这张表必须背熟：

| 模式 | 全称 | 干什么 | 文件不存在时 | ⚠️ 关键坑 |
|------|------|--------|-------------|----------|
| `"r"` | read | 只读 | **报错（返回 NULL）** | 文件**必须存在** |
| `"w"` | write | 只写 | 新建 | **会清空原文件！** |
| `"a"` | append | 追加写 | 新建 | 写到**末尾**，不清空 |
| `"r+"` | read+ | 读写 | **报错（返回 NULL）** | 文件**必须存在** |
| `"w+"` | write+ | 读写 | 新建 | **会清空原文件！** |
| `"a+"` | append+ | 读+追加写 | 新建 | 读从头，写到末尾 |

### ⚠️⚠️ 最大的坑：`"w"` 会清空原文件！（高频考点）

这是最容易出事的点，单独拎出来讲：

> **用 `"w"` 模式打开一个已经有内容的文件，它会瞬间把原内容全部清空，从零开始写！**

```c
// 假设 data.txt 里原本存着很重要的 1000 行数据
FILE *fp = fopen("data.txt", "w");   // ⚠️ 这一行执行完，1000 行全没了！变成空文件
```

只要 `fopen(..., "w")` 一执行，**还没等你写任何东西，原文件就被清空了**。新手经常因为这个把数据搞丢。

想往已有文件**接着加内容、不清空**，要用 `"a"`（追加）：

```c
FILE *fp = fopen("data.txt", "a");   // ✅ 原内容保留，新写的接在末尾
```

### 加 `"b"`：二进制模式

在模式后面加个 `b` 就是**二进制模式**，比如：
- `"rb"` —— 二进制只读
- `"wb"` —— 二进制只写（一样会清空）
- `"ab"` —— 二进制追加
- `"rb+"` / `"wb+"` —— 二进制读写

不加 `b` 默认是**文本模式**。二进制模式专门配合后面讲的 `fread/fwrite` 读写结构体用。

### 三个最常用模式速记 ⭐

| 你想干嘛 | 用哪个 |
|---------|--------|
| 读一个已有文件 | `"r"` |
| 新建/重写一个文件（不要旧内容） | `"w"` |
| 往已有文件末尾加内容（保留旧的） | `"a"` |

> 口诀：**r 读须存在，w 写会清空，a 追不删旧，加 b 转二进制。**

## 五、第 2 步：文本读写函数 ⭐⚠️ 重点

抽屉拉开了，现在往里"放东西（写）"或"拿东西（读）"。文本文件的读写函数，**几乎都是你学过的 printf/scanf 加个 `fp` 参数**，超好记。

### ① fprintf：往文件里"打印"（写）⭐

还记得 `printf` 吗？它往**屏幕**打印。`fprintf` 几乎一模一样，只是**第一个参数多了个 `fp`**，表示"打印到这个文件里去"：

```c
printf("%s %d\n", name, age);        // 打印到屏幕
fprintf(fp, "%s %d\n", name, age);   // 打印到文件 fp，多了开头的 fp
```

逐字对比：
- `fprintf` —— **f**ile + printf，往文件打印。
- **第一个参数 `fp`** —— 写到哪个文件（这是唯一的新东西）。
- 后面的 `"%s %d\n"` 和 `name, age` —— 跟 printf 完全一样，格式串 + 要写的值。

例子：把一个学生信息写进文件

```c
FILE *fp = fopen("student.txt", "w");
if (fp == NULL) return 1;
fprintf(fp, "%s %d %.1f\n", "张三", 20, 89.5);   // 文件里就写入了：张三 20 89.5
fclose(fp);
```

### ② fscanf：从文件里"读取"（读）⭐

`scanf` 从**键盘**读，`fscanf` 从**文件**读，同样是第一个参数加 `fp`：

```c
scanf("%d", &n);          // 从键盘读
fscanf(fp, "%d", &n);     // 从文件 fp 读，多了开头的 fp
```

⚠️ 和 scanf 一样，**变量前面要加 `&`**（字符串数组名除外）：

```c
char name[20];
int age;
fscanf(fp, "%s %d", name, &age);   // name 是数组名不加&，age 要加&
```

⚠️ `fscanf` 也用空格/换行当分隔符，规则和 scanf 完全一致。

### ③ fgetc / fputc：一次读/写一个字符 ⭐

如果你想一个字符一个字符地处理（比如复制文件、统计字符数），用这对：

```c
int fputc(int c, FILE *fp);   // 把字符 c 写进文件 fp
int fgetc(FILE *fp);          // 从文件 fp 读一个字符，返回它
```

- `fputc('A', fp);` —— 把字符 `'A'` 写进文件。
- `char ch = fgetc(fp);` —— 从文件读一个字符存进 ch。

⚠️ 注意 `fgetc` 的返回值类型最好用 `int` 接（不是 char），因为读到文件结尾时它返回 `EOF`（一个特殊值，通常是 -1），用 int 才能正确区分。

```c
int ch;
ch = fgetc(fp);
if (ch == EOF) { /* 读到文件结尾了 */ }
```

### ④ fgets / fputs：一次读/写一行（含空格）⭐⚠️

`scanf("%s")` 有个毛病：**遇到空格就停**，读不了"张 三"这种带空格的整行。想读**一整行**（包含空格），要用 `fgets`：

```c
char *fgets(char *buf, int size, FILE *fp);
```

逐字拆解：
- `buf` —— 用来存读到内容的字符数组（盒子）。
- `size` —— 最多读 `size - 1` 个字符（留一个位置放结束符 `\0`）。
- `fp` —— 从哪个文件读。
- 读到**换行符或文件尾**就停，**换行符 `\n` 会被一起读进来**保留在字符串里。

```c
char line[100];
fgets(line, 100, fp);     // 从 fp 读一行（最多99个字符）存进 line，含空格
```

对应的 `fputs` 把字符串写进文件：

```c
int fputs(const char *s, FILE *fp);   // 把字符串 s 写进文件 fp
fputs("hello world\n", fp);           // ⚠️ fputs 不会自动加换行，要换行自己写 \n
```

⚠️ 区别记牢：`fputs` **不写**字符串末尾的 `\0`，也**不自动加换行**，想换行得自己在字符串里写 `\n`。

### 四组读写函数对比表 ⭐

| 函数 | 单位 | 类比谁 | 用途 |
|------|------|--------|------|
| `fprintf` / `fscanf` | 格式化数据 | printf / scanf | 读写混合的数字+文字（最常用） |
| `fputc` / `fgetc` | 单个字符 | putchar / getchar | 一个字符一个字符处理 |
| `fputs` / `fgets` | 一行字符串 | puts / gets | 读写整行文本（含空格） |
| `fwrite` / `fread` | 数据块（字节） | 无 | 二进制读写结构体（第七节讲） |

> 口诀：**带 f 的就是文件版，第一个（或末尾）参数填 fp；printf 家族写，scanf 家族读。**

## 六、判断文件结束 EOF ⭐⚠️⚠️ 最易错的考点

读文件时有个绕不开的问题：**我怎么知道文件读完了，该停了？** 这就是"判断文件结束（EOF, End Of File）"。这一节是本章最容易栽跟头的地方，慢慢看。

### ✅ 推荐写法：靠读取函数的返回值判断

`fscanf` 有个**返回值**：它返回"**成功读到了几个数据项**"。读到文件末尾、没东西可读时，它返回 `EOF`（特殊值）。所以最稳的循环写法是：

```c
int n;
while (fscanf(fp, "%d", &n) == 1) {   // 成功读到 1 个整数才进循环
    printf("%d\n", n);                // 处理读到的 n
}
```

拆解 `while (fscanf(fp, "%d", &n) == 1)`：
- `fscanf(fp, "%d", &n)` —— 试着读一个整数，**返回成功读到的个数**。
- `== 1` —— 因为只读 1 个（一个 `%d`），成功就返回 1。
- 所以这句话是问："**还能成功读到一个整数吗？能就继续，不能（读到结尾）就停。**"

这种写法**一行都不多读**，干净利落，**强烈推荐，考试也这么写**。

> 💡 用 `fgetc` 时同理，靠返回值判断：`while ((ch = fgetc(fp)) != EOF) { ... }`。

### ❌ 易错写法：`while (!feof(fp))` 会多读一次！

很多新手（和老教材）爱写这个：

```c
while (!feof(fp)) {            // ⚠️ 危险写法
    fscanf(fp, "%d", &n);
    printf("%d\n", n);         // 最后一个数会被多打印一次！
}
```

`feof(fp)` 是"检测是否到文件末尾"的函数，到了返回非0（真），没到返回0（假）。`!feof(fp)` 就是"还没到末尾就继续"。听起来很合理，**但它有个致命 bug**。

### 为什么 feof 会多读一次？（讲清原理，重点）⚠️

关键在于：**`feof` 不是"预判"，而是"事后才知道"。** 它只有在你**已经尝试读、并且读失败（碰到了文件尾）之后**，才会变成"真"。

走一遍流程你就懂了。假设文件里只有 3 个数 `10 20 30`：

1. 读到 10，feof 还是假 → 打印 10 ✅
2. 读到 20，feof 还是假 → 打印 20 ✅
3. 读到 30，**注意：此时虽然 30 是最后一个，但还没碰到"尾巴"，feof 仍是假** → 打印 30 ✅
4. 再循环：`fscanf` 去读，**这次没东西了，读失败，碰到了文件尾**，feof 现在才变真——**但这次 fscanf 没读到任何东西，n 还是上一次的 30（旧值）**，于是 → **又打印了一次 30！** ❌

看到了吗？**30 被打印了两次。**问题就出在：feof 是在"读失败之后"才为真的，可这一次失败的读已经发生了，循环体里那次没成功的打印也已经执行了。

### 对照总结 ⭐

```c
// ✅ 正确：靠返回值，多一次都不会
while (fscanf(fp, "%d", &n) == 1) {
    printf("%d\n", n);
}

// ❌ 易错：feof 会在最后多处理一次
while (!feof(fp)) {
    fscanf(fp, "%d", &n);
    printf("%d\n", n);
}
```

🎯 **真题考法**：给你一段 `while(!feof(fp))` 的代码，问"输出结果"或"有什么问题"——答案就是**最后一行/最后一个数据会被重复处理一次**。或者直接让你改成正确写法（用 fscanf 返回值）。

> 口诀：**判结束靠返回值，别信 feof 多读一遍；feof 是读败后才为真，循环里它会坑你一遍。**

## 七、第 3 步：fclose 关闭文件 ⭐

读写完了，**必须关文件**：

```c
fclose(fp);
```

- `fclose` —— **f**ile + **close**，关闭。
- 参数 `fp` —— 关哪个文件。
- 返回值：成功返回 0，失败返回非0（`EOF`）。

### 为什么必须关？（联系缓冲区，重点）⚠️

还记得第一节讲的**缓冲区**吗？你写文件时，数据是先攒在内存缓冲区里的，**缓冲区没满时还没真正写到硬盘**。`fclose` 会做一件关键的事：

> **把缓冲区里还没写完的数据，强制刷进硬盘，然后释放资源。**

如果你**忘了 fclose**，缓冲区里那批"半截数据"可能就丢了——你以为写进文件了，打开一看是空的或不全。所以：

> **写文件后不 fclose = 数据可能丢失。** 这是另一个常见 bug。

⚠️ 三步走里，**fopen 和 fclose 必须成对出现**，就像抽屉拉开了就要推回去。

## 八、完整范例：把三步串起来看 ⭐

光说不练假把式。看两个完整小程序，把"开-写-关""开-读-关"走一遍。

### 范例 1：写文件（把数字存进去）

```c
#include <stdio.h>

int main(void)
{
    FILE *fp = fopen("nums.txt", "w");   // 1. 打开（w 模式，新建/清空）
    if (fp == NULL) {                    // ⚠️ 判 NULL
        printf("打开失败\n");
        return 1;
    }
    for (int i = 1; i <= 5; i++) {
        fprintf(fp, "%d\n", i * 10);     // 2. 写：把 10 20 30 40 50 写进去
    }
    fclose(fp);                          // 3. 关
    printf("写入完成\n");
    return 0;
}
```
运行后，`nums.txt` 文件里就有了：
```
10
20
30
40
50
```

### 范例 2：读文件（把数字读回来求和）

```c
#include <stdio.h>

int main(void)
{
    FILE *fp = fopen("nums.txt", "r");   // 1. 打开（r 模式，只读）
    if (fp == NULL) {                    // ⚠️ 判 NULL
        printf("打开失败\n");
        return 1;
    }
    int n, sum = 0;
    while (fscanf(fp, "%d", &n) == 1) {  // 2. 读：靠返回值循环，读到结尾停
        sum += n;
    }
    fclose(fp);                          // 3. 关
    printf("总和 = %d\n", sum);          // 输出：总和 = 150
    return 0;
}
```

⚠️ 体会一下：这两个程序的骨架完全一样——**fopen → 判NULL → 读写循环 → fclose**。把这套模板背下来，文件题就拿下一大半了。

## 九、二进制读写：fwrite / fread（了解）⭐

前面 `fprintf` 写的是**文本**（人能看懂的字符）。如果想把**一整个结构体**（比如一个学生的全部信息）直接存进文件，用文本一个字段一个字段写太麻烦。这时用**二进制读写** `fwrite`/`fread`，把内存里的数据**原样搬进文件**。

```c
size_t fwrite(const void *ptr, size_t size, size_t n, FILE *fp);   // 写
size_t fread(void *ptr,        size_t size, size_t n, FILE *fp);   // 读
```

四个参数（fwrite 和 fread 一样）：
- `ptr` —— 数据在内存里的地址（写：从哪拿；读：放到哪）。
- `size` —— **一个**数据块多大（字节数），一般用 `sizeof(类型)`。
- `n` —— 写/读**几个**这样的块。
- `fp` —— 操作哪个文件。

### 例子：存一个学生结构体

```c
struct Student { char name[20]; int age; double score; };

struct Student stu = {"张三", 20, 89.5};
FILE *fp = fopen("stu.dat", "wb");        // ⚠️ 二进制写用 "wb"
if (fp == NULL) return 1;
fwrite(&stu, sizeof(stu), 1, fp);         // 把 1 个 stu 结构体原样写进文件
fclose(fp);
```

拆解 `fwrite(&stu, sizeof(stu), 1, fp)`：
- `&stu` —— stu 这个结构体在内存的地址（数据从这儿拿）。
- `sizeof(stu)` —— 一个 stu 有多大（自动算字节数）。
- `1` —— 写 1 个。
- `fp` —— 写进 fp。

读回来：

```c
struct Student stu;
FILE *fp = fopen("stu.dat", "rb");        // ⚠️ 二进制读用 "rb"
if (fp == NULL) return 1;
fread(&stu, sizeof(stu), 1, fp);          // 从文件读 1 个结构体填进 stu
fclose(fp);
printf("%s %d %.1f\n", stu.name, stu.age, stu.score);
```

存**结构体数组**也一样，把 `1` 换成元素个数：

```c
struct Student arr[50];
fwrite(arr, sizeof(struct Student), 50, fp);   // 一次写 50 个
fread(arr,  sizeof(struct Student), 50, fp);   // 一次读 50 个
```

### 二进制 vs 文本，啥时候用哪个？

| | 文本（fprintf/fscanf） | 二进制（fwrite/fread） |
|--|----------------------|----------------------|
| 内容 | 人能看懂的字符 | 机器原始数据，打开是乱码 |
| 适合 | 给人看、跨程序通用 | 整批存结构体、效率高 |
| 优点 | 可读、可编辑 | 快、紧凑、安全（非明文） |

⚠️ 二进制本模块**只要求了解**：知道 `fwrite(&stu, sizeof(stu), 1, fp)` 是"把一个结构体原样存进文件"，能看懂就行。

## 十、文件定位：fseek / ftell / rewind（了解）⭐

文件里有个看不见的"**位置指针**"，记录着"现在读/写到第几个字节了"，每读写一点它就往后挪。一般顺序读写不用管它，但有时想**跳着读**（比如直接跳到第 5 条记录），就要手动挪它。

```c
void rewind(FILE *fp);                          // 把位置指针拨回文件开头
int  fseek(FILE *fp, long offset, int whence);  // 把位置指针移到指定处
long ftell(FILE *fp);                           // 返回当前位置（离文件头多少字节）
```

- **`rewind(fp)`** —— 最简单，**回到文件开头**，想重新从头读时用。
- **`fseek(fp, offset, whence)`** —— 移动位置指针：
  - `offset` —— 偏移多少字节（正数往后，负数往前）。
  - `whence` —— 从哪算起：`SEEK_SET`(0，文件头)、`SEEK_CUR`(1，当前位置)、`SEEK_END`(2，文件尾)。
  - 例：`fseek(fp, 20L, SEEK_SET);` —— 移到离文件头 20 字节处。
- **`ftell(fp)`** —— 告诉你现在在第几字节。配合 `fseek` 到文件尾，能算出**文件总大小**。

### 顺序文件 vs 随机文件（概念，了解）📚

- **顺序文件**：只能从头到尾一条条读写（前面的例子都是）。
- **随机文件**：每条记录**长度固定**，可以用 `fseek` 直接跳到任意一条，速度快。定位公式：**位置 = 记录号 × 每条记录长度**。适合银行账户、订票这种要快速查某一条的场景。

⚠️ 这节整体**了解即可**，知道 `rewind` 回开头、`fseek` 能跳位置、`ftell` 报位置就够了。

## 十一、报错与排查：文件操作的常见坑 ⚠️

文件程序"编译能过但运行出问题"特别常见，对照排查：

1. **程序崩溃（闪退）** → 八成是 **fopen 没判 NULL**，文件没打开成功还硬读。补 `if (fp == NULL)`。
2. **文件写完打开是空的 / 数据不全** → 八成是 **忘了 fclose**，缓冲区数据没刷进硬盘。补 `fclose(fp)`。
3. **原来的数据莫名其妙没了** → 八成是误用了 **`"w"` 模式**清空了文件，应该用 `"a"` 追加。
4. **最后一条数据重复了一次** → 八成是用了 **`while(!feof(fp))`**，改成 `while(fscanf(...)==1)`。
5. **读 double 读出乱值** → 和普通 scanf 一样，`fscanf` 读 double 也要用 **`%lf`**。
6. **路径打不开** → 检查文件名/路径对不对，Windows 路径里 `\` 要写成 `\\`。

> 万能口诀：**崩溃查 NULL，空文件查 fclose，丢数据查 "w"，多一条查 feof。**

## 十二、本模块考点速记 ⭐

> 仿照各模块惯例，把本章高频考点编号列出。复习时重点背这一节。

1. **文件操作四步走**：① `FILE *fp;` 定义指针 ② `fopen` 打开 ③ 读/写 ④ `fclose` 关闭。
2. **`FILE *fp` 是文件指针**：`FILE` 是 `stdio.h` 里用 `typedef` 定义的结构体类型，程序员只用指针不管内部。
3. **fopen 必须判 NULL**：打开失败返回 `NULL`，不判就崩溃——**改错题第一高频考点**。
4. **打开模式**：`"r"` 只读须存在 / `"w"` 只写**会清空** / `"a"` 追加到末尾 / `"r+"` 读写须存在 / 加 `"b"` 二进制。
5. **`"w"` 会清空原文件**——想保留旧内容往末尾加，必须用 `"a"`。高频考点。
6. **文本读写函数**：`fprintf`(像printf) / `fscanf`(像scanf) / `fputc`-`fgetc`(单字符) / `fputs`-`fgets`(整行含空格)，都比对应函数多个 `fp` 参数。
7. **判断文件结束靠返回值**：`while(fscanf(fp,"%d",&n)==1)` 最稳；**`while(!feof(fp))` 会多读一次**——feof 在读失败后才为真，循环里会重复处理最后一条。
8. **必须 fclose**：不关文件，缓冲区数据可能没刷进硬盘而丢失。
9. **二进制读写**：`fwrite(&stu, sizeof(stu), 1, fp)` 把结构体原样存文件，`fread` 读回；配合 `"wb"`/`"rb"` 模式，适合存结构体数组。
10. **fscanf 读 double 同样用 `%lf`**，写 double 用 `%f`（和 scanf/printf 规则一致）。
11. **文件定位（了解）**：`rewind` 回开头，`fseek(fp,offset,whence)` 跳到指定位置，`ftell` 报当前位置；随机文件靠"记录号×记录长度"定位。
12. **文本文件 vs 二进制文件**：文本存 ASCII 字符（记事本可读），二进制存原始数据（乱码）；整数 1234 文本存 4 字节、二进制存 2 字节。
