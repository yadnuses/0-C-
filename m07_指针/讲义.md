## 〇、开始之前：内存和地址到底是什么？⭐⚠️ 地基，必看

学指针之前，必须先把"内存"和"地址"想明白，不然后面全是天书。

### 内存 = 一长排带编号的小格子

想象电脑的内存是**一栋超长的宿舍楼**，楼里有无数个房间，**每个房间都有一个唯一的房间号（门牌号）**。
- 房间 = 一个存储单元（最小是 1 个字节）
- 房间号 = **地址**（一个编号，本质就是个数字）

你在 M1 学的变量（盒子），其实**就是住在内存这栋楼里的某个/某几个房间**。

```
内存这栋楼（每个格子1字节，左边是门牌号/地址）：

地址:  1000   1001   1002   1003   1004   1005  ...
      ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐
      │    │ │    │ │    │ │    │ │    │ │    │
      └────┘ └────┘ └────┘ └────┘ └────┘ └────┘
```

### 变量住进内存：它有"名字"也有"地址"

当你写 `int age = 20;`，电脑做了两件事：
1. 在内存里**划一块房间**给 age 住（int 通常占 4 个字节，即 4 个连续房间）。
2. 把 20 这个值放进去。

```
       age 这个变量（占4个字节，假设从地址1000开始）
       ┌──────────────────┐
地址:  1000               （里面装着值 20）
       │       20         │
       └──────────────────┘
       ↑
       age 的"名字"贴在这块房间上
       age 的"地址"就是 1000
```

所以每个变量都有**两样东西**：
- **名字**（age）—— 给人看的标签，你写代码用它。
- **地址**（1000）—— 这块房间在内存里的门牌号，是个数字。

⚠️ 关键观念：**值和地址是两码事。** age 的"值"是 20，age 的"地址"是 1000（房间号）。指针玩的就是"地址"这个东西。

### 两种找东西的方式：直接 vs 间接 ⭐

翁恺笔记反复强调这组对比，是理解指针的钥匙：
- **直接访问**：直接喊名字拿值。`printf("%d", age);` —— "把 age 房间里的东西拿出来。"
- **间接访问**：先拿到地址（房间号），再顺着地址去那个房间拿东西。"先查到 age 住 1000 号，再去 1000 号敲门看里面住啥。"

指针，就是专门用来玩**间接访问**的工具。下面正式开讲。

## 一、什么是指针 ⭐⚠️ 核心

一句话定义，背下来：

> **指针就是一个专门用来存放"地址"的变量。**

普通变量（盒子）里装的是**值**（20、3.14、'A'）；
指针变量里装的是**另一个变量的地址**（门牌号，比如 1000）。

### 用"门牌号"理解指针

- 变量 `n` 是一间房子，里面住着值 10。
- 指针 `p` 是一张**小纸条**，上面写着 n 房子的**门牌号**（地址）。
- 拿着这张纸条，你随时能顺着门牌号找到 n，进去看里面住啥，甚至换掉里面的东西。

画成箭头图（这张图请刻在脑子里）：

```
   p（指针，存的是地址）            n（变量，存的是值）
   ┌──────────┐                   ┌──────────┐
   │  1000    │ ────────────────> │    10    │
   └──────────┘   "p 指向 n"       └──────────┘
   p 里装的是 n 的地址1000          地址就是1000
```

我们说"**p 指向 n**"，意思就是：p 里存着 n 的地址，顺着它能找到 n。

### 第一段指针代码

```c
int n = 10;        // 普通变量 n，里面装 10
int *p = &n;       // 指针 p，里面装 n 的地址（&n）
```

- `int n = 10;` —— 老朋友，造个装整数的盒子 n，放 10。
- `int *p = &n;` —— 造个**指针** p，让它指向 n。
  - `&n` 是"n 的地址"（取地址，下一节细讲）。
  - 所以这行是"把 n 的地址装进 p"，也就是"让 p 指向 n"。

⚠️ 注意：**p 里装的不是 10，是 n 的地址（比如 1000）**。10 在 n 那儿，不在 p 这儿。新手最容易把这俩搞混。

## 二、两个核心符号：`&` 取地址 和 `*` 解引用 ⭐⚠️⚠️ 全章最重要

整个指针，翻来覆去就玩两个符号。把这两个吃透，指针就赢了一半。

| 符号 | 名字 | 作用 | 大白话 |
|------|------|------|--------|
| `&` | 取地址符 | `&n` 得到 n 的**地址** | "n 住几号房间？"（查门牌号） |
| `*` | 解引用 / 间接访问符 | `*p` 得到 p 所指地址里的**值** | "顺着纸条上的门牌号，去敲门看里面住啥" |

这两个互为逆操作：`&` 是"由值找地址"，`*` 是"由地址找值"。

### `&` 取地址：呼应 M1 的 scanf！⭐

还记得 M1 里 `scanf("%d", &n);` 那个甩不掉的 `&` 吗？现在揭晓答案：

> `&n` 就是"**取出 n 的地址（门牌号）**"。

scanf 要把你键盘输入的值写进 n 房间，**它得先知道 n 住哪个房间**，所以你要把 `&n`（n 的地址）告诉它。这就是 scanf 必须加 `&` 的真正原因！

```c
int n = 10;
printf("%d\n", n);    // 打印 n 的值：10
printf("%p\n", &n);   // 打印 n 的地址：类似 0x7ffe... 一串十六进制（%p 是地址占位符）
```

### `*` 解引用：顺着地址去拿值

`*p` 的意思是："**p 里存着一个地址，顺着这个地址去那个房间，把里面的值拿出来。**"

```c
int n = 10;
int *p = &n;          // p 指向 n
printf("%d\n", *p);   // *p 顺着地址找到 n，拿出 10 → 打印 10
```

配合箭头图理解：

```
   p              n
   ┌──────┐       ┌──────┐
   │ 1000 │ ────> │  10  │
   └──────┘       └──────┘
     p 的值是1000    *p 就是顺着1000找过去，拿到的 10
```

### ⭐⚠️ 重头戏：`*p` 不仅能读，还能写！通过指针改变量

`*p` 既能"读出"n 的值，也能"写入"——给 `*p` 赋值，等于直接改 n！

```c
int n = 10;
int *p = &n;
printf("%d\n", n);    // 10
*p = 99;              // 顺着地址，把 n 房间里的值换成 99
printf("%d\n", n);    // 99！n 真的被改了
```

⚠️ 划重点：`*p = 99;` 完全等价于 `n = 99;`。因为 p 指向 n，操作 `*p` 就是操作 n 本人。这个"**通过指针改别的变量**"的能力，就是指针最大的威力（第七节传参全靠它）。

### ⚠️⚠️ `*` 的两种含义，千万别混（高频坑）

同一个 `*` 符号，在不同位置意思完全不同：

```c
int *p = &n;   // ← 这里的 * 是"声明 p 是个指针"（定义时）
*p = 99;       // ← 这里的 * 是"解引用，取 p 指向的东西"（使用时）
```

- **定义变量时**写的 `*`（`int *p;`）：表示"p 是一个指针类型"。
- **使用变量时**写的 `*`（`*p`、`*p = 99`）：表示"顺着 p 去取/改它指向的值"。

记法：**定义时的 `*` 是名词（"这是指针"），使用时的 `*` 是动词（"去取值")。** 刚学时分不清很正常，多写几遍就有感觉了。

## 三、指针变量的定义与使用（三步走）⭐

标准格式我们拆成"三步走"，跟着做就行。

### 定义格式：`类型名 *指针变量名;`

```c
int  *p;     // p 是"指向 int 的指针"
char *cp;    // cp 是"指向 char 的指针"
double *dp;  // dp 是"指向 double 的指针"
```

- 前面的**类型**（int / char / double）表示"这个指针**指向什么类型**的变量"。
- `*` 表示"这是个指针"。
- 后面是指针的名字。

读法：`int *p;` 读作"**p 是一个指向 int 的指针**"。

⚠️ 为什么指针要分类型？因为指针得知道"它指的房间有多大"（int 占 4 字节、char 占 1 字节），这样 `*p` 取值、`p+1` 跳格子才知道跳多远。**类型必须匹配**：指向 int 的指针只能装 int 变量的地址。

### ⚠️⚠️⚠️ 全场超级大坑：`int *p, q;` 里只有 p 是指针！（必考改错题）

定义多个指针时，**每一个指针前面都要单独写 `*`**：

```c
int *p, q;     // ❌ 陷阱！p 是指针，但 q 只是普通 int！
int *p, *q;    // ✅ 这样 p 和 q 才都是指针
```

为什么？因为 `*` 是**跟着变量名走的**，不是跟着 `int` 走的。`int *p, q;` 拆开看就是"`*p` 是 int"和"`q` 是 int"，所以只有 p 沾上了那个 `*`。

记死这句话：**一行定义几个指针，就得写几个 `*`，一个都不能省。** 这是选择题、改错题的常客。

### 给指针赋值：三种合法写法

```c
int a = 5;
int *p;          // 先定义一个指针 p（此时它是"野指针"，下一节讲）
p = &a;          // ① 让 p 指向 a（最常用）
p = NULL;        // ② 让 p 指向"空"，表示不指向任何东西
int *q = &a;     // ③ 定义的同时就指向 a（初始化，推荐）
```

⚠️ 翁恺特别提醒：**不能用整型常量（除了 0）直接给指针赋值**。比如 `p = 1000;` 是非法的（你不能随便瞎编个数字当地址），但 `p = 0;` 或 `p = NULL;` 是允许的（表示空指针）。

### 使用三步走完整示例

```c
#include <stdio.h>
int main(void)
{
    int a = 5;        // 第1步：有个变量 a
    int *p;           // 第2步：定义指针 p
    p = &a;           // 第3步：让 p 指向 a
    printf("%d\n", *p);   // 5   —— 读 a
    (*p)++;               // 让 a 自增1（注意括号！）
    printf("%d\n", a);    // 6   —— a 真的变成6了
    return 0;
}
```

⚠️ 注意 `(*p)++` 的括号。要先 `*p` 取到 a，再 `++` 加1。如果写成 `*p++` 含义完全不同（那是先用 p 再让 p 自己移位），这个细节考研偶尔抠，先记住"想给指向的值加1，写 `(*p)++`"。

## 四、空指针 NULL 与野指针 ⚠️⚠️ 崩溃之源

指针很强，但也很危险。新手写指针程序最常见的现象就是"莫名其妙崩溃 / 闪退"，**九成是下面两个坑**。

### 空指针 NULL：明确表示"我不指向任何东西"

```c
int *p = NULL;     // p 是空指针，它谁也不指
```

- `NULL` 本质就是地址 0，表示"这是一个不指向任何有效房间的指针"。
- 作用：当你暂时还不知道让指针指哪儿时，先让它 `= NULL`，**明确标记"它现在是空的"**，比让它乱指安全。
- 用前可以检查：`if (p != NULL) { ... }` 确认它确实指向了东西再用。

⚠️⚠️ **绝对不能对 NULL 解引用**：

```c
int *p = NULL;
*p = 10;       // ❌ 程序当场崩溃！（试图往"0号不存在的房间"写东西）
```

### 野指针：没初始化就乱指的指针（最危险）

```c
int *p;        // ⚠️ 只定义没赋值！p 里现在是个随机的垃圾地址
*p = 10;       // ❌ 灾难！往一个随机房间写东西，可能崩溃、可能改坏别的数据
```

- **野指针**：定义了指针但没让它指向任何确定的地方，它里面是上一个住客留下的**随机垃圾值**，被当成地址用就会乱指。
- 野指针比 NULL 更阴险：NULL 解引用一定崩（好歹崩得干脆），野指针可能"时好时坏"，bug 极难找。

### 防坑铁律 ⭐

> **定义指针时立刻初始化**：要么 `int *p = &a;` 指向确定的变量，要么 `int *p = NULL;` 先置空。**绝不让指针处于"定义了却没指向任何东西"的状态。**

```c
int *p1 = &a;     // ✅ 指向确定变量
int *p2 = NULL;   // ✅ 先置空，用前再检查
int *p3;          // ⚠️ 野指针，危险，别这么裸着
```

## 五、指针与数组 ⭐⚠️⚠️ 高频中的高频

这是考研选择题、填空题的重灾区，必须吃透。

### 核心真相：数组名就是"首元素的地址" ⭐

翁恺笔记 8.3.2 第一句话：**数组名是一个地址常量，代表数组首元素（a[0]）的地址。**

```c
int a[5] = {10, 20, 30, 40, 50};
```

画成内存图（数组的5个元素在内存里是**连成一排**的）：

```
        a[0]   a[1]   a[2]   a[3]   a[4]
       ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐
       │ 10 │ │ 20 │ │ 30 │ │ 40 │ │ 50 │
       └────┘ └────┘ └────┘ └────┘ └────┘
地址:  1000   1004   1008   1012   1016
        ↑
        a（数组名）就代表这个地址 1000，也就是 &a[0]
```

所以这两个是完全等价的：

```c
int *p = a;        // 让 p 指向数组首元素
int *p = &a[0];    // 完全一样的意思
```

⚠️ 记住：**`a` 等价于 `&a[0]`**。数组名拿出来用，就是首元素地址。

### 核心等式：`a[i]` ⟺ `*(a + i)` ⭐⚠️⚠️ 必考

这是整个第8章最高频的考点，背到滚瓜烂熟：

> **`a[i]` 等价于 `*(a + i)`**
> （以及指针版：`p[i]` 等价于 `*(p + i)`）

也就是说，**下标访问 `a[i]` 本质上就是指针运算**。`a[2]` 编译器内部其实就是算 `*(a+2)`。

```c
int a[5] = {10, 20, 30, 40, 50};
int *p = a;

printf("%d\n", a[2]);      // 30
printf("%d\n", *(a + 2));  // 30  ← 和上面完全一样
printf("%d\n", p[2]);      // 30  ← 指针也能用下标！
printf("%d\n", *(p + 2));  // 30  ← 四种写法，同一个值
```

四种写法访问的是**同一个房间**。这个等价关系是考研指针题的命根子。

### 用指针遍历数组（两种等价写法对比）

```c
int a[5] = {10, 20, 30, 40, 50};
int i;

// 写法一：传统下标
for (i = 0; i < 5; i++)
    printf("%d ", a[i]);          // 10 20 30 40 50

// 写法二：指针遍历
int *p = a;
for (i = 0; i < 5; i++)
    printf("%d ", *(p + i));      // 10 20 30 40 50（一模一样）
```

### ⚠️ 数组名 vs 指针变量：一个能动一个不能动（考点）

翁恺笔记 8.3.2 第3点强调的区别：
- **数组名 `a` 是地址常量**，它的值锁死了，**`a++` 非法**（你不能让数组名挪位置）。
- **指针 `p` 是变量**，它能改，**`p++` 合法**（让 p 指向下一个元素）。

```c
int a[5] = {10, 20, 30, 40, 50};
int *p = a;
p++;                  // ✅ 合法，现在 p 指向 a[1]
printf("%d\n", *p);   // 20
// a++;               // ❌ 非法！数组名是常量，不能自增
```

## 六、指针的算术运算：p+1 到底加了多少？⭐⚠️

新手最大的误会：以为 `p + 1` 就是地址数字加 1。**错！**

> **`p + 1` 是"跳过一整个元素"，地址实际增加 `1 × sizeof(指向的类型)` 个字节。**

```c
int a[5] = {10, 20, 30, 40, 50};   // int 占 4 字节
int *p = a;        // p 指向 a[0]，假设地址 1000
// p + 1 不是 1001，而是 1004（跳过一个 int = 4 字节）
// p + 2 是 1008
```

图解：

```
        a[0]   a[1]   a[2]
       ┌────┐ ┌────┐ ┌────┐
       │ 10 │ │ 20 │ │ 30 │
       └────┘ └────┘ └────┘
地址:  1000   1004   1008
        ↑      ↑      ↑
        p      p+1    p+2     ← 每 +1 跳 4 字节（一个int），不是跳 1 字节
```

为什么这么设计？这样 `*(p+i)` 才能正好对应 `a[i]`——加 1 就刚好跳到下一个元素，跟下标完美对上。**这就是指针要分类型的根本原因**：编译器靠类型知道"一格"有多大。
- `int *`：p+1 跳 4 字节
- `char *`：p+1 跳 1 字节
- `double *`：p+1 跳 8 字节

### 指针相减：得到"相隔几个元素"

```c
int a[5] = {10, 20, 30, 40, 50};
int *p = &a[1];
int *q = &a[4];
printf("%ld\n", q - p);   // 3  ← 相差3个元素（不是字节数！）
```

⚠️ 两个**同类型**指针相减，结果是它们之间**隔了几个元素**，不是隔了几个字节。

### 指针的关系运算（了解）
同类型指针可以用 `<`、`>`、`==` 比较，判断谁指向的地址在前在后。常用于数组遍历到没到末尾。

## 七、指针作函数参数（传地址）⭐⚠️⚠️ 核心考点，呼应 M4

这是指针**最重要的实战用途**，也是考研大题最爱。先回忆 M4 的痛点。

### 问题：传值调用改不动外面的变量

M4 讲过，普通参数是**值传递**——函数拿到的是实参的**复制品**，改复制品不影响原件。所以下面这个交换函数**根本没用**：

```c
void swap_wrong(int x, int y) {   // 收到的是 a、b 的复制品
    int t = x;
    x = y;
    y = t;
}                                 // 函数里 x、y 换了，但 a、b 纹丝不动

int main(void) {
    int a = 1, b = 2;
    swap_wrong(a, b);
    printf("%d %d\n", a, b);      // 1 2 —— 没换成功！
    return 0;
}
```

为什么失败？画图看：

```
main 里:   a=1   b=2
              │ 复制 │ 复制
              ↓     ↓
swap里:    x=1   y=2   ← 函数操作的是这俩复制品，换了也白换
```

### 解决：传地址！函数收指针 ⭐

把变量的**地址**传进去，函数顺着地址就能改到外面真正的变量：

```c
void swap(int *a, int *b) {   // 收两个地址（指针）
    int t = *a;               // 顺着地址取出 a 指向的值
    *a = *b;                  // 把 b 指向的值写进 a 指向的房间
    *b = t;
}

int main(void) {
    int x = 1, y = 2;
    swap(&x, &y);             // 传地址进去！注意 &
    printf("%d %d\n", x, y);  // 2 1 —— 真的交换了！
    return 0;
}
```

为什么成功？因为传进去的是 x、y 的**门牌号**，函数里的 `*a`、`*b` 顺着门牌号操作的就是 main 里真正的 x、y 本人：

```
main 里:   x=1(地址2000)   y=2(地址2004)
              │ 传地址2000      │ 传地址2004
              ↓                ↓
swap里:    a─────────────>x   b─────────────>y
           *a 就是 x 本人      *b 就是 y 本人 ← 改 *a *b = 改 x y
```

### 传址三件套口诀 ⭐

> 想让函数改外面的变量，记住三步：
> 1. **函数形参写成指针**：`void swap(int *a, int *b)`
> 2. **调用时传地址**：`swap(&x, &y)`（加 `&`）
> 3. **函数里用 `*` 操作**：`*a = *b;`

⚠️ 这就彻底解释了 M1/M5 那个 `scanf("%d", &n)` 为啥要加 `&`：scanf 是个函数，它要把输入**写回你的变量**，所以你必须把变量地址 `&n` 传给它。printf 只是读出来看，不改你的变量，所以不用 `&`。

### 一函数"返回"多个值（指针的妙用）

普通函数 `return` 只能带回一个值。想一次带回多个结果？用多个指针参数：

```c
// 同时算出商和余数，通过指针带回两个结果
void divmod(int x, int y, int *quotient, int *remainder) {
    *quotient = x / y;     // 商写回 quotient 指向的变量
    *remainder = x % y;    // 余数写回 remainder 指向的变量
}

int main(void) {
    int q, r;
    divmod(17, 5, &q, &r);
    printf("商=%d 余=%d\n", q, r);   // 商=3 余=2
    return 0;
}
```

⚠️ 翁恺例8-4（把年内第几天换算成月、日）就是这个套路：用 `*pmonth`、`*pday` 两个指针参数把两个结果同时传回主函数。

## 七点五、⭐⭐ 考研要求的格式：真题里的 `&L` 不是"取地址"！（必看）

> 你将来刷**王道、真题、408 大题**时，会反复看到一种函数写法，长这样：
>
> ```c
> bool ListInsert(SqList &L, int i, ElemType e) { ... }
> //                     ↑ 这个 &L 是什么鬼？
> ```
>
> 一个纯 C 出身的同学看到 `&L`，**99% 会误读成"取 L 的地址"**——因为前面第二节刚学过 `&` 是取地址符。**但在这里它完全不是这个意思。** 这一节专门拆穿这个坑。

### ① 真相：`&L` 是 **C++ 的"引用"语法，不是 C** ⭐⚠️

408 大题的"参考代码"其实写的是 **C++**（大纲允许"C 或 C++"，王道为了写起来省事，普遍用 C++）。在 C++ 里：

```c
void func(SqList &L)    // 形参里的 & 叫"引用"，意思是"L 就是外面那张表本人，不是复制品"
```

- 这里的 `&` **贴在形参类型后面**（`SqList &L`），是**定义引用**，读作"L 是个引用"。
- 它和你学的"取地址 `&n`"**长得一样、意思完全不同**：取地址的 `&` 贴在变量前面去**算出一个地址**；引用的 `&` 贴在形参声明里，表示"这个参数直接就是外面的变量本体"。
- 效果：函数里对 `L` 的任何修改，**直接改到外面的实参**（和传指针的效果一样，能把结果带回去）。

⚠️ 纯 C **没有"引用"这个语法**。你拿 `gcc` 编译一段带 `SqList &L` 的代码会报错，必须用 `g++`（C++ 编译器）才认。

### ② 同一件事，纯 C 怎么写？→ 用指针（你已经学会了）⭐

考研里 `&L`（C++ 引用）能干的事，**纯 C 用指针 `*L` 一模一样能干**——就是你第七节刚学的"传地址三件套"：

```c
// 【考研/王道 写法：C++ 引用】
void ListInsert(SqList &L, ...) {
    L.length++;            // 直接用 . ，改的是外面的表
}
// 调用：ListInsert(L, ...);   ← 调用时不用加任何符号

// 【纯 C 写法：传指针】（功能完全等价）
void ListInsert(SqList *L, ...) {
    L->length++;           // 用 -> ，改的是外面的表
}
// 调用：ListInsert(&L, ...);  ← 调用时要加 & 取地址
```

对照记忆表 ⭐：

| | 考研/C++ 写法（引用） | 纯 C 写法（指针） |
|---|---|---|
| 形参 | `SqList &L` | `SqList *L` |
| 函数里访问成员 | `L.length`（用点） | `L->length`（用箭头） |
| 调用时 | `func(L)`（啥也不加） | `func(&L)`（加 `&` 取地址） |
| 能否改到外面 | 能 | 能 |

### ③ 怎么不被坑？一句话判断 ⭐

> **看到 `&` 贴在"形参类型后面"（如 `SqList &L`）→ 是 C++ 引用，把它当成"传指针"理解就行。**
> **看到 `&` 贴在"变量前面"（如 `&L`、`&n`，出现在调用或 scanf 里）→ 才是取地址。**

⚠️ 一个简单的区分窍门：
- `func(SqList &L)` —— `&` 在**函数定义的形参表**里 → **引用**（C++）。
- `func(&L)` —— `&` 在**调用时的实参**里 → **取地址**（C）。

> 📌 一句话记住：**真题里形参表的 `&L`，脑子里直接换成"这是传进来的那张表本人，等价于纯 C 的 `SqList *L` + `L->`"。** 别被那个 `&` 唬住，它跟取地址没关系。

### ④ 为什么你现在就要知道这件事？

因为 M8 学结构体、M10 学链表后，你照着王道代码敲，**满屏都是 `&L`、`&T`、`&S`**。要是不懂它是 C++ 引用，你会：
1. 误以为要传地址，调用时画蛇添足又加个 `&`；
2. 看不懂为什么函数里能用 `.` 直接改外面的表（明明没传指针啊）。

现在认清"**`&L`（形参）= C++ 引用 = 纯 C 的传指针**"，以后读真题代码就不卡壳了。

## 七点六、⭐ 一张总表：`&` 的三种身份，彻底别再混（必看）

学到这儿，`&` 这个符号你已经在三个完全不同的场合见过它了。新手最容易把它们搅成一锅粥，这里一次性摆平。**关键判断只有一句：先看 `&` 贴在谁身上、在什么位置。**

| 场合 | 长什么样 | 这里的 `&` 是什么 | 怎么读 |
|------|---------|------------------|--------|
| ① 取地址（运算符） | `scanf("%d", &n)`、`p = &n`、`func(&x)` | **取地址符**，算出某变量的门牌号 | "n 的地址" |
| ② 数组名/字符串，**不用 `&`** | `scanf("%s", str)`、`func(arr)` | **没有 `&`！** 数组名本身就是首地址 | "数组名天生是地址，别再取了" |
| ③ C++ 引用（真题形参） | `void f(SqList &L)` | **引用声明**，L 直接就是外面那张表本人 | "L 是个引用" |

三句话区分窍门，背下来：

> - **`&` 贴在变量前、出现在"调用/scanf"里** → 取地址（场合①）。`&n`、`&x`、`&a[i]`。
> - **形参/实参是数组名（或字符串数组）** → 压根不写 `&`（场合②）。它本身是地址，再加 `&` 反而多余/出错。
> - **`&` 贴在"形参的类型后面"** → C++ 引用（场合③）。`SqList &L`、`int &x`，只在 408 真题这种 C++ 代码里出现，纯 C 没有。

⚠️ 最坑的对照（真题里经常同屏出现，务必分清）：

```c
void ListInsert(SqList &L, int i, ElemType e);   // 形参的 &L → 引用（场合③）
// ...
ListInsert(L, 3, x);          // 调用时 L 啥也不加（因为形参是引用，不是要地址）

scanf("%d", &i);              // 这里的 &i → 取地址（场合①），两个 & 意思完全不同！
```

同一段真题代码里，`SqList &L` 的 `&` 和 `scanf("%d", &i)` 的 `&` **是两种东西**——一个是"引用声明"，一个是"取地址"。看位置（类型后 vs 变量前）就能秒辨。

> 📌 一句话收尾：**变量前的 `&` 取地址、数组名不用 `&`、类型后的 `&` 是引用。** 三种身份对号入座，真题代码再不卡壳。

## 八、数组名作函数参数 ⭐⚠️ 高频

接着上一节。数组传进函数，有个重要特点。

### 真相：传数组名 = 传首元素地址（没复制整个数组）

```c
// 下面两种写法完全等价，编译器都当成"收一个指针"
int sum(int a[], int n) { ... }   // 数组形式（看着像数组，其实是指针）
int sum(int *a, int n)  { ... }   // 指针形式
```

⚠️ 重点：把数组传给函数，**传的只是首元素的地址，不是把整个数组复制一份**。所以函数里改数组元素，外面的数组**真的会变**（不像普通变量那样改不动）。

```c
void doubleAll(int a[], int n) {   // a 收到的是数组首地址
    int i;
    for (i = 0; i < n; i++)
        a[i] *= 2;                 // 直接改外面的数组！
}

int main(void) {
    int arr[3] = {1, 2, 3};
    doubleAll(arr, 3);             // 传数组名（=首地址）
    printf("%d %d %d\n", arr[0], arr[1], arr[2]);  // 2 4 6 —— 真的改了
    return 0;
}
```

### ⚠️ 因此：函数里 `sizeof(数组参数)` 量不出原数组大小（考点）

既然函数收到的只是个指针，它就不知道原数组有多长。所以**必须额外传一个长度参数 `n`**（看上面所有例子都带个 n）。这也是为什么 C 里传数组几乎总是"数组 + 长度"成对出现。

### 灵活传"一段"：传 a+7 从中间开始

因为数组名是地址、指针能算术，你可以传"从某个元素开始的一段"：

```c
sum(b, 100);      // 算 b[0]~b[99]
sum(b + 7, 3);    // 从 b[7] 开始算 3 个：b[7]+b[8]+b[9]
```

⚠️ 翁恺还在这节讲了**二分查找（例8-8）**：在**有序**数组里找 x，每次比中间元素 `a[mid]`，比它大就往右半边找、小就往左半边找，范围每次砍一半，效率很高。考研算法题常考，记住"二分查找的前提是数组必须有序"。

## 九、字符指针与字符串 ⭐⚠️

字符串在 C 里就是"一串 char + 末尾一个 `\0`"，和指针关系极密切。

### 字符串常量的本质：以 `\0` 结尾，值是首字符地址

```c
char *s = "hello";   // s 是字符指针，指向字符串首字符 'h'
```

内存里的 "hello"（注意末尾自动有个 `\0` 结束标志）：

```
        s
        │
        ↓
      ┌───┬───┬───┬───┬───┬────┐
      │ h │ e │ l │ l │ o │ \0 │
      └───┴───┴───┴───┴───┴────┘
       s[0] s[1]            末尾 \0 标记字符串结束
```

```c
char *s = "hello";
printf("%s\n", s);      // hello   —— %s 从 s 指的地方一直打印到 \0
printf("%c\n", *s);     // h       —— *s 是首字符
printf("%c\n", s[1]);   // e       —— 字符串也能用下标
```

⚠️ `%s` 占位符配字符串/字符指针，从指针指的位置开始打，**碰到 `\0` 才停**。所以 `\0` 是字符串的"句号"，丢了它打印会冲过头。

### ⚠️⚠️ 字符数组 vs 字符指针：一个能改内容，一个不能（考点）

翁恺 8.4.2 重点区分这两个：

```c
char sa[] = "array";   // 字符数组：自己有一块空间存这些字符，内容【可以改】
sa[0] = 'A';           // ✅ 合法，把 a 改成 A

char *sp = "point";    // 字符指针：指向【字符串常量】，那块是只读的
sp[0] = 'P';           // ❌ 危险！多数系统会崩溃，不能改字符串常量
sp = "hello";          // ✅ 但可以改 sp 的【指向】，让它指向别的字符串
```

记法对比表 ⭐：

| | `char sa[] = "array";` | `char *sp = "point";` |
|---|---|---|
| 本质 | 字符**数组**，自带存储空间 | 字符**指针**，指向只读常量区 |
| 改**内容**（`sa[0]='X'`） | ✅ 可以 | ❌ 不可以（会崩） |
| 改**指向**（`sa = ...`） | ❌ 不可以（数组名是常量） | ✅ 可以 |

### 常用字符串处理函数（需 `#include <string.h>`）⭐

| 函数 | 作用 | 记忆 |
|------|------|------|
| `strlen(s)` | 返回字符串长度（不含 `\0`） | **str** + **len**gth |
| `strcpy(s1, s2)` | 把 s2 复制到 s1 | **cp**y 复制 |
| `strcat(s1, s2)` | 把 s2 接到 s1 末尾 | **cat** 连接 |
| `strcmp(s1, s2)` | 比较两串：相等返0，s1<s2返负，s1>s2返正 | **cmp**are 比较 |

```c
#include <stdio.h>
#include <string.h>
int main(void) {
    char s[20] = "hello";
    printf("%lu\n", strlen(s));     // 5（不算 \0）
    strcat(s, " world");            // s 变成 "hello world"
    printf("%s\n", s);              // hello world
    printf("%d\n", strcmp("abc", "abd"));  // 负数（c < d）
    return 0;
}
```

⚠️ 高频坑：**比较字符串相等不能用 `==`！** `if (s1 == s2)` 比的是两个**地址**，不是内容。要用 `strcmp(s1, s2) == 0` 判断内容相等。

## 十、动态内存：malloc / free 简介（M10 详讲，这里先认脸）⭐

到目前为止，数组大小都得在写代码时定死（`int a[5]`）。但有时**运行时才知道要多大**（比如用户先输入 n，再要 n 个数）。这时用**动态内存分配**。

### 申请和释放（需 `#include <stdlib.h>`）

```c
#include <stdio.h>
#include <stdlib.h>
int main(void) {
    int n;
    scanf("%d", &n);                       // 运行时才知道要几个

    int *p = (int *)malloc(n * sizeof(int)); // 申请 n 个 int 的空间
    if (p == NULL) {                       // ⚠️ 必须检查是否申请成功
        printf("内存不够\n");
        return 1;
    }

    int i;
    for (i = 0; i < n; i++)
        p[i] = i;                          // 当普通数组用：p[i] 或 *(p+i)

    free(p);                               // ⚠️ 用完必须释放，否则内存泄漏
    p = NULL;                              // 好习惯：释放后置空，防止变野指针
    return 0;
}
```

### 四个函数速记（翁恺 8.5.2）

| 函数 | 作用 |
|------|------|
| `malloc(size)` | 申请 size 字节，**不初始化**（里面是垃圾值） |
| `calloc(n, size)` | 申请 n 个 size 字节，**自动清零** |
| `realloc(p, size)` | 调整已申请内存块的大小 |
| `free(p)` | 释放 p 指向的内存 |

### 使用要点（考点）⭐
1. malloc/calloc 返回的是 `void *`，常**强制转换**成需要的指针类型：`(int *)malloc(...)`。
2. **必须检查返回值是否为 NULL**（申请可能失败）。
3. **用完必须 `free`**，否则"**内存泄漏**"（内存被占着不还，越用越少）。
4. **不要访问已经 free 掉的内存**（变成野指针了）。
5. 好处：按需分配，避免固定数组"开大了浪费、开小了不够"。

> 现在只要认得 malloc/free 的脸、知道它们干嘛就行，细节 M10 再深挖。

## 十一、新手最常犯的错（专门排查）⚠️⚠️

指针的 bug 特别隐蔽，这里集中列出，出问题先对照这张表：

1. **`p` 和 `*p` 混淆** —— `p` 是地址，`*p` 是地址里的值。想要值就加 `*`。
2. **野指针**：`int *p;` 定义后没赋值就 `*p = 5;` → 随机崩溃。**定义即初始化**（`= &a` 或 `= NULL`）。
3. **解引用 NULL**：`int *p = NULL; *p = 5;` → 当场崩溃。用前检查 `if (p != NULL)`。
4. **`int *p, q;` 以为 q 也是指针** → 实际只有 p 是。要写 `int *p, *q;`。
5. **指针类型不匹配**：把 double 变量地址塞进 int 指针，取值会乱。类型必须对上。
6. **以为 `p+1` 加1字节** → 实际跳一整个元素（sizeof 个字节）。
7. **scanf 漏 `&`**：`scanf("%d", n)` 漏了取地址 → 崩溃。除非 n 本身已经是指针/数组名。
8. **字符串比较用 `==`** → 比的是地址不是内容。要用 `strcmp(a, b) == 0`。
9. **malloc 后忘记 free** → 内存泄漏；**free 后继续用** → 野指针。
10. **`*p++` 和 `(*p)++` 搞混** → 想给指向的值加1要写 `(*p)++`。

## 十二、本节考点速记 ⭐

1. **内存按字节编号，每个变量都有地址**；变量有"值"也有"地址"，两码事。
2. **指针 = 存放地址的变量**。`int *p = &n;` 读作"p 指向 n"。
3. **`&` 取地址，`*` 解引用（取/改指向的值）**，互为逆操作。`*p = 99` 等价于改 n。
4. `*` 两种含义：定义时表"这是指针"，使用时表"取值"，别混。
5. **`int *p, q;` 只有 p 是指针**，要每个变量前都写 `*`（高频改错题）。
6. **指针必须初始化**：`= &a` 或 `= NULL`，否则成野指针，危险。
7. **不能解引用 NULL 或野指针**，会崩溃。
8. **数组名 = 首元素地址**，`a` 等价 `&a[0]`。
9. **核心等式 `a[i]` ⟺ `*(a+i)`**（必考），指针也能用下标 `p[i]`。
10. **`p+1` 跳一个元素（sizeof 字节）**，不是1字节；指针相减得元素个数。
11. **数组名是地址常量（`a++` 非法）**，指针是变量（`p++` 合法）。
12. **传地址才能让函数改外面变量**：形参用指针、调用加 `&`、函数内用 `*`（swap 经典）。这解释了 scanf 为何加 `&`。
12.5 ⭐ **真题里形参表的 `&L`（如 `SqList &L`）是 C++ 引用，不是取地址**！等价于纯 C 的 `SqList *L` + `L->`。区分窍门：`&` 在形参类型后→引用；`&` 在变量前→取地址。
13. **多个指针参数可"返回"多个值**。
14. **传数组给函数 = 传首地址**，函数里能改原数组；必须额外传长度 n。
15. **字符数组内容可改、字符指针指向常量不可改内容**；`%s` 打到 `\0` 为止。
16. **字符串比较用 `strcmp`，不能用 `==`**。
17. **malloc/calloc/realloc/free**：申请要检查 NULL、用完要 free（防内存泄漏）。
