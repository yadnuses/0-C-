## 〇、开始之前：程序到底是什么？

程序就是**一串命令清单**，电脑从上到下一条一条照着做。

想象你给朋友写纸条："①烧水 ②放面 ③等3分钟 ④关火"。朋友照着做，就完成了煮面。
C 程序也一样：你写一串命令，电脑从上往下一条条执行。

我们写的每一行代码，基本都是在对电脑下一道命令。下面开始认识这些命令。

### 一个最最重要的观念：电脑是个"死脑筋"

电脑不会"猜你想干嘛"。你说一句，它做一句，**一个字都不能错**。
- 你少打一个分号，它就罢工报错。
- 你把大写写成小写，它就当成两个不同的东西。
- 你以为它懂"换行"，但你不写 `\n`，它就把所有字挤成一坨。

所以学 C 的痛苦，一大半来自"标点符号、大小写、空格"这些细节。别灰心，**这不是你笨，是 C 本来就龟毛**。多写几遍就形成肌肉记忆了。

### 写代码用什么字符？必须是英文半角！⚠️

C 代码里所有的符号——括号 `()`、大括号 `{}`、引号 `""`、分号 `;`、逗号 `,`——**全部都要用英文输入法打出来**。
- ❌ 中文的全角分号 `；`、全角括号 `（）`、全角引号 `“”` 都会报错。
- ✅ 只有**双引号里面要给人看的文字**可以是中文（比如 `printf("你好");` 里的"你好"）。

新手最常见的崩溃就是：明明看着一模一样，就是编译不过——一查，是中文标点。记住：**写代码先切到英文输入法。**

## 一、程序的"外壳"——每个C程序都长这样 ⭐

先看这段最小的程序，**所有 C 程序都套着这个壳**：

```c
#include <stdio.h>

int main(void)
{

}
```

逐块解释，一个都不跳：

### ① `#include <stdio.h>`
读作"引入 stdio.h"。
- `stdio` = **st**andard **i**nput **o**utput（标准输入输出），`.h` 是"头文件"。
- 这行的作用：**把打印、输入的功能借过来用**。
- 类比：你想用微波炉，得先把它搬进厨房。`printf`（打印）就住在 `stdio.h` 这个"库"里，不 include 它，你就用不了 printf。
- ⚠️ 这行几乎每个程序开头都要写。

### ② `int main(void)`
这是**主函数**，程序的入口。
- 电脑运行你的程序时，**就是从 main 这里开始执行**的。
- `main` 是固定名字，不能改（改了电脑找不到入口）。
- 前面的 `int` 和括号里的 `void` 先不用纠结，照抄就行，后面模块会讲。
- 类比：一栋楼的大门。电脑进你的程序，必须走 main 这扇门。

### ③ `{ }` 大括号
- 一对大括号framing起 main 的"身体"。
- **你写的命令，全都放在这对大括号中间。**
- 类比：大门进去后的房间，你的家具（代码）都摆在房间里。
- ⚠️ 大括号必须**成对**出现，有左 `{` 就得有右 `}`。新手常漏掉那个 `}`，编译就报错。

### ④ `return 0;`（壳的最后一行）
完整的壳其实长这样：

```c
#include <stdio.h>

int main(void)
{
    // 你的命令写在这里

    return 0;
}
```

最后那行 `return 0;` 是什么意思？
- 它表示"**程序正常结束，没出毛病**"，向电脑交差报告"任务完成，一切正常"。
- `0` 在这里就代表"成功、没错误"。
- 这行写在 main 的最后，照抄就行。深层原因（函数返回值）M4 讲函数时会细说，现在你只要记住：**main 的结尾习惯写一句 `return 0;`**。
- 小提示：现在的编译器其实你不写它也能跑，但考试和规范都要求写，所以**养成习惯一直写上**。

### ⑤ 注释：写给人看的"便利贴"
代码里有时要写点说明文字，提醒自己这段在干嘛。这种"只给人看、电脑会忽略"的文字叫**注释**。

```c
// 这是单行注释，从 // 到这一行结尾，电脑全当没看见

/* 这是多行注释
   可以跨好几行
   电脑同样忽略 */
```

- `//` —— 管到本行末尾，写一句话注释最方便。
- `/* */` —— 成对使用，中间夹的内容（哪怕很多行）都被忽略。
- 用途：给代码做笔记；或者临时"关掉"某行代码不让它运行（叫"注释掉"）。
- ⚠️ 注释不影响程序运行，但能帮你和别人读懂代码。本课的练习里 TODO 提示就是写在注释里的。

记住这个壳，以后每次写程序，先把壳敲出来，再往 `{ }` 里填东西。

## 二、printf：让电脑在屏幕上打字 ⭐

`printf` 是你学的第一个真正的命令，作用是**把文字显示到屏幕上**。

```c
printf("你好，世界\n");
```

拆开看：
- `printf` —— 命令的名字（print + f，f 代表 format 格式化）。
- `( )` —— 括号里放"要打印的东西"。
- `" "` —— **双引号**框住的是一段文字（叫"字符串"），引号里写啥就打印啥。
- `\n` —— 这是**换行符**，让光标跳到下一行。`\n` 算一个整体，不是两个字符。
- `;` —— **分号**，表示"这条命令到此结束"。⚠️ C 里几乎每条命令结尾都要加分号，漏了就报错。

### \n 到底有啥用？看对比

```c
printf("第一行");
printf("第二行");
```
输出（挤在一起）：
```
第一行第二行
```

加上 `\n`：
```c
printf("第一行\n");
printf("第二行\n");
```
输出（分两行）：
```
第一行
第二行
```

⚠️ 所以想换行，就在文字末尾加 `\n`。这是新手最容易忘的。

### 除了 \n，还有几个"转义字符" ⭐

像 `\n` 这种"反斜杠 `\` 开头、代表一个特殊效果"的写法，叫**转义字符**。它在代码里看着是两个符号，其实只算**一个字符**。常用的有：

| 转义字符 | 作用 | 记忆 |
|---------|------|------|
| `\n` | 换行（newline） | 最常用 |
| `\t` | 制表符（Tab），用来对齐 | **t**ab |
| `\\` | 打印一个真正的反斜杠 `\` | 两个 `\` 才出一个 |
| `\"` | 在双引号字符串里打印一个双引号 `"` | 不转义会被当成字符串结尾 |

```c
printf("姓名\t年龄\n");      // 姓名 和 年龄 之间用 Tab 隔开
printf("路径是 C:\\code\n"); // 输出：路径是 C:\code
printf("他说\"你好\"\n");     // 输出：他说"你好"
```

⚠️ 为什么 `\` 要写两个？因为单个 `\` 被 C 当作"转义的开关"，想打印反斜杠本身，就得用 `\\` 告诉它"这是个真的反斜杠，别当开关"。同理，字符串里想要双引号，必须写 `\"`，否则那个 `"` 会被当成字符串提前结束。

## 三、变量：电脑帮你记东西的"盒子" ⭐⚠️ 重点

### 什么是变量？

变量就是**内存里一个有名字的盒子，用来装数据**。

类比：你有个贴了标签的盒子，标签写"年龄"，盒子里放着数字 20。
- 盒子的标签 = 变量名（比如 `age`）
- 盒子里装的东西 = 变量的值（比如 20）
- 你随时可以打开盒子看里面（读取），也可以换成别的数字（修改）。

### 怎么造一个盒子？

```c
int age = 20;
```

逐字拆解：
- `int` —— 盒子的**类型**，这里是 integer（整数）。它规定这个盒子只能装整数。
- `age` —— 盒子的**名字**（你自己起的，见名知意最好）。
- `=` —— **赋值**，意思是"把右边的值放进左边的盒子"。⚠️ 注意：这不是数学的"等于"，是"装进去"的动作。
- `20` —— 要装进盒子的值。
- `;` —— 结束分号。

读这行：**"造一个装整数的盒子，叫 age，里面放 20。"**

### 三种最常用的盒子类型 ⭐

| 类型 | 装什么 | 例子 |
|------|--------|------|
| `int` | 整数 | `int age = 20;` |
| `double` | 小数 | `double price = 3.14;` |
| `char` | 单个字符 | `char grade = 'A';` |

⚠️ 注意 `char` 的值用**单引号** `'A'`，不是双引号。单引号装一个字符，双引号装一串文字。

### 给盒子起名字的规矩 ⭐⚠️
变量名不是随便起的，有几条硬规矩，违反了就报错：
1. **只能用字母、数字、下划线 `_`**。不能有空格、不能有 `+ - * /` 这些符号，不能用中文。
   - ✅ `age`、`my_score`、`num1`、`_count`
   - ❌ `my age`（有空格）、`1num`（数字开头）、`年龄`（中文）、`a+b`（有符号）
2. **不能用数字开头**。`num1` 可以，`1num` 不行。
3. **区分大小写**：`age`、`Age`、`AGE` 是三个完全不同的盒子！⚠️ 这个坑新手常踩。
4. **不能用 C 的关键字**：像 `int`、`return`、`if` 这些是 C 自己的保留字，不能拿来当变量名。
5. **建议见名知意**：存年龄就叫 `age`，存价格就叫 `price`，别叫 `a`、`b`、`x` 让自己以后看不懂。

### 盒子里的值可以换吗？可以！
变量之所以叫"变"量，就是因为里面的值能改：

```c
int score = 60;        // 盒子里先放 60
printf("%d\n", score); // 打印 60
score = 90;            // 把盒子里的值换成 90（注意：第二次不用再写 int）
printf("%d\n", score); // 打印 90
```

⚠️ 关键点：
- **第一次**用 `int score = 60;`——带类型 `int`，叫"声明并赋值"，相当于"造盒子并装东西"。
- **后面再改**只写 `score = 90;`——不写 `int`了，因为盒子已经造好了，只是换里面的东西。
- 如果你第二次又写 `int score = 90;`，电脑会以为你要再造一个同名盒子，反而报错。

### ⚠️ 零基础也要记住的规矩
C 语言里，**盒子必须先声明类型才能用**。不能像某些语言那样直接 `age = 20` 就完事。
必须先 `int age;` 告诉电脑"我要一个装整数的盒子叫 age"，才能往里放东西。

### 还有一种"盒子"：常量（值不会变的那种）⭐

变量是"能改的盒子"，那有没有"不准改的盒子"？有，叫**常量**。

最常见的就是**直接写在代码里的数字 / 字符**，它们本身就是常量：
- `20`、`3.14`、`'A'`、`"你好"` —— 这些写死在代码里的值都叫常量（也叫"字面量"）。
- 类比：变量是可擦写的白板，常量是刻在石头上的字。

如果你想给一个"永远不变的值"起个名字（比如圆周率），可以用 `const`：

```c
const double PI = 3.14;   // 给常量起名叫 PI，值锁死成 3.14
PI = 3.15;                // ❌ 报错！const 修饰的盒子不准改
```

- `const` 加在类型前面，表示"这个盒子造好后就**焊死了，不准再改**"。
- 好处：防止手滑改错；也让读代码的人一眼知道"这个值是固定的"。
- ⚠️ 现在记住有这么个东西就行：**`const` = 只读盒子**。考试里见到 `const` 别慌，就是"不能改的变量"。

### 赋值 `=` 的真相："先算右边，再装进左边" ⭐⚠️

前面说 `=` 是"把右边的值放进左边的盒子"。再补一个关键细节：**电脑永远是先把右边算完，再整个塞进左边的盒子。**

正因为这样，下面这行**看起来像病句，其实完全合法**，而且超常用：

```c
int x = 5;
x = x + 1;     // 这不是数学方程！
```

数学里 `x = x + 1` 是无解的废话，但在 C 里它的意思是：
1. **先算右边** `x + 1`：此时 x 是 5，算出 6。
2. **再把 6 装回左边的盒子 x**：现在 x 变成 6。

所以 `x = x + 1` 读作"**把 x 自己加 1 后再存回 x**"，是"让盒子里的数加一"的标准写法。

⚠️ 千万别用数学的"等于"去理解 `=`。C 里的 `=` 是**动作**（装进去），不是**判断**（相等）。判断两个数相等用的是 `==`（两个等号），这个 M2 会讲，现在先有个印象：**一个 `=` 是赋值，两个 `==` 才是比较。**

## 四、printf 打印变量：占位符 ⭐⚠️ 高频考点

光打印固定文字没意思，我们想打印盒子里的值。问题来了：

```c
int age = 20;
printf("我 age 岁");   // ❌ 这样会原样打印"我 age 岁"，不会显示20
```

因为双引号里的东西**一律当文字原样输出**。想插入变量的值，要用**占位符**。

```c
int age = 20;
printf("我 %d 岁\n", age);   // ✅ 输出：我 20 岁
```

原理：
- `%d` 是个**占位符**，先占着位置，像填空题的横线。
- 逗号后面的 `age` 是"用来填空的值"。
- 运行时，电脑把 `age` 的值（20）填进 `%d` 的位置。

### 常用占位符（对应三种盒子）⭐

| 占位符 | 配哪种类型 | 记忆 |
|--------|-----------|------|
| `%d` | int 整数 | **d**igit 数字 |
| `%f` | double 小数 | **f**loat 浮点；`%.2f` = 保留2位小数 |
| `%c` | char 字符 | **c**haracter 字符 |

### ⚠️ 占位符必须和类型配对（改错题高频）

```c
int age = 20;
printf("%f\n", age);   // ❌ 错！age是整数(int)，却用了小数占位符%f，输出乱码
printf("%d\n", age);   // ✅ 对
```

记法：**整数配 %d，小数配 %f，字符配 %c**，配错就出乱码。

### 多个占位符

```c
int age = 20;
double height = 1.75;
printf("我 %d 岁，身高 %.2f 米\n", age, height);
// 输出：我 20 岁，身高 1.75 米
```

⚠️ 规则：前面有几个 `%`，后面逗号就跟几个值，**顺序一一对应**。第一个 `%d` 配 `age`，第二个 `%.2f` 配 `height`。

### 进阶：控制打印的"宽度"和"小数位"（了解即可）⭐

有时候想让输出对得整整齐齐，可以在 `%` 和字母中间加数字：

```c
printf("%5d\n", 42);     // 输出：   42   （总宽度5位，不够的前面补空格）
printf("%.2f\n", 3.14159);  // 输出：3.14    （小数点后保留2位）
printf("%6.2f\n", 3.14159); // 输出：  3.14  （总宽6位，其中小数2位）
```

拆解：
- `%5d` —— `5` 是**最小宽度**，数字不够 5 位就在前面填空格凑够。常用来让一列数字右对齐。
- `%.2f` —— `.2` 是**小数位数**，保留 2 位小数（会四舍五入）。这个最常用，比如打印金额。
- `%6.2f` —— 合起来：总宽 6 位，小数 2 位。

⚠️ 入门阶段你只要会用 **`%.2f` 保留两位小数**就够了，宽度控制 `%5d` 见到知道是"对齐用的"即可，不用死记。

## 五、scanf：让用户从键盘输入 ⭐⚠️ 重点中的重点

前面都是程序自说自话。`scanf` 能让程序**停下来等你打字**，把你输入的值存进盒子。

```c
int n;
scanf("%d", &n);
```

逐字拆解：
- `int n;` —— 先造一个空盒子 n（这次没赋初值，等用户来填）。
- `scanf` —— 命令名（scan + f，扫描格式化输入）。
- `"%d"` —— 告诉电脑"我要读一个整数"。
- `&n` —— **把输入的值存进盒子 n**。

### ⚠️⚠️ 全场最大的坑：那个 `&` 别忘了！

```c
scanf("%d", &n);    // ✅ 正确，n 前面有 &
scanf("%d", n);     // ❌ 错误，漏了 &，程序会崩溃
```

为什么要加 `&`？现在你只需记住一句话：
> **scanf 里，变量名前面必须加 `&`。**

深层原因（指针，M7 会细讲）：`&n` 表示"n 这个盒子的位置（地址）"。scanf 要知道"把用户输入放到哪个盒子里"，所以要给它盒子的位置。`printf` 只是读出来给你看，不用改盒子，所以**printf 不加 `&`**。

记法对比：
- `printf("%d", n)` —— 读出来看，**不加** &
- `scanf("%d", &n)` —— 写进去存，**要加** &

### 完整的"对话"流程

```c
int n;
printf("请输入一个整数: ");   // 提示用户
scanf("%d", &n);              // 等用户打字，存进 n
printf("你输入的是 %d\n", n); // 把存的值打印出来
```

运行时：屏幕先显示"请输入一个整数:"，光标停在那等你；你打个 5 回车；程序接着打印"你输入的是 5"。

### 一次输入多个值
scanf 也能一次读好几个值，多个占位符、多个 `&` 变量，一一对应：

```c
int a, b;
printf("输入两个整数（用空格隔开）: ");
scanf("%d %d", &a, &b);    // 比如你输入：3 5
printf("a=%d, b=%d\n", a, b);  // 输出 a=3, b=5
```

⚠️ 输入时两个数字之间用**空格或回车**隔开都行。每个变量前面都得有自己的 `&`，一个都不能少。

### ⚠️⚠️ 读小数要用 `%lf`，不是 `%f`（超高频坑）
打印（printf）小数用 `%f`，但**输入（scanf）一个 double 小数，必须用 `%lf`**（多一个 l，读作"L"，代表 long）：

```c
double price;
scanf("%lf", &price);     // ✅ 读 double 用 %lf
printf("%f\n", price);    // ✅ 打印 double 用 %f（或 %.2f）
```

```c
scanf("%f", &price);      // ❌ 读 double 用 %f，读进来是错的乱值
```

记死这张配对表 ⭐：

| 类型 | scanf 读用 | printf 打印用 |
|------|-----------|--------------|
| int 整数 | `%d` | `%d` |
| double 小数 | `%lf` | `%f` |
| char 字符 | `%c` | `%c` |

⚠️ 唯一不对称的就是 double：**读用 `%lf`，打印用 `%f`**。这是新手最容易栽的地方。

## 六、运算符：让电脑算数 ⭐

```c
+    加
-    减
*    乘（注意是星号，不是 ×）
/    除
%    取余数（余数！不是百分号）
```

例子：

```c
int a = 7, b = 3;
printf("%d\n", a + b);   // 10
printf("%d\n", a - b);   // 4
printf("%d\n", a * b);   // 21
printf("%d\n", a / b);   // 2  ← 注意！不是2.33
printf("%d\n", a % b);   // 1  ← 7除以3余1
```

### ⚠️⚠️ 整数除法陷阱（考试最爱考）

```c
printf("%d\n", 5 / 2);     // 结果是 2，不是 2.5！
```

为什么？**两个整数相除，结果也是整数，小数部分直接被砍掉。**

想得到 2.5 怎么办？让算式里**有小数参与**：

```c
printf("%.1f\n", 5.0 / 2);   // 2.5  ← 把5写成5.0
printf("%.1f\n", 5 / 2.0);   // 2.5  ← 或者把2写成2.0
```

⚠️ 记死这条：**整数 ÷ 整数 = 整数（砍小数）**。这是选择题、改错题的常客。

### `%` 取余是干嘛的？
取余 = 求"除不尽剩下多少"。
- `7 % 3` = 1（7里有2个3，剩1）
- `10 % 2` = 0（整除，余0）—— ⚠️ 余0常用来判断偶数！
- `8 % 5` = 3

⚠️⚠️ **`%` 取余只能用在整数上**！两边都必须是整数：

```c
printf("%d\n", 7 % 3);     // ✅ 对，整数取余
printf("%f\n", 7.5 % 2);   // ❌ 报错！小数不能取余
```

记住：**`/` 和 `%` 配合用在整数上**——`/` 拿商，`%` 拿余数。小数没有"余数"这一说。

## 七点五、报错了别慌：看懂编译器在骂什么 ⚠️

新手一看到一屏红字就吓到了。其实编译器是在帮你，它会告诉你**第几行、错在哪**。学会看报错，你就能自己救自己。

报错信息一般长这样（下面是个**虚构的小例子**，文件叫 hello.c，跟本模块那个能正常跑的 demo.c 没关系）：

```
hello.c:9:5: error: expected ';' after expression
    printf("hello")
                  ^
```

怎么读？
- `hello.c:9:5` —— 出错在 **hello.c 文件第 9 行第 5 列**。先去第 9 行看。
- `error: expected ';'` —— 错误内容：这里**少了一个分号 `;`**。
- 下面那个 `^` 箭头 —— 指着出问题的大概位置。

### 新手最常见的 5 个错，照着排查
1. **少分号** `;` —— 报错带 `expected ';'`。去那一行末尾补分号。
2. **中文标点** —— 把 `；（）""` 换成英文 `;()""`。报错可能很怪，先怀疑这个。
3. **大括号 / 引号没配对** —— 有 `{` 没 `}`，或者引号只写了一半。数一数是否成对。
4. **变量没声明就用** —— 报错带 `undeclared`（未声明）。检查是不是名字打错了，或者忘了写 `int xxx;`。
5. **占位符配错类型** —— 编译能过但输出是乱码/怪数字。回去检查 `%d %f %c` 有没有配对。

### 万能口诀
> **报错先看行号 → 跳到那一行 → 八成是分号、中文标点、或括号没配对。**

实在看不懂报错，就把**你的代码**和**完整报错**一起发给我，我帮你看。

## 七点六、第1章理论速记（专攻选择题）📚⭐

> 前面讲的是"怎么写代码"（动手）。这一节是翁恺第1章里那些**不用写代码、但选择题爱考的纯理论**。背下来就是送分题。第一次学可以先扫一眼，复习时重点记。

### ① 程序的三种基本结构 ⭐⚠️ 必考
任何程序，再复杂，都是由这**三种结构**搭出来的：
1. **顺序结构**——从上到下一条条执行（我们现在写的都是它）。
2. **分支结构**（也叫选择结构）——"如果……就……否则……"，M2 学的 `if`。
3. **循环结构**——"重复做某件事直到满足条件"，M3 学的 `for`/`while`。

⚠️ 选择题常问"C 语言的三种基本控制结构是哪三种"，答案就是**顺序、分支（选择）、循环**。

### ② 冯·诺依曼体系结构（了解）
现代电脑的设计思想叫"**存储程序**"：把**程序和数据一起存在内存里**，CPU 从内存里一条条取出指令来执行。提出者是**冯·诺依曼**。
- 选择题可能问"存储程序思想是谁提出的"→ 冯·诺依曼。

### ③ 编译只查"语法"，不查"逻辑" ⭐⚠️ 高频
这是个特别容易考的点：
- **编译器只检查你语法对不对**（分号、括号、拼写……），它**管不了你算法对不对**。
- 举例：你想算 `a+b` 结果写成了 `a-b`，编译**照样通过**，因为语法没毛病，只是结果错了。这种叫**逻辑错误**，编译器抓不到，得你自己运行后发现。
- 对比：少个分号是**语法错误**，编译器当场报错。

⚠️ 记牢这组对比：**语法错误→编译器会报；逻辑错误→编译器不报，要运行才发现。**

### ④ C 语言的几个"数字"考点（了解）
- C 语言一共有 **32 个关键字**（`int`、`return`、`if`、`for` 这些保留字）。
- C 有 **9 种控制语句**。
- 数量不用死背，但选择题偶尔会问"C 有多少个关键字"，留个印象：**32 个**。

### ⑤ 算法与流程图（了解）
- **算法**：解决问题的步骤。好算法有 5 个特征——**有穷性、确定性、可行性、有输入、有输出**。
- **流程图**：用图形把算法画出来。常见框：椭圆=开始/结束，矩形=处理，菱形=判断，平行四边形=输入/输出。
- 选择题可能给个流程图问你"菱形代表什么"→ 判断/分支。

## 八、本节考点速记 ⭐

1. 程序壳：`#include` + `int main(void)` + `{ }` + `return 0;`
2. 代码符号全用**英文半角**；只有双引号里给人看的文字能用中文
3. `printf` 打印，`\n` 换行，结尾分号 `;`
4. 变量先声明类型再用：`int / double / char`；变量名不能数字开头、区分大小写、不能用中文
5. 再次给变量赋值**不写类型**：`score = 90;`（不是 `int score = 90;`）
6. 占位符配对：`%d`整数 `%f`小数 `%c`字符，配错出乱码
7. `scanf` 变量前**必须加 `&`**；printf 不加
8. **整数除法砍小数**：`5/2=2`，要小数得 `5.0/2`
9. `%` 取余数，`x % 2 == 0` 判断偶数；**取余两边必须是整数**
10. 报错先看**行号**，八成是分号 / 中文标点 / 括号没配对
11. **scanf 读 double 用 `%lf`**，printf 打印 double 用 `%f`（唯一不对称，超高频坑）
12. `=` 是赋值（装进去），`==` 才是比较；`x = x + 1` 合法，意思是"x 加1后存回 x"
