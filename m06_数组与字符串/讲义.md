## 〇、开始之前：为什么要有数组？

前面 M1 学过"变量是一个有标签的盒子，装一个数据"。

那问题来了：如果我要存**全班 50 个同学的成绩**呢？难道要造 50 个盒子？

```c
int score1, score2, score3, score4, score5;   // ……写到 score50？要疯了
```

太蠢了。于是 C 给了我们一个神器——**数组**。

数组就是**一排连在一起、带编号的盒子**，它们类型相同、名字相同，靠"编号"区分谁是谁。

一个绝佳的类比：**一栋楼的房间号**。
- 整栋楼有一个名字（数组名，比如 `score`）。
- 每个房间有个门牌号（下标，比如 0、1、2……）。
- 每个房间住一个人（一个数据）。
- 你想找某个人，就报"楼名 + 门牌号"：`score[3]` = score 楼的 3 号房。

⚠️ 注意这栋楼的门牌号**从 0 开始数**，不是从 1。这是 C 语言全场最大的坑之一，下面专门讲。

记住一句话：**数组 = 一串同类型的盒子，用下标（编号）挑出其中一个来用。**

## 一、一维数组：怎么造、怎么用 ⭐⚠️ 重点

### 1. 怎么造一排盒子（定义）

```c
int a[5];
```

逐字拆解，一个都不跳：
- `int` —— 每个盒子的**类型**，这里规定每个盒子都装整数。
- `a` —— 这排盒子的**总名字**（你自己起的）。
- `[5]` —— 中括号里的数字 `5` 表示**一共造 5 个盒子**。
- `;` —— 结束分号。

读这行：**"造 5 个连在一起、装整数的盒子，整体叫 a。"**

⚠️ 中括号里的 `5` 叫"数组长度"，它必须是个**确定的整数**（常量），不能是变量随便变。比如 `int a[5];` 可以，但早期 C 标准里 `int n=5; int a[n];` 是不允许的（这点考试偶尔抠，知道"长度要写确定的数"就行）。

### 2. 怎么用其中一个盒子（引用 / 下标）⭐⚠️ 全场最大的坑

造好 5 个盒子后，怎么挑出某一个来用？用**下标**（中括号里写编号）：

```c
int a[5];        // 造了 5 个盒子
a[0] = 10;       // 往第 1 个盒子放 10
a[1] = 20;       // 往第 2 个盒子放 20
a[4] = 50;       // 往第 5 个（最后一个）盒子放 50
printf("%d\n", a[0]);   // 读第 1 个盒子，输出 10
```

⚠️⚠️⚠️ **下标从 0 开始！！！** 这是初学者栽得最惨的地方，给你掰开揉碎：

- `int a[5];` 造了 5 个盒子。
- 它们的门牌号（下标）是 **0、1、2、3、4**，不是 1、2、3、4、5！
- 所以第 1 个盒子是 `a[0]`，最后一个（第 5 个）是 `a[4]`。
- **根本没有 `a[5]` 这个盒子！** 写 `a[5]` 就越界了（下面专门讲越界有多可怕）。

记死这张对照表 ⭐：

| 你以为 | 实际是 |
|--------|--------|
| 第 1 个 | `a[0]` |
| 第 2 个 | `a[1]` |
| 第 3 个 | `a[2]` |
| 第 5 个（最后一个） | `a[4]` |
| `a[5]` | ❌ 不存在，越界！ |

口诀：**`int a[n]` 的合法下标是 `0` 到 `n-1`，一共 n 个。最后一个永远是 `a[n-1]`，不是 `a[n]`。**

### 3. 造盒子的同时就装好东西（初始化）⭐

每次 `a[0]=`、`a[1]=` 一个个填太麻烦。可以**造的时候就一次填好**，用大括号 `{ }`：

```c
int a[5] = {10, 20, 30, 40, 50};
```

- 大括号里用逗号隔开，**第 1 个值给 a[0]，第 2 个给 a[1]……依次对应**。
- 上面这行等价于：`a[0]=10; a[1]=20; a[2]=30; a[3]=40; a[4]=50;`

#### 几种常见花样（考点都在这）⭐⚠️

```c
int a[5] = {1, 2, 3};   // ① 只给前 3 个，后面 a[3]、a[4] 自动补 0
int b[5] = {0};         // ② 一个 0，但效果是"全部清零"——超常用！
int c[] = {1, 2, 3, 4}; // ③ 不写长度，编译器数大括号里有 4 个，自动定成 c[4]
```

逐条解释：
- **① 部分初始化补 0**：大括号里给的值不够数组长度时，**剩下的盒子自动填 0**。所以 `a` 其实是 `{1,2,3,0,0}`。⚠️ 这是高频考点：问你"`int a[5]={1,2,3};` 那 `a[4]` 是多少"，答案是 **0**。
- **② `{0}` 全部清零**：因为第一个填 0，剩下的也自动补 0，所以整排都是 0。想要一个"全 0 数组"就这么写。
- **③ 省略长度**：如果你写了大括号把值列全，可以不写中括号里的数字，编译器自己数。`int c[] = {1,2,3,4};` 等于 `int c[4]`。⚠️ 翁恺建议：**正式写代码时别省略长度，写上更清楚**。

⚠️ 一个**反面坑**：如果定义时**完全不初始化**，比如 `int a[5];` 然后直接读 `a[0]`，里面是**随机的垃圾值**（不一定是 0）！局部数组不会自动清零。想清零必须自己写 `={0}`。

## 二、数组越界：C 语言最危险的坑 ⚠️⚠️⚠️ 重点警告

这一节单独拎出来，因为它是 C 语言**最恐怖、最坑新手**的设计。

### 什么叫越界？

访问了**根本不存在的盒子**，就叫越界。比如：

```c
int a[5];        // 合法盒子是 a[0]~a[4]
a[5] = 100;      // ❌ 越界！a[5] 不存在
a[10] = 1;       // ❌ 越界更远
a[-1] = 1;       // ❌ 负下标也是越界
```

### C 不检查越界！这才是最可怕的 ⚠️

你以为越界了电脑会报错拦住你？**不会！** 这是 C 和别的语言最大的区别：

> **C 语言根本不检查下标越没越界。你写 `a[5]`，它就老老实实去操作"第 5 个盒子本该在的那块内存"——哪怕那块内存根本不属于这个数组。**

类比：你住 5 号楼 0~4 号房（你家的范围）。你跑去敲"5 号房"的门——但 5 号房是**邻居家**（别的变量，甚至系统的重要数据）。C 不拦你，你就直接闯进去乱改邻居家的东西。

### 越界会发生什么？

后果**完全不可预测**，可能是：
- 程序当场崩溃（运气好，你能马上发现）。
- 程序**没崩，但悄悄改坏了别的变量**，结果莫名其妙错——这种最难查，能让你debug到怀疑人生。
- 偶尔"看起来正常运行"，换台电脑/换次运行就出问题。

⚠️ 所以记住这条铁律：**写循环、写下标时，永远盯紧别超过 `n-1`。** 数组的所有 bug，一大半都是越界引起的。

口诀：**C 不查越界，越界是灾难。下标范围 0 到 n-1，越雷池半步全凭你自己把关。**

## 三、遍历数组：和 for 循环是绝配 ⭐

"遍历"就是**从头到尾把每个盒子都走一遍**。数组天生就该配 `for` 循环用，因为下标 0、1、2…… 正好可以让循环变量 `i` 来当。

```c
int a[5] = {10, 20, 30, 40, 50};
for (int i = 0; i < 5; i++) {
    printf("%d ", a[i]);     // i 依次是 0,1,2,3,4，正好取遍每个盒子
}
// 输出：10 20 30 40 50
```

逐字看这个 for 的头（M3 学过，这里复习）：
- `int i = 0` —— 从下标 **0** 开始（因为第一个盒子是 a[0]）。
- `i < 5` —— 条件是 `i < 5`，所以 i 最大走到 **4**，正好是最后一个盒子 a[4]。
- `i++` —— 每轮 i 加 1，下标往后挪一格。

### ⚠️ 循环条件为什么是 `< 5` 不是 `<= 5`？（高频坑）

```c
for (int i = 0; i <= 5; i++)   // ❌ 错！i 会取到 5，a[5] 越界
for (int i = 0; i < 5; i++)    // ✅ 对！i 最大是 4
```

记法：**长度是 5，就用 `i < 5`（小于号，不带等号）**。一旦写成 `<=`，就会多走一格碰到 `a[5]`，越界！这个错改错题里常出现。

### ⚠️ 数组不知道自己有多长

这是个反直觉的事实：**数组自己不记得自己有几个元素**。你没法问它"你多长？"。所以你**必须自己用一个变量记住长度**：

```c
int n = 5;                       // 自己记住长度
int a[5] = {10, 20, 30, 40, 50};
for (int i = 0; i < n; i++) {    // 用 n 控制，改长度只改一处
    printf("%d ", a[i]);
}
```

（注意：后面 M5/指针里会学到 `sizeof` 能算字节数，但那有适用条件，入门阶段就老老实实自己用变量 `n` 记长度最稳。）

## 四、一维数组的四大经典套路 ⭐🎯（翁恺 7.1.4 重点）

考试里数组题翻来覆去就这几个套路，全是"建一个变量 + for 遍历"。背熟模板，改改就能用。

### 套路1：求和 / 求平均值

```c
int a[5] = {10, 20, 30, 40, 50};
int sum = 0;                      // 累加器先清 0（⚠️ 必须先清 0）
for (int i = 0; i < 5; i++) {
    sum = sum + a[i];             // 把每个盒子的值累加进去
}
double avg = sum / 5.0;           // ⚠️ 除以 5.0 而不是 5，否则整数除法砍小数
printf("和=%d 平均=%.2f\n", sum, avg);   // 和=150 平均=30.00
```

⚠️ 两个坑：① `sum` 一定要先 `=0`，不然里面是垃圾值；② 求平均要 `sum / 5.0`（带小数），写 `sum / 5` 会因整数除法把小数砍掉（M1 学过的整数除法陷阱）。

### 套路2：求最大值 / 最小值（带下标）🎯

```c
int a[5] = {30, 70, 20, 90, 50};
int max = a[0];      // ⚠️ 先假设第 1 个最大
int index = 0;       // 记住最大值在哪个位置
for (int i = 1; i < 5; i++) {   // ⚠️ 从 1 开始，因为 0 已经当擂主了
    if (a[i] > max) {
        max = a[i];
        index = i;
    }
}
printf("最大值=%d，在下标 %d\n", max, index);   // 最大值=90，在下标 3
```

套路精髓（翁恺例7-4）：**"打擂台"**。先让第一个元素当擂主（`max=a[0]`），后面每个上来比一比，谁更大谁当新擂主。求最小值就把 `>` 改成 `<`。

### 套路3：顺序查找（找某个数在不在）🎯

```c
int a[5] = {10, 20, 30, 40, 50};
int x = 30;          // 要找的数
int flag = 0;        // 标志：0=没找到，1=找到了
for (int i = 0; i < 5; i++) {
    if (a[i] == x) {
        printf("找到了，在下标 %d\n", i);
        flag = 1;
        break;       // ⚠️ 找到就跳出，不用继续找了
    }
}
if (flag == 0) {
    printf("Not Found\n");
}
```

套路精髓（翁恺例7-3）：**设一个"标志变量" `flag`**。找到就置 1 并 `break`；循环结束后看 flag 还是不是 0，是 0 就说明没找到。

### 套路4：数组逆序（首尾对调）🎯

```c
int a[5] = {1, 2, 3, 4, 5};
int n = 5;
for (int i = 0; i < n / 2; i++) {   // ⚠️ 只走一半！
    int t = a[i];                   // 三句话交换 a[i] 和 a[n-1-i]
    a[i] = a[n-1-i];
    a[n-1-i] = t;
}
// 现在 a 变成 {5, 4, 3, 2, 1}
```

套路精髓：**第一个和最后一个换，第二个和倒数第二个换……** 用一个临时变量 `t` 中转（M1 讲过的交换三步：存、搬、放）。⚠️ 循环只走 `n/2` 趟，走全程会换两次又换回来。

## 四点五、⭐⭐ 考研要求的辨析：真题的"第 i 个元素"位序从 1 开始（必看）

> 你学数组时，老师反复敲黑板：**下标从 0 开始**，`a[5]` 造 5 个盒子，合法下标 `0~4`，没有 `a[5]`。这是 C 的铁律，没错。
>
> 但你将来刷**王道、408 线性表大题**时，题目和代码会冒出另一套说法——"**第 i 个元素**""**在第 i 个位置插入**"，这里的 i 是**位序（从 1 开始数）**，不是下标。两套"编号"混在一起，是新手读真题最容易栽的坑。这一节专门掰开。

### ① 位序 vs 下标：差一个 1 ⭐⚠️

- **下标（index）**：C 语言数组天生的编号，**从 0 开始**。第一个元素是 `a[0]`。
- **位序（position）**：数据结构里"第几个元素"的人话说法，**从 1 开始**。第 1 个元素、第 2 个元素……

对照表（一张长度为 5 的顺序表）：

| 人话（位序） | 真题里的 i | C 代码里的下标 |
|------------|-----------|--------------|
| 第 1 个元素 | i = 1 | `data[0]` |
| 第 2 个元素 | i = 2 | `data[1]` |
| 第 i 个元素 | i | **`data[i-1]`** |
| 第 length 个（最后一个） | i = length | `data[length-1]` |

> ⭐ 核心换算公式，背死：**第 i 个元素 ⟺ 下标 `i-1`**。真题代码里看到 `L.data[i-1]`，脑子里立刻翻译成"第 i 个元素"。

### ② 真题里的遍历长这样：`for (i = 1; i <= length; i++)`

你学数组遍历是从 0 数：

```c
for (int i = 0; i < n; i++)      // 你学的：下标 0 到 n-1
    printf("%d ", a[i]);
```

但 408 线性表大题，因为按"位序"描述，遍历常写成从 1 数：

```c
for (i = 1; i <= L.length; i++)  // 真题风格：位序 1 到 length
    printf("%d ", L.data[i-1]);  // ⚠️ 注意取元素是 data[i-1]，差那个 1
```

⚠️ 两种都对，区别只是"用下标数还是用位序数"。**看到 `i` 从 1 起、条件是 `<= length`，就知道这是位序遍历，取元素要 `data[i-1]`。**

### ③ 为什么真题非要用位序？

因为数据结构讲的是"**第几个数据元素**"这种贴近人类直觉的概念（线性表的定义就是"n 个元素的有限序列，第 1 个、第 2 个……"）。位序从 1 开始更符合人话，所以王道、教材统一用位序描述算法。等真正写成 C 代码落地时，再 `-1` 转成下标去访问数组。

> 📌 一句话记住：**真题说"第 i 个"是位序（从 1 起），落到 C 数组代码里要 `data[i-1]`（下标从 0 起）。两者差一个 1，这是读真题、写大题不丢分的关键。** 顺带：插入"在第 i 个位置"时，合法的 i 范围是 `1 ~ length+1`（可以插到最后一个的后面），这点 M2 的边界判断会再用到。

## 五、二维数组：表格 / 棋盘 ⭐⚠️ 重点

一维数组是"**一排**盒子"。那如果我要存一张**表格**（有行有列），比如课程表、棋盘、矩阵呢？用**二维数组**。

类比：**一栋楼有很多层，每层有很多间房**，或者**Excel 表格有行有列**。要找一个房间，得报"第几行 + 第几列"。

### 1. 怎么造（定义）

```c
int m[2][3];
```

逐字拆解：
- `int` —— 每个格子装整数。
- `m` —— 整张表的名字。
- `[2]` —— **2 行**。
- `[3]` —— 每行 **3 列**。
- 合起来：一张 2 行 3 列的表，共 `2 × 3 = 6` 个格子。

### 2. 怎么用某个格子（引用）⚠️

用**两个下标**：`m[行][列]`，**行列下标都从 0 开始**：

```c
int m[2][3];
m[0][0] = 1;     // 第 1 行第 1 列
m[0][2] = 3;     // 第 1 行第 3 列
m[1][2] = 6;     // 第 2 行第 3 列（最后一个格子）
```

⚠️ 老规矩：`m[2][3]` 的合法行下标是 **0~1**，合法列下标是 **0~2**。`m[2][3]` 这个格子**不存在**（越界）。

### 3. 初始化（两种写法）⭐

```c
// 写法① 分行写（推荐，一眼看清行列结构）
int m[2][3] = {
    {1, 2, 3},      // 第 0 行
    {4, 5, 6}       // 第 1 行
};

// 写法② 拉成一条写（按"行优先"顺序自动填）
int m[2][3] = {1, 2, 3, 4, 5, 6};   // 效果和上面一样
```

- **写法①分行**：每对小括号 `{}` 装一行，结构清楚，翁恺推荐这种。
- **写法②一条龙**：编译器按"先填满第 0 行，再填第 1 行"的顺序塞进去。
- 同样支持**部分初始化补 0**：`int m[2][3] = {{1,2},{4}};` 没写到的格子（如 `m[0][2]`、`m[1][1]`、`m[1][2]`）全自动补 0。
- **省略行长度**：如果把值列全了，可以省略第一个中括号的数（行数），但**列数 `[3]` 绝不能省**：`int m[][3] = {1,2,3,4,5,6};` 编译器自己数出是 2 行。⚠️ 记住：**二维数组省略只能省行、不能省列**。

### 4. 行优先存储 ⭐⚠️（考点）

虽然我们脑子里把它想成"表格"，但内存其实是**一条直线**。二维数组在内存里是**按行一行一行地连续存放**的，叫"**行优先**"（行序优先）：

```
内存里的真实顺序： m[0][0] m[0][1] m[0][2] m[1][0] m[1][1] m[1][2]
                  └────── 第 0 行 ──────┘ └────── 第 1 行 ──────┘
```

⚠️ 考点：先存完第 0 行的所有元素，再存第 1 行。这也解释了为什么省略时**只能省行不能省列**——编译器得先知道"一行有几个"，才能切分。

### 5. 遍历二维数组：双重循环 ⭐

一维用一层 for，二维就用**两层 for 嵌套**：外层管行 `i`，内层管列 `j`：

```c
int m[2][3] = {{1, 2, 3}, {4, 5, 6}};
for (int i = 0; i < 2; i++) {          // 外层：第 i 行（0~1）
    for (int j = 0; j < 3; j++) {      // 内层：第 j 列（0~2）
        printf("%d ", m[i][j]);
    }
    printf("\n");                      // ⚠️ 每打印完一行换个行
}
// 输出：
// 1 2 3
// 4 5 6
```

记法：**外层 i 走行，内层 j 走列；`m[i][j]` 取格子；每跑完一整行（内层结束）打个 `\n` 换行。** ⚠️ 建议让外层管行、内层管列，正好顺着"行优先"的内存顺序走，效率最高（翁恺 7.2.4 提到）。

### 6. 方阵常用术语（选择题爱考）📚⭐

如果是 N×N 的方阵（行列一样多），下标 i、j 有几个固定关系，常考：

| 部位 | 下标关系 |
|------|---------|
| 主对角线（左上→右下） | `i == j` |
| 副对角线（右上→左下） | `i + j == N - 1` |
| 上三角 | `i <= j` |
| 下三角 | `i >= j` |

🎯 真题类型：**方阵转置**（行列互换）= 交换 `a[i][j]` 和 `a[j][i]`。技巧：只遍历上三角（`i < j`）去交换，避免换两次又换回来（翁恺例7-9）。

## 六、字符串：C 语言里"一串文字"的真相 ⭐⚠️ 全模块最核心

前面 M1 学过 `char` 只能装**一个**字符（`char c = 'A';`）。那"Hello"这种**一串文字**怎么存？

答案出人意料：**C 语言根本没有专门的"字符串"类型！** 字符串就是**一个 `char` 数组**——把一串字符一个一个塞进 char 盒子里。

### 1. 字符串 = 字符数组 + 结尾的 `'\0'` ⭐⚠️⚠️ 最重要的考点

但光是"一串 char"还不够。问题来了：电脑怎么知道这串字符**到哪儿结束**？数组又不记得自己多长（前面讲过）。

C 的解决办法极其巧妙：**在这串字符的最后，偷偷塞一个特殊字符 `'\0'`（叫"空字符"），当作"句号 / 终止标志"。** 电脑从头读字符，一读到 `'\0'` 就知道："哦，字符串到这儿结束了。"

类比：**一串字符 + 一个结尾的句号。** 你念一句话，念到句号就停。`'\0'` 就是 C 字符串的"句号"。

```c
char s[] = "Hi";
```

这行看起来只有 'H'、'i' 两个字符，但**实际在内存里存了 3 个**：

```
下标:   0    1    2
内容:  'H'  'i'  '\0'      ← 编译器自动在末尾加了 '\0'！
```

⚠️⚠️⚠️ **超高频考点**：问你 `char s[] = "Hi";` 占几个字节？答案是 **3，不是 2**！因为末尾那个 `'\0'` 也要占一个字节。

关于 `'\0'`，记死这几条：
- `'\0'` 是**一个字符**（反斜杠 + 数字0），它的 ASCII 码值就是 **0**。
- 它**看不见**（打印不出东西），但它真实占一个字节。
- 它叫"字符串结束标志"/"空字符"，⚠️ **不要和数字字符 `'0'`（ASCII 48）搞混**，更不要和空格 `' '` 搞混。三个是完全不同的东西。

### 2. 定义字符串的两种写法 ⭐

```c
// 写法① 用双引号字符串（推荐，自动加 '\0'）
char s1[] = "Happy";        // 占 6 字节：'H''a''p''p''y''\0'
char s2[20] = "Happy";      // 给 20 个位置，只用了前 6 个，够大就行

// 写法② 一个一个字符列出来（容易翻车）
char t[5] = {'H','a','p','p','y'};          // ❌ 这是字符数组，不是字符串！没有 '\0'
char w[6] = {'H','a','p','p','y','\0'};     // ✅ 手动加了 '\0'，才是合法字符串
```

⚠️ 关键区别（翁恺 7.3.2 重点考点）：
- 用**双引号 `"..."`** 定义，编译器**自动帮你加 `'\0'`**，省心。
- 用**大括号一个个列字符**，如果你**不手动加 `'\0'`，它就不是字符串**，只是一堆字符。后面用 `%s`、`strlen` 处理它会出乱子（因为找不到结束标志，会一直往后冲到越界）。

### 3. 数组要留够空间！⚠️（高频坑）

定义字符数组时，**长度至少要 = 字符个数 + 1**（多出来的 1 个留给 `'\0'`）：

```c
char s[5] = "Hello";   // ❌ 危险！"Hello" 是 5 个字母 + '\0' 共 6 个，盒子只有 5 个，装不下 '\0'
char s[6] = "Hello";   // ✅ 6 个刚好：5 字母 + 1 个 '\0'
char s[20] = "Hello";  // ✅ 开大点更安全
```

口诀：**定字符数组，长度记得 +1，给句号 `'\0'` 留个位置。**

### 4. 字符串的输入输出：`%s` ⭐⚠️

打印和读取整个字符串，用占位符 **`%s`**（s = string）：

```c
char name[20];
printf("请输入名字: ");
scanf("%s", name);          // ⚠️ 注意：name 前面【不加 &】！
printf("你好, %s\n", name); // %s 从头打印到 '\0' 为止
```

#### ⚠️⚠️ 坑1：scanf 读字符串，变量前【不加 `&`】！

M1 学过 scanf 读 int/double 要加 `&`（如 `scanf("%d", &n)`）。但**读字符串数组时，名字前面不加 `&`**：

```c
scanf("%s", name);    // ✅ 对，name 前面没有 &
scanf("%s", &name);   // ❌ 错，多此一举
```

为什么？现在记住一句话就行：
> **数组名本身就代表这排盒子的地址（首地址），它"天生就是个地址"，所以不用再加 `&` 去取地址了。**

（深层原因 M7 指针会讲。对比记忆：普通变量 `&n` 才是地址；数组名 `name` 本身就是地址。）

#### ⚠️⚠️ 坑2：`scanf("%s")` 遇空格就停！

`scanf` 用 `%s` 读字符串，**读到空格、Tab、回车就认为结束了**，后面的读不进来：

```c
char s[50];
scanf("%s", s);        // 你输入：Hello World
printf("%s\n", s);     // 只输出 Hello ！World 被丢掉了
```

⚠️ 所以 `scanf("%s")` **读不了带空格的句子**。想读一整行（含空格），要用 **`fgets`**：

```c
char s[50];
fgets(s, 50, stdin);   // 能读一整行含空格，最多读 49 个字符
```

入门阶段记住结论：**单个词用 `scanf("%s")`，整句话（带空格）用 `fgets`。**

### 5. 遍历字符串：靠 `'\0'` 停 ⭐

处理字符串不像普通数组那样靠"长度 n"，而是**靠检测 `'\0'`**：一边走一边看，遇到 `'\0'` 就停。

```c
char s[] = "Happy";
for (int i = 0; s[i] != '\0'; i++) {    // ⚠️ 条件：当前字符不是 '\0' 就继续
    printf("%c ", s[i]);                // 一个一个字符打印
}
// 输出：H a p p y     （走到 '\0' 自动停，不会打印 '\0'）
```

记法：**字符串遍历的循环条件是 `s[i] != '\0'`**——"还没碰到句号就接着走"。这是字符串题的万能模板。

## 七、常用字符串函数（必须 `#include <string.h>`）⭐⚠️ 重点

自己写循环处理字符串太累，C 在 `<string.h>` 这个库里准备了几个现成的工具函数。⚠️ **用之前必须在文件开头写 `#include <string.h>`**（就像用 printf 要 include stdio.h 一样）。

```c
#include <stdio.h>
#include <string.h>      // ⚠️ 用字符串函数必须加这行
```

### 1. `strlen(s)` —— 求字符串长度 ⭐

```c
char s[] = "Hello";
printf("%d\n", strlen(s));   // 输出 5
```

⚠️ 关键考点：**`strlen` 数的是 `'\0'` 之前的有效字符个数，不包括 `'\0'`**。所以 "Hello" 的 strlen 是 5，不是 6。

### 2. `strcpy(目标, 源)` —— 复制字符串 ⭐

```c
char a[20];
strcpy(a, "Hello");    // 把 "Hello" 拷进 a，a 现在是 "Hello"
char b[20];
strcpy(b, a);          // 把 a 的内容拷进 b
```

记法：`strcpy(d, s)` = copy s **到** d，**从右往左搬**（和赋值 `=` 方向一致）。⚠️ 目标数组 `a` 必须**足够大**装得下，否则越界。

### 3. `strcmp(a, b)` —— 比较两个字符串 ⭐⚠️ 改错必考

```c
char a[] = "abc";
char b[] = "abc";
if (strcmp(a, b) == 0) {        // ⚠️ 相等返回 0！
    printf("两个字符串相同\n");
}
```

⚠️⚠️ 两个反直觉的点，记死：
- **相等时返回 `0`**（不是 1，不是 true）。所以判断相等要写 `strcmp(a,b) == 0`。
- 返回值含义：`a` 比 `b`"大"返回正数，"小"返回负数，相等返回 0（按字典序 / ASCII 逐字符比）。

### 4. `strcat(目标, 源)` —— 拼接字符串

```c
char a[20] = "Hello";
strcat(a, " World");    // 把 " World" 接到 a 后面，a 变成 "Hello World"
```

记法：cat = concatenate（连接）。把第二个接到第一个**屁股后面**。⚠️ 目标数组要留够空间装下拼接后的总长度。

### ⚠️⚠️⚠️ 字符串三大铁律（改错题重灾区）

#### 铁律1：字符串不能用 `==` 比较，要用 `strcmp`！

```c
char a[] = "abc", b[] = "abc";
if (a == b)              // ❌ 错！这比的是两个数组的"地址"，不是内容，永远不相等
if (strcmp(a, b) == 0)   // ✅ 对！比内容要用 strcmp
```

为什么？`a` 和 `b` 是两个数组名，代表两块**不同的内存地址**。`a == b` 比的是地址，俩地址当然不一样，所以永远为假。比"内容一不一样"必须用 `strcmp`。

#### 铁律2：字符串不能用 `=` 赋值，要用 `strcpy`！

```c
char s[20];
s = "Hello";          // ❌ 错！数组名不能整体赋值，编译直接报错
strcpy(s, "Hello");   // ✅ 对！用 strcpy 复制
```

⚠️ 只有**定义的那一刻**能用 `=` 写 `char s[] = "Hello";`（那叫初始化）。一旦定义完了，后面再想换内容，**只能用 strcpy，不能用 `=`**。

#### 铁律3：数组名不能整体赋值给另一个数组

```c
int a[5] = {1,2,3,4,5};
int b[5];
b = a;                          // ❌ 错！数组不能整体赋值
for (int i=0;i<5;i++) b[i]=a[i];// ✅ 对！只能一个一个元素拷
```

这条对**所有数组**（不只字符串）都成立：**数组名不能直接 `=` 赋值，要复制只能逐个元素拷（或字符串用 strcpy）。**

## 八、字符运算：字符其实是数字 ⭐🎯（翁恺 7.3.4 重点）

这一节是字符串编程题的"内功"，理解了能秒杀一堆真题。

### 核心事实：`char` 本质是个小整数

每个字符在电脑里都对应一个 **ASCII 码**（一个整数）。比如 `'A'` 其实就是数字 **65**，`'a'` 是 **97**，`'0'` 是 **48**。所以**字符可以直接参与加减运算**：

```c
char c = 'A';
printf("%d\n", c);       // 输出 65（用 %d 当数字看）
printf("%c\n", c + 1);   // 输出 B（'A' 加 1 变 'B'）
```

### 几个超实用的字符技巧 🎯

#### 技巧1：判断是不是数字字符 / 字母

```c
if (c >= '0' && c <= '9')   // c 是数字字符 '0'~'9'
if (c >= 'a' && c <= 'z')   // c 是小写字母
if (c >= 'A' && c <= 'Z')   // c 是大写字母
```

因为同类字符的 ASCII 码是**连续**的，所以用范围 `>=` 和 `<=` 就能判断。

#### 技巧2：数字字符转成真正的数字 —— 减 `'0'` 🎯⚠️ 超高频

```c
char c = '7';
int num = c - '0';     // num = 7（不是字符 '7'，是真正的整数 7）
```

为什么减 `'0'`？因为 `'7'` 的 ASCII 是 55，`'0'` 的 ASCII 是 48，`55 - 48 = 7`。**任何数字字符减掉 `'0'`，就得到它代表的真实数值**。这是把"字符 7"变成"数字 7"的标准做法。

🎯 经典真题（翁恺例7-13）：把字符串里的数字字符拼成一个整数，核心一行：
```c
number = number * 10 + (str[i] - '0');   // 逐位拼数字，比如把 "123" 变成整数 123
```

#### 技巧3：大小写转换

```c
char c = 'a';
char up = c - 'a' + 'A';     // 小写转大写：'a'→'A'
char c2 = 'B';
char low = c2 - 'A' + 'a';   // 大写转小写：'B'→'b'
// 也可以利用 'a'-'A'=32 这个固定差值：大写+32变小写，小写-32变大写
```

🎯 真题套路（翁恺例7-12/7-14）：遍历字符串统计数字个数、提取数字、十六进制转十进制，全靠"字符就是数字 + 减基准字符"这套思路。

## 九、新手最常犯的错，照着排查 ⚠️

1. **下标从 1 开始数** —— 永远记住第一个是 `a[0]`，`int a[5]` 最后一个是 `a[4]`，没有 `a[5]`。
2. **循环写成 `i <= n`** —— 多走一格越界。长度 n 就用 `i < n`。
3. **忘了 `'\0'` 占空间** —— `char s[5]="Hello"` 装不下（要 6）。定字符数组长度记得 +1。
4. **用大括号列字符却忘加 `'\0'`** —— `{'H','i'}` 不是字符串，`%s`/strlen 会出乱子。
5. **scanf 读字符串加了 `&`** —— `scanf("%s", &name)` 多余，数组名本身是地址，不加 `&`。
6. **`scanf("%s")` 想读带空格的句子** —— 读不全，遇空格就停，要用 `fgets`。
7. **用 `==` 比较字符串** —— 比的是地址不是内容，必须 `strcmp(a,b)==0`。
8. **用 `=` 给字符串赋值** —— 定义后不能 `s="..."`，要 `strcpy`。
9. **用了字符串函数忘 `#include <string.h>`** —— strlen/strcpy/strcmp/strcat 都需要它。
10. **数组没初始化就读** —— 局部数组里是垃圾值，要清零得自己写 `={0}`。
11. **求平均写成整数除法** —— `sum/n` 砍小数，要 `sum/(n*1.0)` 或 `1.0*sum/n`。

## 十、strlen vs sizeof：最爱考的一对 ⭐⚠️🎯

这俩特别容易混，单独拎出来对比（涉及 M5 的 sizeof）：

```c
#include <string.h>
char s[20] = "Hello";
printf("%d\n", strlen(s));   // 5  ← 有效字符个数，不含 '\0'
printf("%d\n", sizeof(s));   // 20 ← 整个数组占的字节数（盒子总数）
```

记死区别 ⭐：

| | `strlen(s)` | `sizeof(s)` |
|--|------------|-------------|
| 是什么 | 库函数（要 `#include <string.h>`） | 运算符（不用 include） |
| 数什么 | `'\0'` 之前的**有效字符数** | 数组**申请的总字节数** |
| 含 `'\0'` 吗 | **不含** | 申请了多大就是多大 |
| 上例结果 | 5 | 20 |

🎯 真题陷阱：`char s[] = "Hello";` 时，`strlen(s)` = 5，`sizeof(s)` = **6**（这时数组大小正好等于字符数+1，含 `'\0'`）。看清是 `s[20]` 还是 `s[]`，结果不一样！

## 十二、本节考点速记 ⭐

1. 数组 = 一排同类型盒子，`int a[5]` 造 5 个；**下标从 0 开始**，合法下标 `0~n-1`，最后一个是 `a[n-1]`，没有 `a[n]`。
2. 初始化 `{1,2,3}` 不够的**自动补 0**；`{0}` 全清零；`int a[]={...}` 省略长度让编译器自己数。
3. **C 不检查越界，越界是灾难**——可能崩溃、可能悄悄改坏别的数据。下标永远盯紧别超 `n-1`。
4. 遍历数组配 for，长度 n 用 **`i < n`**（不是 `<=`）；数组不知道自己多长，要自己用变量记。
5. 一维四套路：求和/平均（累加器先清0、平均要除小数）、求最值（打擂台）、查找（flag 标志 + break）、逆序（首尾对调走半程）。
6. 二维数组 `m[2][3]` = 2 行 3 列；用 `m[i][j]`；**行优先存储**；遍历用**双重循环**（外行内列）；省略只能省行不能省列。
7. 方阵：主对角线 `i==j`，副对角线 `i+j==N-1`，上三角 `i<=j`，下三角 `i>=j`。
8. **字符串 = 字符数组 + 结尾 `'\0'`**；`"Hi"` 占 **3** 字节（含 `'\0'`）；`'\0'` 的 ASCII 是 0，是结束标志。
9. 定字符数组**长度 = 字符数 + 1**（给 `'\0'` 留位）；双引号定义自动加 `'\0'`，大括号列字符要自己加。
10. `%s` 输入输出字符串；**scanf 读字符串不加 `&`**（数组名本身是地址）；`scanf("%s")` 遇空格停，整句用 `fgets`。
11. 字符串遍历靠 `s[i] != '\0'` 停。
12. 四个字符串函数要 **`#include <string.h>`**：`strlen`(不含\0)、`strcpy(d,s)`、`strcmp`(相等返回**0**)、`strcat`。
13. 三大铁律：字符串比较用 `strcmp` 不用 `==`；赋值用 `strcpy` 不用 `=`；**数组名不能整体赋值**，要逐个拷。
14. 字符就是小整数（ASCII）；**数字字符减 `'0'` 得真实数值**；`number=number*10+(s[i]-'0')` 拼整数。
15. **strlen vs sizeof**：strlen 数有效字符（不含\0），sizeof 数总字节数；`char s[]="Hello"` 时 strlen=5、sizeof=6。
