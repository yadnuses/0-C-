## 〇、开始之前：为什么这一章是"陷阱"？

前面 M1 你学了 `int`、`double`、`char` 三种盒子。当时我们说"`int` 装整数、`double` 装小数"，**没细讲它们到底有多大、装不下会怎样**。

这一章就是来填这些坑的。出题老师特别喜欢在这里设套，因为很多东西**反直觉**：
- 你以为 `127 + 1 = 128`？在 `char` 里它可能变成 `-128`。
- 你以为 `(int)3.99` 会四舍五入成 4？不，它直接砍成 3。
- 你以为字符 `'A'` 和数字没关系？其实 `'A'` 就是数字 65。

这些不是 bug，是 C 语言**故意设计**成这样的。理解了背后的道理，这些"陷阱"就变成你的送分题。慢慢读，别跳。

## 一、基本数据类型与大小：盒子有多大？ ⭐⚠️ 必考

### 先建立一个核心观念：盒子是有容量上限的

M1 我们把变量比作"盒子"。现在要补一个关键事实：**每种盒子的大小是固定的，能装的数有上限。**

类比：盒子就像不同容量的杯子。
- `char` 是个小酒盅，只能装一点点。
- `int` 是个普通水杯。
- `double` 是个大水桶。

你往小酒盅里倒一桶水，会**溢出来**——这就是后面要讲的"溢出"陷阱的根源。

### 盒子大小一览表 ⭐（这张表要背）

| 类型 | 含义 | 占几个字节 | 能装的范围（典型） | printf 占位符 |
|------|------|----------|------------------|------|
| `char` | 字符 / 小整数 | **1 字节** | -128 ~ 127 | `%c` 或 `%d` |
| `short` | 短整数 | 2 字节 | -32768 ~ 32767 | `%hd` |
| `int` | 整数 | **4 字节** | 约 ±21 亿 | `%d` |
| `long` | 长整数 | 8 字节(Mac/Linux 64位) | 很大 | `%ld` |
| `float` | 单精度小数 | **4 字节** | 约 7 位有效数字 | `%f` |
| `double` | 双精度小数 | **8 字节** | 约 15 位有效数字 | `%lf` / `%f` |

⭐ **必背的四个数**：`char` 1 字节、`int` 4 字节、`float` 4 字节、`double` 8 字节。`short`、`long` 了解即可。

### "字节"是什么？1 字节能装多少？

- **1 字节（byte）= 8 个二进制位（bit）**。
- 1 个二进制位只能是 0 或 1，像一个开关。8 个开关排一起，能组合出 `2^8 = 256` 种不同状态。
- 所以 1 字节最多能区分 **256 个不同的数**。这就是为什么 `char` 的范围（-128 到 127，或 0 到 255）刚好是 256 个数——记住这个 256，后面溢出全靠它。

类比：1 个开关 2 种状态（开/关），2 个开关 4 种（开开、开关、关开、关关）……8 个开关 256 种。位数越多，能表示的数越多。

### `sizeof`：亲自量一量盒子有多大 ⭐⚠️ 本身就是考点

光记表格不踏实，C 给了你一把"尺子"叫 `sizeof`，能量出任何类型或变量占几个字节：

```c
#include <stdio.h>

int main(void)
{
    printf("char   占 %d 字节\n", (int)sizeof(char));
    printf("int    占 %d 字节\n", (int)sizeof(int));
    printf("float  占 %d 字节\n", (int)sizeof(float));
    printf("double 占 %d 字节\n", (int)sizeof(double));
    return 0;
}
```

输出（在 Mac/Linux 64位上）：
```
char   占 1 字节
int    占 4 字节
float  占 4 字节
double 占 8 字节
```

拆解：
- `sizeof` 是个**运算符**（不是函数，虽然长得像）。`sizeof(int)` 读作"int 占几个字节"。
- 它既能量类型 `sizeof(int)`，也能量变量 `sizeof(a)`。
- ⚠️ 为什么前面加 `(int)`？因为 `sizeof` 算出来的结果类型比较特殊（叫 `size_t`），用 `%d` 打印可能警告，强转成 `int` 最稳妥。这个细节了解即可。

⚠️ **考点提醒**：`sizeof` 在编译时就算好了，不是运行时才算。考试常问"`sizeof(int)` 的值是多少"——答案看平台，但考研默认 `int` 是 **4 字节**。

## 二、整型常量的三种写法：十进制、八进制、十六进制 ⭐⚠️ 选择题常考

平时写数字 `10`、`255`，这是**十进制**（逢十进一），是默认的。但 C 里同一个数能用三种"进制"写出来，出题老师就爱在这设套。

### 三种写法对照

| 进制 | 长什么样 | 规则 | 例子 | 等于十进制 |
|------|---------|------|------|----------|
| 十进制 | `10` | 0-9，**首位不能是 0** | `10` | 10 |
| 八进制 | `010` | 0-7，**必须以 0 开头** | `010` | **8** |
| 十六进制 | `0x10` | 0-9 和 a-f，**以 0x 开头** | `0x10` | **16** |

⚠️⚠️ **最大的坑就在八进制**：

```c
int a = 10;     // 十进制，就是 10
int b = 010;    // ⚠️ 以 0 开头！这是八进制，等于十进制的 8，不是 10！
int c = 0x10;   // 十六进制，等于 16

printf("%d %d %d\n", a, b, c);   // 输出：10 8 16
```

为什么 `010 = 8`？八进制逢八进一，`010` 表示"1 个八 + 0 个一" = 8。
为什么 `0x10 = 16`？十六进制逢十六进一，`0x10` 表示"1 个十六 + 0 个一" = 16。

🎯 **真题套路**：题目给你 `int x = 012;` 问 x 是多少，很多人脱口而出"12"，**错！** 它以 0 开头是八进制，`012 = 1×8 + 2 = 10`。看到数字**前面带 0**，立刻警觉是八进制。

⚠️ 注意：这只影响**整数常量**的写法，不影响小数。`0.5` 那个 0 是小数点前的 0，跟八进制无关。

### 整数常量的后缀（了解）

数字后面可以加字母后缀，告诉编译器它是什么类型：
- 加 `l` 或 `L` → `long` 型，如 `100L`。
- 加 `u` 或 `U` → `unsigned`（无符号）型，如 `100U`。
- 不加后缀 → 编译器根据数值大小自己判断。

了解即可，知道 `100L` 是个 long、`100U` 是无符号就行。

## 三、补码：整数在内存里到底怎么存的 ⭐⚠️ 高频

> 这一节稍微烧脑，但它是理解"溢出回绕"的钥匙。看不懂没关系，先记结论，回头再悟。

### 为什么要懂这个？

因为有个高频考点：**`char` 装 127，再加 1，结果变成 -128**。想搞懂这个诡异现象，得知道负数在内存里是怎么存的——答案是**补码**。

### 计算机只认 0 和 1，那负号怎么存？

计算机内存里只有 0 和 1，没有"负号"这个东西。那 `-5` 怎么存？聪明人发明了**补码**这套规则：

- **正数**：补码 = 它本身的二进制。比如 `5`（8位）就是 `00000101`。
- **负数**：分三步走——
  1. 先写出**绝对值**的二进制（比如 -5，先写 5 = `00000101`）。
  2. **取反**（0 变 1，1 变 0）：`11111010`。
  3. **加 1**：`11111011`。这就是 -5 的补码。

记不住三步没关系，**结论才是考点**。

### 关键结论：8 位能表示 -128 ~ 127 ⭐⚠️

一个 `char` 是 8 位（1 字节），按补码规则，它能表示的范围是：

> **-128 ~ 127**（一共 256 个数）

⚠️ 考试爱问：**为什么是 -128 到 127，而不是 -127 到 127？**
- 8 位有 256 个状态，要分给正数、负数和 0。
- **0 占了"正数那边"一个位置**，所以正数最多到 127（不是 128）。
- 负数那边就能多一个，到 -128。
- 记法：**负数比正数多一个**（因为 0 算在正数阵营）。

### 最高位是"符号位" ⭐

补码里，**最左边那一位（最高位）专门表示正负**：
- 最高位是 `0` → 正数。
- 最高位是 `1` → 负数。

这就是溢出回绕的秘密：当你不停加，加到最高位变成 1，这个数"看起来"就突然变成负数了。下一节细讲。

## 四、有符号 vs 无符号 + 溢出回绕 ⭐⚠️ 重灾区中的重灾区

### signed 和 unsigned 是什么意思？

- `signed`（有符号）：能装负数，比如 `char` 范围 -128~127。**默认就是 signed**，平时不用写。
- `unsigned`（无符号）：**只装非负数**（0 和正数），不要负数那一半，省下来的空间全给正数。

对比 `char`：

| 类型 | 范围 | 一共几个数 |
|------|------|----------|
| `char`（= signed char） | **-128 ~ 127** | 256 |
| `unsigned char` | **0 ~ 255** | 256 |

看出门道没？**总数都是 256**，区别只是"要不要负数"。`unsigned` 把负数那 128 个位置挪去装更大的正数了。

`int` 同理：
- `int`：约 -21 亿 ~ +21 亿。
- `unsigned int`：0 ~ 约 42 亿。

### ⚠️⚠️ 把负数赋给 unsigned，会变成超大正数 ⭐

```c
unsigned int u = -1;
printf("%u\n", u);     // 输出：4294967295（不是 -1！）
```

为什么？`-1` 的补码是"全 1"（`11111111...`）。`unsigned` 不认符号位，把这一堆 1 当成**最大的正数**来读，于是 `-1` 就变成了 4294967295（unsigned int 的最大值）。

- `%u` 是打印无符号整数的占位符。
- 🎯 改错题、选择题最爱考"`unsigned int u = -1;` 输出多少"，答案是该类型的**最大值**。

### ⚠️⚠️⚠️ 溢出回绕：127 + 1 = -128（最高频考点）

来了，整章最爱考的点。用**里程表归零**类比最好理解：

> 老式汽车里程表只有 5 位数，跑到 99999 公里，再跑 1 公里，它不会变成 100000，而是**归零回到 00000**——因为它装不下第 6 位。

`char` 也一样，它最多装到 127，再加 1 就"绕回去"了：

```c
char c = 127;
c = c + 1;
printf("%d\n", c);     // 输出：-128（不是 128！）
```

发生了什么？用补码解释：
- `127` 的补码：`01111111`（最高位 0，是正数）。
- 加 1：`10000000`。
- 这个结果最高位变成了 `1`，按补码规则它代表 **-128**。
- 所以"加过头"的瞬间，正数最大值 127 直接**跳到**负数最小值 -128。

类比图：把 char 的取值想象成一个**首尾相接的圆环**：
```
... 125 → 126 → 127 →(加1) -128 → -127 → ... → -1 → 0 → 1 ...
                         ↑ 从最大跳到最小，绕一圈
```

🎯 **真题变体**：
- `char c = 128;` 会怎样？128 超出 127，绕回去变 -128。
- `unsigned char u = 255; u = u + 1;` → 变成 **0**（unsigned char 到 255 是头，再加 1 归零）。
- `unsigned char u = 0; u = u - 1;` → 变成 **255**（往回绕）。

⚠️ 记死这条：**整型装不下就回绕（溢出），不会报错，而是绕到另一头。** 这跟里程表一模一样。

## 五、小数怎么存 + 科学计数法 + 精度问题 ⭐⚠️

### float 和 double 的区别

两个都装小数，区别是"精度"（能记住几位有效数字）：
- `float`（4 字节）：约 **7 位**有效数字，超过就不准了。
- `double`（8 字节）：约 **15 位**有效数字，精度高得多。

⭐ **考研里写小数默认用 `double`**，因为它准。`float` 了解即可。

⚠️ 一个易错点：**代码里直接写的小数常量（如 `3.14`），默认就是 `double` 类型**，不是 float。

### 小数的两种写法

C 里小数有两种写法，科学计数法常考：

| 写法 | 长啥样 | 说明 |
|------|--------|------|
| 小数形式 | `3.14`、`.5`、`5.` | 必须有小数点 |
| 科学计数法 | `6.02e23`、`1.5E-3` | `e` 或 `E` 表示"乘以 10 的几次方" |

科学计数法拆解：
- `6.02e23` = 6.02 × 10²³（`e` 后面的 23 是指数）。
- `1.5e-3` = 1.5 × 10⁻³ = 0.0015（指数是负数表示很小的数）。
- ⚠️ 规则：`e` 前面**必须有数字**，`e` 后面**必须是整数**（不能是小数）。`e3` 错，`1e3.5` 错。

### ⚠️ 浮点数精度问题：小数不能精确比较 ⭐

这是个隐蔽的坑：

```c
double a = 0.1 + 0.2;
printf("%.17f\n", a);     // 输出：0.30000000000000004（不是 0.3！）
```

为什么 `0.1 + 0.2 != 0.3`？因为小数在内存里是用二进制存的，很多十进制小数（像 0.1）**用二进制存不精确**，会有微小误差。

⚠️ **后果**：永远**不要直接用 `==` 判断两个小数是否相等**：

```c
if (a == 0.3) { ... }              // ❌ 危险！可能永远不成立
if (a - 0.3 < 0.000001 && 0.3 - a < 0.000001) { ... }   // ✅ 判断"差值足够小"
```

记法：**整数才能精确，小数永远有误差，比较小数要看"差值是否够小"。**

## 六、字符与整数互通 + 转义字符 ⭐⚠️ 高频

### 核心观念：字符 char 本质就是个小整数 ⭐

这是最反直觉、也最爱考的点：**`char` 存的根本不是"字母"，而是这个字母对应的 ASCII 码（一个整数）。**

- 屏幕上你看到 `'A'`，内存里存的其实是数字 **65**。
- `'a'` 存的是 **97**。
- `'0'`（字符零）存的是 **48**（注意不是 0！）。

所以字符可以直接当整数来算、来比较：

```c
char ch = 'A';
printf("%d\n", ch);        // 输出：65（用 %d 把字符当整数打印，看到它的 ASCII 码）
printf("%c\n", ch);        // 输出：A（用 %c 当字符打印）
printf("%c\n", 'A' + 1);   // 输出：B（65+1=66，66 对应 'B'）
printf("%d\n", 'A' + 1);   // 输出：66
```

拆解：
- 同一个值，`%d` 打印就显示数字（ASCII 码），`%c` 打印就显示对应字符。
- `'A' + 1` 先按整数算 = 66，再用 `%c` 打印出 66 对应的字符 'B'。

### ⭐⚠️ 三个必背的字符运算技巧（年年考）

**① 字符数字转真正的数字：`'9' - '0' = 9`**

```c
char ch = '9';
int n = ch - '0';      // n = 57 - 48 = 9
printf("%d\n", n);     // 输出：9
```

为什么？字符 `'9'` 的 ASCII 是 57，`'0'` 是 48，相减 `57 - 48 = 9`，正好得到它代表的真实数字。
⚠️ 用处巨大：从键盘读进来的数字字符，减 `'0'` 就能变回能参与计算的数。

**② 大小写转换：差值是 32**

```c
printf("%c\n", 'A' + 32);   // 输出：a（大写转小写，加 32）
printf("%c\n", 'a' - 32);   // 输出：A（小写转大写，减 32）
printf("%d\n", 'a' - 'A');  // 输出：32（小写比大写大 32）
```

记忆：**小写字母的 ASCII 比对应大写大 32**（'a'=97, 'A'=65，差 32）。
- 更稳的写法（翁恺笔记里的）：大写转小写 `ch - 'A' + 'a'`，小写转大写 `ch - 'a' + 'A'`。这样不用记 32，靠字母相对位置算。

**③ 判断是不是某类字符**：因为字母在 ASCII 表里是连续排的：
- 大写字母：`'A'` ~ `'Z'`（65~90）。
- 小写字母：`'a'` ~ `'z'`（97~122）。
- 数字字符：`'0'` ~ `'9'`（48~57）。
- 所以判断"ch 是不是大写字母"可以写 `ch >= 'A' && ch <= 'Z'`。

### 转义字符：用反斜杠开头的特殊字符 ⭐

有些字符没法直接打（比如"换行"、"制表符"），就用反斜杠 `\` 开头的**转义字符**表示。每个转义字符**算一个字符**：

| 转义字符 | 含义 |
|---------|------|
| `\n` | 换行 |
| `\t` | 横向制表符（Tab，跳到下一个对齐位置） |
| `\\` | 一个反斜杠 `\` |
| `\"` | 一个双引号 `"` |
| `\'` | 一个单引号 `'` |
| `\0` | 空字符（字符串结束标志，M6 会讲） |

⚠️ **考点**：`'\n'`、`'\t'`、`'\\'` 这些虽然写出来是两个符号，但它们**只算 1 个字符**，`sizeof('\n')` 也好，存进 char 也好，都是一个字符。

还能用 ASCII 码值直接表示字符（了解）：
- `\ddd`：用八进制 ASCII 码，如 `\102` 表示 'B'（八进制 102 = 十进制 66）。
- `\xhh`：用十六进制 ASCII 码，如 `\x41` 表示 'A'（十六进制 41 = 十进制 65）。

### getchar / putchar：读写单个字符（了解）

除了 scanf/printf，还有一对专门处理单字符的函数：
- `getchar()`：从键盘读一个字符。
- `putchar(ch)`：往屏幕输出一个字符。

```c
char ch = getchar();   // 读一个字符
putchar(ch);           // 把它打印出来
```

⚠️ 用 `scanf("%c", &ch)` 读字符时有个坑：**空格、回车也会被当成字符读进去**。这点考试偶尔考，先有个印象。

## 七、类型转换：不同盒子之间倒水 ⭐⚠️ 核心

不同类型的数据放一起运算或互相赋值时，会发生"类型转换"。分两种：电脑**自动**帮你转（隐式），和你**手动强制**转（显式）。

### (1) 自动类型转换（隐式）：往"大"的转 ⭐

当一个表达式里混了不同类型，电脑会**自动把"小"类型转成"大"类型**再算，免得丢数据。

转换的级别（从低到高）：
```
char → int → unsigned → long → float → double
（小盒子）                              （大盒子）
```

类比：小杯子的水往大杯子倒，安全不溢出；所以电脑总是把小的转成大的。

⚠️⚠️ **最经典的陷阱（呼应 M1 的整数除法）**：

```c
int a = 5;
double b = a / 2;       // ⚠️ 结果是 2.0，不是 2.5！
double c = a / 2.0;     // ✅ 结果是 2.5
```

逐步分析 `a / 2`：
1. `a` 是 int（5），`2` 也是 int → **两个整数相除，先算整数除法**，`5 / 2 = 2`（小数被砍）。
2. 算完才赋给 double 的 b，转成 `2.0`。
3. **"砍小数"发生在转换之前**，所以补救已经晚了，b 就是 2.0。

而 `a / 2.0`：
1. `2.0` 是 double，是"大"类型。
2. 电脑把 a（int 5）**自动提升**成 double 5.0。
3. `5.0 / 2.0 = 2.5`，正确。

🎯 记死：**想要小数结果，必须让算式里至少有一个数是小数**（写成 `2.0`），否则整数除法先把小数砍了。

### (2) 强制类型转换（显式）：你亲手转 ⭐⚠️ 年年考

在值前面加 `(类型名)`，强行把它转成那个类型：

```c
double x = 3.7;
int y = (int)x;        // y = 3
printf("%d\n", y);     // 输出：3
```

⚠️⚠️⚠️ **最重要的考点：`(int)` 是"截断"，不是"四舍五入"！**

```c
printf("%d\n", (int)3.99);    // 输出：3（不是 4！）
printf("%d\n", (int)3.01);    // 输出：3
printf("%d\n", (int)9.99);    // 输出：9
```

`(int)` 把小数部分**整个砍掉**，无论小数多大，都不进位。`3.99` 砍成 3，`9.99` 砍成 9。

类比：切水果，`(int)` 是"咔嚓一刀把小数那截全切掉"，不管那截多长。它不会"看够不够一半来决定进位"。

🎯 真题常这么考：问 `(int)3.99` 等于几，标准答案 **3**。掉进"四舍五入=4"陷阱的人一大片。

⚠️ 强制转换的优先级很高（和 `++`、`--` 一样），结合性从右向左。看这个：
```c
(int)3.8 + 1.3      // 等价于 ((int)3.8) + 1.3 = 3 + 1.3 = 4.3
```
`(int)` 只作用于紧跟它的 `3.8`，先把 3.8 转成 3，再加 1.3 得 4.3。

⚠️ 注意：强制转换**只是临时取个转换后的值**，不改变原变量本身的类型。`(int)x` 之后，x 还是那个 double 3.7。

## 八、表达式与运算符：求值的各种陷阱 ⭐⚠️

### (1) 自增自减 `++` / `--`：先用还是先加？ ⭐⚠️ 输出预测题高频

这是输出预测题的重灾区。`++` 让变量加 1，`--` 让变量减 1，但**放在变量前面还是后面，结果天差地别**：

- `i++`（后缀，i 在前）：**先用旧值，再加 1**。
- `++i`（前缀，++ 在前）：**先加 1，再用新值**。

记忆口诀：**符号在后用旧值，符号在前用新值。** 或者看 `++` 的位置——`++` 在哪边就先动哪边。

```c
int i = 5;
int a = i++;    // 先把旧值 5 给 a，然后 i 才变 6 → a=5, i=6
printf("a=%d, i=%d\n", a, i);   // 输出：a=5, i=6

int j = 5;
int b = ++j;    // 先把 j 加成 6，再把 6 给 b → b=6, j=6
printf("b=%d, j=%d\n", b, j);   // 输出：b=6, j=6
```

逐步拆 `a = i++`：
1. i 现在是 5。
2. `i++` 是后缀，**先把当前值 5 拿出来**当表达式的值。
3. 这个 5 赋给 a，所以 a = 5。
4. 然后 i 自己加 1，变成 6。
5. 最终 a=5, i=6。

拆 `b = ++j`：
1. j 现在是 5。
2. `++j` 是前缀，**先把 j 加 1 变成 6**。
3. 再把新值 6 当表达式的值，赋给 b，b = 6。
4. 最终 b=6, j=6。

🎯 **真题套路**：常和 printf 连在一起考，比如：
```c
int i = 3;
printf("%d %d\n", i++, ++i);    // 这种题各编译器结果不一，属于"未定义行为"，慎答
```
⚠️ 注意：把同一个变量的 `i++` 和 `++i` 写在一个表达式里（像上面），C 标准**没规定求值顺序**，结果因编译器而异，叫"未定义行为"。规范的代码不该这么写，但考试可能拿来吓你——遇到这种题先判断"是否依赖求值顺序"。

⚠️ `++` / `--` 的运算对象**只能是变量**，不能是常量或表达式。`5++` 错，`(a+b)++` 错。

### (2) 复合赋值运算符 `+= -= *= /= %=` ⭐

这是简写，让代码更短：

```c
int x = 10;
x += 5;      // 等价于 x = x + 5;  → x = 15
x -= 3;      // 等价于 x = x - 3;  → x = 12
x *= 2;      // 等价于 x = x * 2;  → x = 24
x /= 4;      // 等价于 x = x / 4;  → x = 6
x %= 4;      // 等价于 x = x % 4;  → x = 2
```

⚠️ 一个易错点（带运算的复合赋值）：
```c
x *= y + 3;     // 等价于 x = x * (y + 3)，不是 x = x*y + 3！
```
右边整个表达式会被当作一个整体（自动加括号）。记法：**`x op= 表达式` = `x = x op (表达式)`**，注意那对括号。

### (3) 赋值表达式的结合性：从右向左 ⭐

```c
int x, y;
x = y = 3;       // 等价于 x = (y = 3)
```
- 先算 `y = 3`，y 变成 3，这个赋值表达式的值也是 3。
- 再 `x = 3`，x 也变成 3。
- 赋值是**从右向左**结合的，所以从最右边开始算。

### (4) 关系表达式：结果是 1 或 0 ⭐⚠️ 经典陷阱

关系运算（`> < >= <= == !=`）的结果只有两个：**真是 1，假是 0**。

```c
int a = 5 > 3;     // 5>3 成立，a = 1
int b = 5 < 3;     // 5<3 不成立，b = 0
printf("%d %d\n", a, b);   // 输出：1 0
```

⚠️⚠️ **经典送命题：`3 <= x <= 5` 不是数学的"3到5之间"！**

```c
int x = 100;
int r = (3 <= x <= 5);    // 你以为 x=100 不在 3~5，r 应该是 0？错！r = 1
```

为什么 r 恒等于 1？因为它是**从左到右**算的，等价于 `(3 <= x) <= 5`：
1. 先算 `3 <= x`：x=100，成立，结果是 **1**。
2. 再算 `1 <= 5`：成立，结果是 **1**。
3. 所以无论 x 是多少，`(3<=x)` 不是 0 就是 1，永远 ≤ 5，**整个表达式恒为 1**。

🎯 想表达"x 在 3 到 5 之间"必须写：`3 <= x && x <= 5`（用 `&&` 连起来）。这是 M2 分支会用到的，记住这个坑。

### (5) 逻辑运算 `&& || !` 与短路求值 ⭐⚠️

- `&&`（与）：两边都真才真。
- `||`（或）：有一边真就真。
- `!`（非）：真变假，假变真。
- C 里**非 0 就算真，0 算假**。

⚠️ **短路求值**（考点）：
- `a && b`：如果 `a` 已经是假（0），**`b` 根本不算了**（反正结果必假）。
- `a || b`：如果 `a` 已经是真，**`b` 不算了**（反正结果必真）。

```c
int x = 0;
int y = (x != 0) && (10 / x > 1);   // x!=0 是假，后半段 10/x 不执行，避免了除以0
```
🎯 短路特性常被用来"防止除以 0"或"防止访问空指针"，考试会考"后半段表达式到底执行没"。

### (6) 条件表达式（三目运算符）`? :` ⭐

唯一一个需要三个部分的运算符，是简化版的 if-else：

```c
int a = 5, b = 8;
int max = (a > b) ? a : b;     // 求 a、b 中的较大值
printf("%d\n", max);           // 输出：8
```

读法：`条件 ? 真时的值 : 假时的值`。
- 先算 `a > b`：5>8 不成立（假）。
- 假 → 取冒号**后面**的 b（8）。
- 所以 max = 8。

记忆：**问号前是条件，问号后是"真值"，冒号后是"假值"。**

### (7) 逗号表达式（了解）

用逗号连起来的一串表达式，**从左到右依次算，整体的值取最后一个**：

```c
int a = (3, 5, 7);     // 依次算 3、5、7，整体值 = 最后的 7
printf("%d\n", a);     // 输出：7

int b;
b = (b = 3, b + 2);    // 先 b=3，再算 b+2=5，整体值 5 → b=5
```

了解即可，知道"逗号表达式的值是最后一个"就行。

### (8) 位运算（了解，偶尔考） ⭐

直接对二进制位操作，只能用于整数和字符：

| 运算符 | 含义 | 例子 |
|--------|------|------|
| `&` | 按位与 | 都是 1 才 1 |
| `\|` | 按位或 | 有 1 就 1 |
| `^` | 按位异或 | 不同为 1，相同为 0 |
| `~` | 按位取反 | 0 变 1，1 变 0 |
| `<<` | 左移 | **左移 1 位 ≈ 乘 2** |
| `>>` | 右移 | **右移 1 位 ≈ 除 2** |

```c
int a = 5;
printf("%d\n", a << 1);    // 5 左移1位 = 10（相当于 ×2）
printf("%d\n", a >> 1);    // 5 右移1位 = 2（相当于 ÷2，砍小数）
```

⚠️ 考点：**左移 = 乘 2 的幂，右移 = 除 2 的幂**（对正数/无符号数）。`a << 3` = a × 8。位运算不改变原操作数。

## 九、运算符优先级与结合性 ⭐⚠️

一个表达式里好几种运算符，**谁先算？** 这就是优先级。

### 优先级口诀（从高到低）

```
①  ! ++ -- (单目) (类型)强转    ← 最高，从右向左
②  *  /  %                      ← 算术：乘除模
③  +  -                         ← 算术：加减
④  <<  >>                       ← 移位
⑤  <  <=  >  >=                 ← 关系
⑥  ==  !=                       ← 相等判断
⑦  &&                           ← 逻辑与
⑧  ||                           ← 逻辑或
⑨  ? :                          ← 条件
⑩  =  +=  -=  *= ...            ← 赋值，最低，从右向左
```

简记一句话：**单目 > 算术 > 关系 > 逻辑 > 条件 > 赋值。**

### 结合性是什么？

同优先级的运算符，按什么方向算：
- **大部分从左向右**：`10 - 5 - 2` = `(10-5)-2` = 3。
- **单目、赋值、条件 从右向左**：`x = y = 3` 先算右边的 `y = 3`。

### 看几个求值例子

```c
printf("%d\n", 2 + 3 * 4);       // 14（先算 3*4=12，再 +2），不是 20
printf("%d\n", 10 % 3 + 1);      // 2（先 10%3=1，再 +1）
printf("%d\n", 5 > 3 == 1);      // 1（先 5>3=1，再 1==1=1）
printf("%d\n", 1 + 2 > 2 + 1);   // 0（先算两边加法 3>3，不成立=0）
```

🎯 **应试技巧**：考试故意不加括号让你算。**拿不准就在心里加括号**，按"先乘除模、后加减、再关系、最后逻辑赋值"的顺序拆。自己写代码时**多加括号**，宁可啰嗦也别算错。

## 十、新手最常踩的坑（排查清单）⚠️

把这一章的坑集中列一遍，做题前过一眼：

1. **以 0 开头的整数是八进制**：`012` 是 10 不是 12。看到数字前面有 0 就警觉。
2. **char 溢出回绕**：127 + 1 = -128，255（unsigned char）+ 1 = 0。装不下就绕圈。
3. **`(int)` 是截断不是四舍五入**：`(int)3.99` = 3。
4. **整数除法先砍小数**：`5/2 = 2`，`double b = 5/2` 还是 2.0。要小数就写 `5.0/2`。
5. **负数赋给 unsigned 变巨大正数**：`unsigned int u = -1` → 42 亿多。
6. **小数别用 `==` 比较**：`0.1+0.2 != 0.3`，要看差值够不够小。
7. **`i++` 和 `++i` 别搞混**：后缀用旧值，前缀用新值。
8. **`3<=x<=5` 不是区间判断**：恒为 1，要用 `3<=x && x<=5`。
9. **字符 `'0'` 不是数字 0**：`'0'` 的 ASCII 是 48；字符转数字要 `ch - '0'`。
10. **`%` 取余只能用整数**：呼应 M1，小数不能取余。
11. **`%` 取余、`/` 整除是 M1 老朋友**：这章把它们的"类型来源"讲清楚了——都因为两边是整数。

## 十一、本章理论速记（专攻选择题）📚⭐

> 不用写代码、但选择题爱考的纯理论，背下来就是送分题。

### ① 三种基本数据类型
C 语言的三种基本数据类型是：**整型、字符型、实型（浮点型）**。选择题常问。

### ② 字节大小（必背）
`char` 1 字节、`int` 4 字节、`float` 4 字节、`double` 8 字节。1 字节 = 8 位 = 256 种状态。

### ③ char 取值范围与"为什么"
`char`：-128 ~ 127；`unsigned char`：0 ~ 255。负数比正数多一个，因为 0 占了正数的位置。

### ④ 实型常量默认是 double
代码里写的 `3.14` 默认是 `double`，不是 float。

### ⑤ 强制转换优先级
`(类型)` 强转的优先级和 `++`、`--` 一样高，结合性从右向左。

### ⑥ 求值顺序未定义
C 没规定运算符两侧的求值顺序（如 `f() + g()`），不要写依赖求值顺序的表达式。

### ⑦ sizeof 是运算符
`sizeof` 是运算符不是函数，编译时就算出结果。

## 十二、本节考点速记 ⭐

1. 字节大小：**char 1、int 4、float 4、double 8**；1 字节 = 8 位 = 256 种状态
2. `sizeof` 量字节数，是**运算符**，编译时确定
3. 整数常量：**0 开头是八进制**（`010`=8），`0x` 开头是十六进制（`0x10`=16）
4. 补码存负数；**char 范围 -128~127**，unsigned char 0~255（负数比正数多一个）
5. **溢出回绕**：char 127+1=-128，unsigned char 255+1=0（里程表归零）
6. **负数赋 unsigned → 超大正数**：`unsigned int u = -1` 是最大值
7. 实型常量默认 **double**；`float` 7 位精度，`double` 15 位
8. **小数有误差，禁止用 `==` 比较**（0.1+0.2≠0.3）
9. **字符本质是 ASCII 整数**：'A'=65, '0'=48；`'9'-'0'=9`，大小写差 32
10. 转义字符 `\n \t \\ \" \'` 各算**1 个字符**
11. **自动转换往大转**：`char→int→...→double`；int/double 混算会提升
12. ⭐ **`(int)` 是截断不是四舍五入**：`(int)3.99=3`
13. ⭐ 整数除法先砍小数：`5/2=2`，要小数得 `5.0/2`
14. **`i++` 用旧值，`++i` 用新值**；运算对象只能是变量
15. 复合赋值 `x*=y+3` = `x=x*(y+3)`（注意括号）
16. 关系结果是 **1 或 0**；`3<=x<=5` 恒为 1，要用 `3<=x && x<=5`
17. 逻辑**短路求值**：`&&` 左假右不算，`||` 左真右不算
18. 三目 `条件 ? 真值 : 假值`；逗号表达式取**最后一个**的值
19. 移位：**左移≈乘2，右移≈除2**
20. 优先级口诀：**单目 > 算术 > 关系 > 逻辑 > 条件 > 赋值**
