## 〇、开始之前：什么是"分支结构"？

M1 里我们写的程序，都是**从上到下、一条不落地全跑一遍**——这叫"顺序结构"。
但真实世界里很多事是"看情况办"的：

- 下雨**就**带伞，不下雨**就**不带。
- 考了 60 分以上**就**算及格，否则**就**挂科。
- 红灯**停**，绿灯**行**。

这种"**如果……就……否则……**"的逻辑，就是**分支结构**（也叫**选择结构**）。

想象你走在路上遇到一个**岔路口**：路牌上写着"如果你要去学校，往左；否则往右"。你看一眼条件（要不要去学校），然后**只走其中一条路**，另一条就不走了。
程序的分支也是这样：电脑先看一个**条件**成不成立，成立走一条路，不成立走另一条路。**两条路不会同时走。**

> ⭐ 记住 M1 提过的"程序三种基本结构"：**顺序、分支（选择）、循环**。这一节学的就是第二种——分支。

这一节的主角是两个东西：`if` 语句（管"如果"）和 `switch` 语句（管"挑一个"）。

## 一、if 单分支：只有"如果……就……" ⭐

最简单的分支：**条件成立就做某件事；不成立就啥也不干，直接跳过。**

```c
if (条件) {
    // 条件成立（为真）时，执行这里
}
// 不管成不成立，都从这里继续往下走
```

逐字拆解，一个都不跳：

- `if` —— 关键字，读作"if"，意思就是"如果"。固定写法，全小写。
- `( )` —— **小括号**，里面放"要判断的条件"。⚠️ 这个括号不能省。
- `条件` —— 一个会算出"真 / 假"的式子，比如 `score >= 60`（分数大于等于60）。
- `{ }` —— **大括号**，里面放"条件成立时要做的事"。

举个最实在的例子：

```c
int score = 75;
if (score >= 60) {
    printf("及格了！\n");
}
printf("程序结束\n");
```

运行过程（电脑的内心戏）：
1. 走到 `if`，先算括号里的条件 `score >= 60`：75 ≥ 60 成立，**为真**。
2. 既然为真，就执行大括号里的 `printf("及格了！\n")`，屏幕打出"及格了！"。
3. 出了大括号，继续往下，打出"程序结束"。

输出：
```
及格了！
程序结束
```

如果把 `score` 改成 50 呢？

```c
int score = 50;
if (score >= 60) {
    printf("及格了！\n");
}
printf("程序结束\n");
```

这次条件 `50 >= 60` **不成立（为假）**，大括号里的话**整段被跳过**，直接打"程序结束"。

输出：
```
程序结束
```

> ⭐ 单分支 `if` 的精髓：**真就做，假就跳过。** 没有"否则"，假的时候就当这段不存在。

## 二、if-else 双分支："如果……否则……" ⭐⚠️ 重点

单分支只能管"成立时干嘛"。但很多时候我们想要"成立干这个，**不成立干那个**"——两条路都要安排。这时用 `if-else`：

```c
if (条件) {
    // 条件为真（成立）→ 走这条路
} else {
    // 条件为假（不成立）→ 走这条路
}
```

- `else` —— 关键字，读作"else"，意思是"否则、不然"。
- 它**紧跟在 `if` 的大括号后面**，管的是"条件不成立时干嘛"。
- ⚠️ `else` 不能单独存在，必须先有 `if`。光写个 `else` 是语法错误。

完整例子——判断及格还是挂科：

```c
int score = 50;
if (score >= 60) {
    printf("及格了\n");
} else {
    printf("挂科了，补考见\n");
}
```

电脑的内心戏：
1. 算条件 `50 >= 60`：不成立，**为假**。
2. 为假，所以**跳过 if 的大括号**，去执行 `else` 的大括号。
3. 打出"挂科了，补考见"。

输出：
```
挂科了，补考见
```

> ⭐ `if-else` 的精髓（翁恺原话）：**"二者必执其一"。** 真走 if，假走 else，**绝不会两条都走，也绝不会两条都不走**。这跟岔路口一个道理：你只能走一条路。

### 用 if-else 判断奇偶（翁恺例3-2，超经典）

判断一个数是奇数还是偶数，用 M1 学过的取余 `%`：偶数除以 2 余 0。

```c
int n = 7;
if (n % 2 == 0) {
    printf("%d 是偶数\n", n);
} else {
    printf("%d 是奇数\n", n);
}
// 输出：7 是奇数
```

拆解条件 `n % 2 == 0`：
- `n % 2` —— n 除以 2 的余数（M1 学过，`%` 是取余）。
- `== 0` —— 判断这个余数**是不是等于 0**。
- 余数是 0 → 偶数；余数是 1 → 奇数。

⚠️ 注意这里是 `== 0`（两个等号，判断"是不是"），**不是 `= 0`**（一个等号是赋值）。这个坑下面第五节专门讲，先有个印象。

## 三、大括号能不能省？复合语句 ⭐⚠️

你可能见过有人写 `if` 不带大括号，像这样：

```c
if (score >= 60)
    printf("及格了\n");
```

这是**合法的**。规则是：

> `if`（或 `else`）后面**不写大括号时，只管紧跟着的那一条语句**。

也就是说，`if (条件) 语句;` 里，条件成立只执行**那一句**。

### ⚠️⚠️ 不写大括号的致命陷阱（改错题高频）

问题来了：如果你想让 `if` 成立时干**两件事**，却没加大括号——

```c
if (score >= 60)
    printf("及格了\n");
    printf("恭喜你\n");      // ❌ 这一行其实不归 if 管！
```

新手以为：成立时打两句，不成立时啥也不打。
**真相**：`if` 没加大括号，只管**紧跟的第一句** `printf("及格了\n")`。第二句 `printf("恭喜你\n")` 跟 `if` **没半毛钱关系**，它是"无论如何都会执行"的独立语句！

所以哪怕 score 是 30（不及格），也会打出"恭喜你"。缩进骗了你的眼睛——**电脑不看缩进，只看语法**。

✅ 正确做法：想让 `if` 管多条语句，**必须用大括号 `{ }` 把它们框成一组**：

```c
if (score >= 60) {
    printf("及格了\n");
    printf("恭喜你\n");
}
```

用大括号框起来的"一组语句"，在 C 里叫**复合语句**（翁恺原词）。电脑把这一整组当成"一条"来对待。

> ⭐ 给新手的硬建议：**`if`/`else` 后面永远写大括号**，哪怕只有一句。这样永远不会踩"漏管"的坑，也不怕以后加代码忘了补括号。考研改错题最爱考"漏大括号导致逻辑错"。

## 四、关系运算符：怎么"提条件" ⭐

`if` 括号里的"条件"是怎么写的？靠**关系运算符**——专门用来"比大小、比相等"的符号，一共 6 个：

| 运算符 | 含义 | 例子 | 例子的真假 |
|--------|------|------|-----------|
| `>` | 大于 | `5 > 3` | 真 |
| `<` | 小于 | `5 < 3` | 假 |
| `>=` | 大于等于 | `5 >= 5` | 真 |
| `<=` | 小于等于 | `4 <= 3` | 假 |
| `==` | **等于（判断相等）** | `5 == 5` | 真 |
| `!=` | 不等于 | `5 != 3` | 真 |

逐个说几个新手容易栽的点：

- `>=` 和 `<=` —— **两个符号连写，中间不能有空格**，而且**大于号/小于号在前，等号在后**。✅ `>=`、`<=`；❌ `=>`、`=<`（写反了报错）。
- `==` —— **两个等号**才是"判断相等"。这是本节最重要的符号，下面单独开一节讲它和 `=` 的区别。
- `!=` —— 读作"不等于"。`!` 在 C 里表示"非 / 不"，所以 `!=` 就是"不等于"。

关系运算的结果，本质上就是一个"真 / 假"。在 C 里，这个真假是用数字表示的（下一节细讲）。

```c
int a = 5, b = 3;
printf("%d\n", a > b);   // 输出 1  ← 真，C 用 1 表示
printf("%d\n", a < b);   // 输出 0  ← 假，C 用 0 表示
```

⚠️ 看到没？比较运算 `a > b` 本身能算出一个数（1 或 0），可以直接 printf 出来。这点理解了，下一节"真和假"就好懂了。

## 五、💀 全场最大的坑：`==` 和 `=` 的天壤之别 ⭐⚠️⚠️ 改错题必考

这一节单拎出来，因为它是**考研改错题、选择题年年考的送命题**。新手十个有九个栽过。

### 它俩到底啥区别？

| 符号 | 名字 | 作用 | 读法 |
|------|------|------|------|
| `=` | 赋值号（一个等号） | **把右边的值装进左边的盒子** | "把…赋给…" |
| `==` | 相等运算符（两个等号） | **判断两边是不是相等**，算出真(1)/假(0) | "等不等于" |

- `x = 5` 的意思是：**把 5 装进盒子 x**（一个动作，M1 学过）。
- `x == 5` 的意思是：**问一句"x 是不是等于 5？"**（一个判断，结果是真或假）。

一个是"**装东西**"，一个是"**问问题**"。完全两回事。

### ❌ 把 `==` 写成 `=` 会发生什么？（必考）

```c
int x = 3;
if (x = 5) {              // ❌ 手滑写成一个等号！
    printf("x 等于 5\n");
}
```

新手以为：判断 x 是不是 5，3 不等于 5，所以不打印。
**真相**：`x = 5` 是个赋值动作！它干了两件事：
1. **把 5 装进 x**（现在 x 被偷偷改成了 5）。
2. 这个赋值表达式**本身的值，就是刚装进去的 5**。

然后 `if` 拿到的条件值是 **5**。C 里非 0 就是真（下面讲），所以**条件恒为真**，永远会打印"x 等于 5"——不管 x 原来是几！

> 💀 这就是为什么 `if (x = 5)` 是个 bug 制造机：它**永远成立**，而且**偷偷篡改了 x 的值**。考研改错题给你一段 `if (a = 0)` / `if (flag = 1)`，让你找错，答案就是"`=` 应改为 `==`"。

### ✅ 正确写法

```c
int x = 3;
if (x == 5) {             // ✅ 两个等号，判断相等
    printf("x 等于 5\n");
}
// x 是 3，不等于 5，条件为假，不打印。x 的值也没被改动。
```

### 🎯 真题级辨析

```c
if (a = 0)    // 把 0 赋给 a，表达式值为 0 → 假 → 永远不执行 if 块
if (a == 0)   // 判断 a 是否为 0 → a 是 0 才执行
```

⚠️ 特别坑：`if (a = 0)` 恒为假，`if (a = 5)` 恒为真——只要等号右边是非 0 就恒真，是 0 就恒假。

### 怎么防止手滑？老程序员的偏方

把常量写在左边：`if (5 == x)`。这样万一手滑写成 `if (5 = x)`，编译器会直接报错（因为不能"把 x 赋给常量 5"），帮你当场抓住错误。这招叫"尤达表达式"，了解即可，考试看到别懵。

## 六、C 语言里"真"和"假"到底是什么？ ⭐⚠️ 必考

这是 C 的一个核心设定，**选择题超爱考**：

> **C 语言没有专门的"真假"，它用数字表示：0 就是假，任何非 0 的数都是真。**

记牢这句话（翁恺原话："C语言用非0表示真，0表示假"）：
- `0` → **假**（false）
- `非 0`（1、-1、5、100、-99……）→ **真**（true）

⚠️ 注意：**不是只有 1 才是真**。`-3` 是真，`100` 是真，`0.5` 是真，只要不是 0 就是真。

### 这意味着什么？看几个例子

```c
if (5) {
    printf("会执行\n");      // ✅ 5 是非 0，为真，会打印
}

if (0) {
    printf("不会执行\n");    // ❌ 0 为假，整段跳过
}

if (-7) {
    printf("也会执行\n");    // ✅ -7 是非 0，照样为真
}
```

### 反过来：关系运算的结果就是 0 和 1

前面第四节看到了，`a > b` 这种比较，算出来的结果**真就是 1，假就是 0**。所以：

```c
int r = (5 > 3);
printf("%d\n", r);    // 输出 1（因为 5>3 为真，真用 1 表示）
```

### ⚠️ 把这两件事连起来，就能解释第五节那个坑

```c
if (x = 5)    // x=5 这个赋值表达式的值是 5，5 是非 0 → 真 → 恒成立
if (x = 0)    // x=0 这个赋值表达式的值是 0，0 → 假 → 恒不成立
```

现在你彻底明白为什么 `if (x = 5)` 永远成立了吧——因为它的"条件值"是 5，而 5 是非 0 的真。

> ⭐ 考点速记：**0 为假，非 0 为真；关系/逻辑运算的结果，真是 1、假是 0。**

## 七、else if 多分支：三条以上的岔路 ⭐⚠️ 重点

`if-else` 只能二选一。但现实里常常有**好几个挡位**：分数 90 以上是 A，80-89 是 B，60-79 是 C，不到 60 是 D。四个分支，怎么办？

用 **`else if` 阶梯**（翁恺叫"多分支结构和else-if语句"）：

```c
int score = 85;
if (score >= 90) {
    printf("等级 A\n");
} else if (score >= 80) {
    printf("等级 B\n");
} else if (score >= 60) {
    printf("等级 C\n");
} else {
    printf("等级 D\n");
}
```

### 它是怎么走的？（电脑内心戏）⭐

关键规则：**从上往下一个一个试，谁先成立就执行谁，然后整个阶梯立刻结束，后面的全部跳过。**

以 `score = 85` 为例：
1. 试 `85 >= 90`？不成立，跳过，往下。
2. 试 `85 >= 80`？**成立！** 打印"等级 B"，然后**整个 if-else if 结构直接结束**，下面的 `else if (score >= 60)` 和 `else` 看都不看。
3. 完事。

输出：`等级 B`

> ⭐ 翁恺原话："一旦某个表达式为真，则执行其对应的语句，并结束整个 else-if 链。" 命中一个就跑路，是 else if 的灵魂。

### ⚠️⚠️ 顺序写错，结果全错（高频坑）

为什么上面 `score >= 80` 能正确抓住 85，而没被 `score >= 60` 抢走？因为**顺序是从大到小排的**。如果顺序写反：

```c
int score = 85;
if (score >= 60) {
    printf("等级 C\n");        // ❌ 85>=60 成立！直接打 C 就跑路了
} else if (score >= 80) {
    printf("等级 B\n");        // 永远轮不到
} else if (score >= 90) {
    printf("等级 A\n");        // 永远轮不到
}
// 输出：等级 C  （85 分被判成 C，错得离谱）
```

85 分明明该是 B，却因为 `score >= 60` 写在最前面、先成立先跑路，被错判成 C。

> ⚠️ 记死：**写 else if 阶梯，条件要按"由严到松"或"由大到小"的顺序排**，否则前面松的条件会把后面的全截胡。这是考研改错题常见考法。

### 为什么不用一堆独立的 if？

有人会想：我写四个独立 `if` 不行吗？

```c
if (score >= 90) printf("A\n");
if (score >= 80) printf("B\n");   // ⚠️ 注意都是独立 if，不是 else if
if (score >= 60) printf("C\n");
```

`score = 95` 时会怎样？三个 `if` 各自独立判断，95 同时满足前三个条件，于是**A、B、C 全打出来**！因为独立 `if` 之间没有"命中一个就跑路"的关系。

> ⭐ 区别记牢：**`else if` 阶梯 = 只命中一个；一堆独立 `if` = 每个都判一遍，可能命中多个。** 想"多选一"必须用 else if。

## 八、逻辑运算符：把多个条件拼起来 ⭐⚠️ 重点

有时一个条件不够用。比如"年龄在 18 到 60 之间"，得同时满足"≥18"**并且**"≤60"。这就要用**逻辑运算符**把多个条件粘起来。三个符号：

| 运算符 | 名字 | 意思 | 什么时候为真 |
|--------|------|------|------------|
| `&&` | 逻辑与 | **并且**（两个都要满足） | 两边都为真，才为真 |
| `\|\|` | 逻辑或 | **或者**（满足一个就行） | 两边至少一个为真，就为真 |
| `!` | 逻辑非 | **取反**（反过来） | 真变假、假变真 |

⚠️ 注意符号：`&&` 是**两个** `&`（与号），`||` 是**两个**竖线 `|`（这个键在回车键上方，shift + 反斜杠）。写一个 `&` 或一个 `|` 是另一种东西（位运算），意思完全不同，别写漏。

### `&&` 逻辑与：两个都得真

```c
int age = 25;
if (age >= 18 && age <= 60) {
    printf("是劳动年龄人口\n");   // 25 既≥18 又≤60，两个都真，整体真
}
```

读法："age 大于等于 18 **并且** age 小于等于 60"。只要有一个不满足，整体就为假。

⚠️ 新手常犯的数学式写法：

```c
if (18 <= age <= 60)    // ❌ 大错特错！C 不能这样写区间
```

这在数学里对，在 C 里**会算出莫名其妙的结果**（C 会先算 `18 <= age` 得到 0 或 1，再拿这个 0/1 去跟 60 比，永远成立）。**C 里表示区间必须拆成两个条件用 `&&` 连**：`age >= 18 && age <= 60`。这是高频坑。

### `||` 逻辑或：有一个真就行

```c
int day = 6;
if (day == 6 || day == 7) {
    printf("是周末\n");   // day 是 6，满足左边，整体就真（右边都不用看了）
}
```

读法："day 等于 6 **或者** day 等于 7"。两个里有一个成立，整体就成立。

### `!` 逻辑非：把真假掉个个儿

```c
int isRaining = 0;       // 0 表示"没下雨"
if (!isRaining) {        // !0 → 真
    printf("不用带伞\n");
}
```

`!` 把后面的真假反过来：`!0` 是真，`!5`（5 是真）是假。常用来写"如果不……"。

### ⚠️⚠️ 短路求值（Short-circuit）⭐ 高频考点

这是逻辑运算最爱考的点。规则（翁恺原话）：

> - `a && b`：**如果 a 已经是假，b 根本不算了**（反正整体不可能真了）。
> - `a || b`：**如果 a 已经是真，b 根本不算了**（反正整体已经真了）。

电脑很懒，结果已经定了就不浪费力气算后面。看例子：

```c
int a = 0, b = 0;
if (a != 0 && b == 0) {   // a!=0 是假，整体已注定为假，b==0 不会去算
    ...
}
```

考试爱这样考：给一个表达式，问你"哪个部分没被执行"或"某个变量值变没变"：

```c
int x = 5;
int y = 0;
if (y != 0 && x / y > 1) {   // y!=0 为假 → 短路 → x/y 根本不执行
    printf("yes\n");
}
printf("没崩溃\n");           // 正常打印
```

⚠️ 这里 `y` 是 0，如果硬算 `x / y` 会"除以 0"崩溃。但因为 `y != 0` 为假触发了**短路**，`x / y` 压根没执行，程序安然无恙。**短路求值常被用来"防止除以 0 / 防止访问空指针"**，这是它的实际用途。

🎯 真题常见问法：`a++ && b++`，如果 a 初值让 `a++` 为假，那 `b++` 执行了吗？答案：**没执行，b 不变**。

### ⚠️ 逻辑运算符的优先级 ⭐

翁恺给的优先级顺序（从高到低）：

> `!`（非） > 算术运算符 > 关系运算符 > `&&`（与） > `||`（或） > 赋值运算符 `=`

通俗记：**`!` 最霸道（最先算），`&&` 比 `||` 优先，`=` 最弱（最后算）**。

举例：`a > 0 && b > 0 || c > 0` 等价于 `((a > 0) && (b > 0)) || (c > 0)`——因为 `&&` 比 `||` 先算。

> ⚠️ 拿不准优先级时，**老老实实加括号**。`(a > 0 && b > 0) || c > 0` 写清楚，自己不晕、改卷老师也舒服。考试别为了炫技省括号。

## 八点五、⭐ 考研衔接：真题里的"合法性卫语句"长这样（认脸）

> 这节把刚学的逻辑运算符、短路求值，对应到你将来刷 **408 数据结构大题**时第一眼会看到的东西。不是新语法，是同一套 `||`/`&&` 在真题里的固定用法，提前认脸。

数据结构大题的插入、删除函数，**开头几乎都顶着一行"边界检查"**，先把不合法的情况挡掉再干活。比如"在顺序表第 i 个位置插入"：

```c
bool ListInsert(SqList *L, int i, ElemType e) {
    if (i < 1 || i > L->length + 1) {   // ← 这行就是"合法性卫语句"
        return false;                   // i 不合法，直接返回，不往下做
    }
    // ……下面才是真正的插入……
}
```

逐字看那行 `if (i < 1 || i > L->length + 1)`，全是你这章学的：

- `i < 1` —— 位序太小（位序从 1 开始，比 1 还小就非法）。
- `i > L->length + 1` —— 位序太大（在末尾后一位插入是允许的，所以上界是 `length + 1`）。
- `||` —— **逻辑或**：上面两种非法情况，**只要中一个**，整体就为真 → 进 if → 返回错误。
- 这正是第八节的 `||`：有一个真就真。两个条件用 `||` 连，把"太小"和"太大"两种越界一网打尽。

⚠️ 几个衔接到本章考点的点：
- **不能写成 `1 <= i <= length+1`**！（第八节的高频坑）这在 C 里是从左到右算、恒不对，必须拆成 `i >= 1 && i <= length+1`，或反过来挡非法 `i < 1 || i > length+1`。
- "卫语句"（guard）的思路：**先用 if 把非法/边界情况挡在门外、提前 return，剩下的代码就能安心假设输入合法。** 这是真题函数的标准开头套路。
- `L->length` 的箭头 `->` 是 M7/M8 的内容（指针取成员），现在只要知道它是"取这张表的当前长度"即可。

> 📌 一句话：**真题函数开头的 `if (i < 1 || i > length+1) return 错误;` 就是用本章的 `||` 做"边界合法性检查"。** 记住"区间判断要拆成两个条件用 `||`/`&&` 连"，这行你就能读懂、也能自己写。

## 九、字符型数据：分支里常拿来判断的 char ⭐

分支里经常要判断"用户输入的是哪个字符"（比如四则运算输入的是 `+` 还是 `-`），所以这里补一下 `char`（翁恺第3章专门讲了字符型，因为四则运算和 switch 都要用）。

### 字符常量用单引号

```c
char op = '+';      // ✅ 单个字符，用单引号 ' '
```

⚠️ M1 提过：**单个字符用单引号 `'A'`，一串文字用双引号 `"A"`**。`'+'` 是字符，`"+"` 是字符串，别混。

### ⭐⚠️ 字符的本质：每个字符背后是一个数字（ASCII 码）

这是个**必考点**。电脑存字符，其实存的是一个编号，这套编号表叫 **ASCII 码**。比如：
- `'A'` 的 ASCII 码是 **65**，`'B'` 是 66，`'C'` 是 67……
- `'a'` 是 **97**，`'b'` 是 98……
- `'0'`（字符零，不是数字零）是 **48**，`'1'` 是 49……

翁恺强调的关键特性：**数字字符 `'0'~'9'`、大写 `'A'~'Z'`、小写 `'a'~'z'`，各自的 ASCII 码都是连续递增的。**

这个"连续"特别有用，比如判断一个字符是不是字母、是不是数字：

```c
char ch = 'k';
if (ch >= 'a' && ch <= 'z') {
    printf("是小写字母\n");      // 因为 a~z 编码连续，落在区间内就是小写
}
if (ch >= '0' && ch <= '9') {
    printf("是数字字符\n");
}
```

字符能直接比大小、能减出差值（比如 `'9' - '0'` 得到 9），就是因为它们底层是数字。

```c
char c = 'A';
printf("%d\n", c);    // 输出 65 ← 用 %d 看 char，会打出它的 ASCII 码
printf("%c\n", 66);   // 输出 B  ← 用 %c 看数字 66，会打出对应字符
```

⚠️ 考点：`%c` 看字符本身，`%d` 看它的 ASCII 编码。同一个 char，换占位符就换看法。

### 字符的输入输出

- 用 `scanf("%c", &ch)` 读、`printf("%c", ch)` 打，占位符都是 `%c`。
- 翁恺还讲了专用函数：`ch = getchar();`（读一个字符）、`putchar(ch);`（输出一个字符），更直接。

⚠️⚠️ 用 `scanf("%c", &ch)` 读字符有个大坑：**它连空格、回车都当字符读进来**。比如四则运算 `scanf("%lf%c%lf", &a, &op, &b)`，输入时操作数和运算符**之间不能加空格**，否则空格会被 `%c` 抢去当成 op。这是翁恺特别提醒的（例3-5）。

## 十、switch 语句：照号入座的多分支 ⭐⚠️ 重点

当你的分支是"**拿一个值去对号入座**"——比如输入 1 显示"一"，2 显示"二"，3 显示"三"——用 `else if` 写一长串能写，但 `switch` 更清爽。

类比：`switch` 就像快递柜，你报一个格子号，它直接打开对应那个格子。

```c
int n = 2;
switch (n) {
    case 1:
        printf("星期一\n");
        break;
    case 2:
        printf("星期二\n");
        break;
    case 3:
        printf("星期三\n");
        break;
    default:
        printf("输入有误\n");
        break;
}
```

逐块拆解：
- `switch (n)` —— 把要判断的值 `n` 放进括号。电脑算出 n 的值，拿去跟下面每个 `case` 比对。
- `case 1:` —— "如果 n 等于 1"。注意 `case` 后面跟**值**，再跟一个**冒号 `:`**（不是分号！）。
- `break;` —— **跳出整个 switch**，结束。这个超级重要，下面专门讲。
- `default:` —— "以上都不匹配时"走这里，相当于 else。可有可无，但建议写上兜底。

n 是 2 时：跳过 case 1，命中 case 2，打"星期二"，遇到 `break` 跳出。输出 `星期二`。

### ⚠️⚠️⚠️ switch 最大的坑：漏写 break 会"贯穿" ⭐ 考研年年考

这是 `switch` 的头号送命题，翁恺反复强调，叫 **fall-through（贯穿 / 击穿）**：

> 一旦某个 `case` 匹配上，电脑就从那里开始**一路往下执行**，**碰不到 `break` 就不停**，把后面 case 的代码也一并执行了，直到遇到 `break` 或 switch 结束。

看漏写 break 的后果：

```c
int n = 1;
switch (n) {
    case 1:
        printf("一\n");      // 命中这里
    case 2:
        printf("二\n");      // ❌ 没 break，继续往下执行！
    case 3:
        printf("三\n");      // ❌ 继续！
    default:
        printf("其他\n");    // ❌ 继续！
}
```

输出（吓人）：
```
一
二
三
其他
```

n 明明是 1，却把"一二三其他"全打出来了！因为 case 1 后面没 `break`，电脑命中后一路冲到底，**case 的值只决定"从哪开始"，不决定"在哪停"——停在哪由 break 决定。**

✅ 加上 break 才正常：

```c
int n = 1;
switch (n) {
    case 1: printf("一\n"); break;   // 打完"一"就 break 跳出
    case 2: printf("二\n"); break;
    case 3: printf("三\n"); break;
    default: printf("其他\n");
}
// 输出只有：一
```

> 💀 记死：**switch 里每个 case 后面几乎都要写 `break`，漏了就贯穿。** 考研选择题最爱给一段漏 break 的代码，让你算输出，专坑没记牢的人。

### ⭐ 故意不写 break：让多个 case 共用代码

贯穿不全是坏事，**故意省 break 能让几个 case 合并处理**（翁恺例3-10就用这招）。比如判断 1-5 是工作日、6-7 是周末：

```c
int day = 6;
switch (day) {
    case 1:
    case 2:
    case 3:
    case 4:
    case 5:
        printf("工作日\n");
        break;
    case 6:
    case 7:
        printf("周末\n");      // day 是 6，命中 case 6，往下贯穿到这句
        break;
}
// 输出：周末
```

case 1~5 后面都空着没 break，所以 1、2、3、4、5 任何一个命中，都会一路贯穿到 `printf("工作日")`。这是"多个 case 共享一段代码"的正经用法。

### ⚠️ switch 的几条硬规矩 ⭐

1. **`case` 后面只能跟整数或字符常量**（像 `case 1`、`case 'A'`），**不能跟变量、不能跟范围**。
   - ❌ `case x:`（变量不行）、`case 1...5:`（范围不行）、`case score > 60:`（表达式不行）。
   - 想判断范围（比如分数段），用 `else if`，别用 switch。
2. **switch 括号里的值，只能是整数或字符**（本质都是整数），**不能是小数 double**。
   - ❌ `switch(3.14)` 报错。
3. 各个 `case` 的值**不能重复**。
4. `default` 可以放在任意位置，但习惯放最后；没有匹配项又没 default 时，整个 switch 啥也不做。

> ⭐ 一句话对比：**能用 switch 的，else if 都能写；但 switch 只能比"等于某个具体值"，比大小、比范围只能用 else if。**

## 十一、三目运算符 `?:`：一行写完的迷你 if-else ⭐

有时候 if-else 只是为了"二选一给变量赋个值"，写四行嫌啰嗦。C 给了个简写——**条件表达式**，也叫**三目运算符**（因为它要三个部分）：

```c
表达式1 ? 表达式2 : 表达式3
```

读法：**"表达式1 成立吗？成立就取表达式2，否则取表达式3。"**

经典用法——求两个数的最大值（翁恺例子）：

```c
int a = 5, b = 8;
int max = (a > b) ? a : b;     // a>b 吗？不成立，所以 max 取 b，得 8
printf("%d\n", max);           // 输出 8
```

它完全等价于这段 if-else：

```c
int max;
if (a > b) {
    max = a;
} else {
    max = b;
}
```

但三目运算符**一行搞定**，所以爱被考"读代码算结果"。

再看一例：

```c
int score = 55;
printf("%s\n", score >= 60 ? "及格" : "挂科");   // 55>=60 不成立 → 取"挂科"
// 输出：挂科
```

⚠️ 为什么叫"三目"？因为它有**三个操作数**（条件、真时的值、假时的值），是 C 里唯一一个要三个部分的运算符。`?` 和 `:` 必须成对出现。

> 🎯 真题常考：给 `c = a > b ? a : b;` 让你说 c 的值，或者反过来让你把一段 if-else 改写成三目。记住读法"问号前是条件，问号后是真值，冒号后是假值"就不会错。

## 十二、嵌套 if 与"悬空 else"陷阱 ⭐⚠️ 难点

### 什么是嵌套 if？

`if` 里面还能再套 `if`，叫**嵌套**。比如先判断及格，及格里再分优良：

```c
if (score >= 60) {
    if (score >= 90) {
        printf("优秀\n");
    } else {
        printf("及格但不优秀\n");
    }
} else {
    printf("不及格\n");
}
```

这没什么难的，就是大括号一层套一层。只要每层都老实写大括号，逻辑很清晰。

### ⚠️⚠️ 悬空 else 问题（dangling else）⭐ 难点考点

麻烦出在**不写大括号**的时候。看这段：

```c
if (a > 0)
    if (b > 0)
        printf("都正\n");
else                          // ⚠️ 这个 else 到底是谁的？
    printf("???\n");
```

新手凭缩进以为：这个 `else` 配的是外层 `if (a > 0)`。
**真相**（翁恺原话）：**else 总是和它上面最近的、还没配对的那个 if 配对。**

所以这个 `else` 实际配的是**里层** `if (b > 0)`，不管你缩进成什么样。缩进骗人，规则不骗人。实际等价于：

```c
if (a > 0) {
    if (b > 0)
        printf("都正\n");
    else                      // 真正配的是 if(b>0)
        printf("???\n");
}
```

✅ 解决办法：**老老实实加大括号，把意图写死**。想让 else 配外层，就给外层 if 框上：

```c
if (a > 0) {
    if (b > 0)
        printf("都正\n");
} else {                      // 现在这个 else 明确配 if(a>0)
    printf("???\n");
}
```

> ⭐ 翁恺建议：**内嵌 if 时总是用 `{}` 明确范围**，从根上避免悬空 else 的歧义。考研选择题爱用没大括号的嵌套 if 考你"else 配哪个 if"，答案永远是"最近的、未配对的那个 if"。

### else if 和嵌套 if-else 哪个好？

翁恺给了结论：**`else if` 逻辑更清晰，是处理多分支的首选；嵌套 if-else 容易因配对问题出错，要谨慎用。** 能用 else if 阶梯就别套一堆 if。

## 十三、浮点数不能用 `==` 判等！⭐⚠️ 高频坑

这是个特别隐蔽、特别爱考的坑：**小数（float/double）之间，不要用 `==` 判断相等。**

为什么？因为小数在电脑里是**近似存储**的，存进去就有微小误差。你以为是 0.3 的东西，电脑里可能存成 0.30000000000004。

```c
double a = 0.1 + 0.2;
if (a == 0.3) {
    printf("相等\n");
} else {
    printf("居然不相等！\n");   // ❌ 实际打印这句
}
```

`0.1 + 0.2` 在数学上是 0.3，但电脑算出来是 0.30000000000000004，跟 0.3 不完全相等，于是 `==` 判定为**假**。新手一脸懵。

✅ 正确做法：**判断两个小数的差，小到可以忽略，就当它们相等。** 用绝对值函数 `fabs`（在 `math.h` 里）：

```c
#include <math.h>
double a = 0.1 + 0.2;
if (fabs(a - 0.3) < 1e-6) {        // 两者之差小于 0.000001 就算相等
    printf("相等\n");              // ✅ 这次正常打印
}
```

`1e-6` 就是 0.000001（一个很小的数，叫"误差容忍度"）。`fabs(a - 0.3)` 求 a 和 0.3 之差的绝对值，差足够小就认为相等。

> ⚠️ 记死：**整数判等用 `==` 没问题；小数判等绝不能用 `==`，要用"差的绝对值 < 一个极小数"。** 这是考研常设的陷阱。

## 十四、新手最常犯的错，照着排查 ⚠️

分支结构这一章，新手翻车集中在这几处，一个个对照检查：

1. **`==` 写成 `=`** —— `if (x = 5)` 恒为真还偷改 x。这是头号杀手，看到 if 里只有一个等号先怀疑它。
2. **漏写 break** —— switch 里某个 case 忘了 break，结果"贯穿"把后面 case 全执行了。
3. **`if`/`else` 漏大括号** —— 想管多句却没框起来，结果只管了第一句，后面的"逃出"了 if 的管辖。
4. **else if 顺序写反** —— 把宽松条件（如 `>=60`）写在前面，把后面严格的条件全截胡。
5. **区间写成数学式** —— `if (0 <= x <= 100)`，C 里必须写 `x >= 0 && x <= 100`。
6. **小数用 `==` 判等** —— `if (a == 0.3)`，要用 `fabs(a-0.3) < 1e-6`。
7. **case 后面用了冒号外的符号** —— `case 1;`（写成分号）、`case 1`（漏冒号），都报错；正确是 `case 1:`。
8. **`&&` `||` 写成单个 `&` `|`** —— 单个是位运算，意思全变。逻辑判断要双写。
9. **else 找不到对应 if** —— 光写 else 没有配套的 if，或嵌套时 else 配错了 if（悬空 else）。
10. **条件括号里用了中文标点** —— `if（x>0）` 用了全角括号，报错。切英文输入法。

### 万能口诀
> **if 里看见单等号先怀疑、switch 数 break 够不够、多句必加大括号、小数判等用 fabs。**

## 十五、第3章理论速记（专攻选择题）📚⭐

> 这些是不用写代码、但选择题爱考的纯结论，背下来就是送分题。

### ① 分支是三种基本结构之一
程序三种基本结构：**顺序、分支（选择）、循环**。这一章学的是分支。流程图里**菱形框 = 判断 / 分支**。

### ② 真假的本质
**C 用非 0 表示真，0 表示假。** 关系/逻辑运算的结果，真是 1、假是 0。

### ③ 逻辑运算符优先级（必背）
`!` > 算术运算符 > 关系运算符 > `&&` > `||` > 赋值 `=`。
口诀：非最高、与高于或、赋值最低。

### ④ 短路求值
`&&` 左边假就不算右边；`||` 左边真就不算右边。常考"右边表达式有没有执行""变量值变没变"。

### ⑤ switch 的铁律
`case` 后跟整型/字符常量（不能是变量、范围、小数）；漏 `break` 会贯穿；`switch()` 里不能放 double。

### ⑥ else 配对规则
**else 与上面最近的、尚未配对的 if 配对**，跟缩进无关（悬空 else）。

### ⑦ 字符与 ASCII
字符底层是整数（ASCII 码）；`'0'~'9'`、`'A'~'Z'`、`'a'~'z'` 各自编码连续；`'A'`=65，`'a'`=97，`'0'`=48。

### ⑧ 多分支三种实现
else-if 语句、嵌套 if-else、switch 语句。else-if 是首选（清晰），嵌套 if-else 易错（要谨慎）。

## 十六、本节考点速记 ⭐

1. `if (条件){...}` 单分支：真就做，假就跳过；`if-else` 双分支：二者必执其一。
2. `if`/`else` 后**最好永远写大括号**；不写时只管紧跟的一条语句（漏大括号是经典坑）。
3. 关系运算符 6 个：`> < >= <= == !=`；`>=`/`<=` 不能写反成 `=>`/`=<`。
4. 💀 **`=` 是赋值，`==` 才是判等**；`if (x=5)` 恒为真且偷改 x，`if (x=0)` 恒为假。改错题必考。
5. **C 里 0 为假、非 0 为真**；关系/逻辑结果真是 1、假是 0。
6. `else if` 阶梯：从上往下，命中一个就跑路；**顺序要由严到松**，写反会被截胡。
7. 一堆独立 `if` 会各判一遍（可能多命中）；想"多选一"必须用 `else if`。
8. 逻辑运算符 `&&`（与，都真才真）、`||`（或，一真就真）、`!`（非，取反）；双写，别漏成单个。
9. **区间不能写 `0<=x<=100`**，要 `x>=0 && x<=100`。
10. ⭐ **短路求值**：`&&` 左假不算右、`||` 左真不算右；常考"哪句没执行"，也能防除 0。
11. 优先级：`!` > 算术 > 关系 > `&&` > `||` > `=`；拿不准就**加括号**。
12. 字符底层是 ASCII 整数，`'0'~'9'`/`'A'~'Z'`/`'a'~'z'` 编码连续；`%c` 看字符、`%d` 看编码。
13. ⭐ **switch 漏 `break` 会贯穿**；`case` 只能跟整型/字符常量；故意省 break 可让多 case 共用代码。
14. 三目运算符 `条件 ? 真值 : 假值`，等价于迷你 if-else；`max=(a>b)?a:b`。
15. ⭐ 悬空 else：**else 配最近的、未配对的 if**，与缩进无关；嵌套时务必加大括号。
16. ⭐ **小数判等不能用 `==`**，要 `fabs(a-b) < 1e-6`（需 `#include <math.h>`）。
