## 〇、开始之前：用"档案表"理解结构体

前面学的数组（M6）有个硬规矩：**一个数组里只能装同一种类型的东西**。比如 `int scores[5]` 只能装整数，不能一个格子装整数、另一个格子装名字。

可现实里的东西往往是"混搭"的。想想学校里一张**学生登记表**：

```
姓名：张三       ← 文字
年龄：20        ← 整数
分数：95.5      ← 小数
```

这张表上有三个**不同类型**的信息（文字、整数、小数），但它们说的是**同一个人**。数组装不了这种混搭，于是 C 给了我们一个新工具——**结构体（struct）**。

一句话理解：**结构体就是一张自定义的"登记表 / 档案卡"，把几个相关、但类型不同的数据打包成一个整体，当成一样东西来用。**

- 登记表的"表头"（姓名、年龄、分数这些栏目）= 结构体的**成员**。
- 填好的一张具体的表（张三，20，95.5）= 一个结构体**变量**。

学完本模块你要掌握：怎么"设计这张表"（定义类型）、怎么"填一张表"（声明变量并初始化）、怎么"读写表里某一栏"（访问成员）、以及考研最爱考的两个符号 `.` 和 `->` 的区别。

## 一、为什么要结构体？（先想清楚它解决什么问题）⭐

假设不用结构体，要存一个学生的三项信息，你只能拆成三个孤零零的变量：

```c
char name[20] = "张三";
int  age      = 20;
float score   = 95.5;
```

存一个学生还行。要是存 **3 个学生**呢？你得开 9 个变量：`name1/age1/score1`、`name2/age2/score2`……乱成一锅粥，而且谁和谁是一伙的全靠你脑子记。

结构体的价值就在这：**把"属于同一个人"的几样数据捆成一包**，名字、年龄、分数从此是一个整体，搬来搬去、传给函数都是一整包，不会散。

> 📌 一句话考点：**数组装的是"同类型的一堆"，结构体装的是"不同类型的一组"。** 这是选择题最爱拿来对比的一句话。

## 二、定义结构体类型：先设计"表头" ⭐⚠️ 重点

用结构体分两步走，**第一步是"设计这张表长什么样"**，也就是定义一个结构体**类型**：

```c
struct Student {
    char  name[20];
    int   age;
    float score;
};        // ⚠️⚠️ 这个分号绝对不能漏！
```

逐字拆解，一个都不跳：

- `struct` —— 关键字，读作"司track克特"，意思是"我要定义一个结构体"。固定写法，照抄。
- `Student` —— **结构体的名字**（你自己起的，习惯首字母大写，见名知意）。`struct Student` 合起来才是这个新类型的完整名字，就像 `int`、`double` 那样，以后是个"类型"。
- `{ }` —— 大括号里列出**这张表有哪些栏目（成员）**。
- `char name[20];` —— 第一个成员：一个能装 20 个字符的字符数组，用来存名字。
- `int age;` —— 第二个成员：整数，存年龄。
- `float score;` —— 第三个成员：小数，存分数。
- 每个成员后面有自己的分号 `;`，和普通变量声明一样。
- 最后大括号外面**还有一个分号** `};` —— ⚠️ 这是新手头号大坑，下面单独强调。

### ⚠️⚠️ 头号大坑：结构体定义结尾那个分号

```c
struct Student {
    char name[20];
    int  age;
}          // ❌ 漏了分号！编译器会报一堆莫名其妙的错
```

```c
struct Student {
    char name[20];
    int  age;
};         // ✅ 大括号后面必须有分号
```

为什么这里要分号，而函数 `int main(){ }` 后面就不用？因为定义结构体在语法上算**一条声明语句**，语句就得用分号结尾。记不住原理没关系，**死记：`struct ... { ... };` 结尾必有分号。** 考研改错题极爱在这埋雷。

### 重点提醒：定义类型 ≠ 造出变量

上面这段 `struct Student { ... };` **只是画了一张空白表格的模板**，电脑里此刻一个学生数据都没有，也没占多少内存。

类比：你设计了"学生登记表"的格式打印出来一沓空表，但还没人填。要真正存数据，得进入第二步——**声明变量（填一张具体的表）**。

## 三、声明变量并初始化：填一张具体的表 ⭐⚠️ 重点

定义好类型后，就能像用 `int`、`double` 那样，用 `struct Student` 造变量了。

```c
struct Student s1;     // 造一张空表，叫 s1（成员里现在是垃圾值）
```

注意：类型名是 `struct Student` 两个词连写，**`struct` 不能省**（除非用了 typedef，见第十二节）。

### 造的同时填好数据：初始化

跟普通变量一样，可以在造的时候就用 `{ }` 一次性填进去：

```c
struct Student s1 = {"张三", 20, 95.5};
```

拆解：大括号里的值**按成员定义的顺序**一一对应填进去：
- `"张三"` → 填给第一个成员 `name`
- `20` → 填给第二个成员 `age`
- `95.5` → 填给第三个成员 `score`

⚠️ 顺序绝对不能乱！第二个位置写的就是 `age`，你要是写成 `{"张三", 95.5, 20}` 意思就全错了。

### 没填满会怎样？

```c
struct Student s2 = {"张三", 18};   // 只填了前两个，score 没填
```

没填到的成员（这里的 `score`）会被**自动补 0**。所以 `s2.score` 此时是 0.0。这点和数组初始化"补 0"的规则一样。

### ⭐ 翁恺补充：定义类型的三种写法（了解，选择题可能考）

翁恺笔记里列了三种定义结构体变量的方式，第一次学**只要会用第 1 种**就够，后两种见到能认出来即可：

```c
// 写法1：先定义类型，再单独定义变量（最常用、最清晰，推荐）
struct Student { char name[20]; int age; };
struct Student s1, s2;
```

```c
// 写法2：定义类型的同时顺便定义变量（大括号后面直接跟变量名）
struct Student {
    char name[20];
    int  age;
} s1, s2;
```

```c
// 写法3：连结构体名都省了（匿名结构体），之后没法再定义同类型变量了
struct {
    char name[20];
    int  age;
} s1, s2;
```

⚠️ 写法 3 因为没名字，以后想再造个同款变量都没法写 `struct xxx`，所以实际很少用。**考试认得这是合法写法就行。**

## 四、点运算符 `.`：读写表里的某一栏 ⭐⚠️ 重点

表填好了，怎么单独看/改其中一栏（一个成员）？用**点运算符 `.`**：

格式：**`变量名.成员名`**，读作"某变量的某成员"。

```c
struct Student s1 = {"张三", 20, 95.5};

printf("%s\n", s1.name);    // 读 name，输出：张三
printf("%d\n", s1.age);     // 读 age， 输出：20
printf("%.1f\n", s1.score); // 读 score，输出：95.5

s1.age = 21;                // 改 age：把这一栏改成 21
s1.score = 88.0;            // 改 score
```

把 `s1.age` 整个看成一个"普通的 int 变量"就行——能打印、能赋值、能参与运算，和你之前学的变量没区别。`s1.score` 就当成一个 double/float 来用。

> 📌 占位符还是按成员的类型配：`name` 是字符串用 `%s`，`age` 是整数用 `%d`，`score` 是小数用 `%f`。（占位符配对见 M1）

### ⚠️ 不能整张表一把打印

```c
printf("%d\n", s1);   // ❌ 错！不能直接打印整个结构体变量
```

`printf` 不认识"一整张表"该怎么显示，**必须一个成员一个成员地打印**，每个成员配自己的占位符：

```c
printf("姓名:%s 年龄:%d 分数:%.1f\n", s1.name, s1.age, s1.score);  // ✅
```

## 五、字符串成员要用 strcpy，不能用 `=` ⭐⚠️ 高频坑

数字成员直接赋值没问题（`s1.age = 21;`），但**字符串成员（char 数组）不能用 `=` 重新赋值**：

```c
struct Student s1;
s1.age = 20;                 // ✅ 数字成员，直接 = 没问题
s1.name = "张三";             // ❌ 错！name 是 char 数组，不能这样赋值
```

为什么？`name` 是个 `char[20]` 数组，数组名不能整个被 `=` 赋值（这条规则 M6 字符串讲过）。要给字符串成员赋值，得用 `string.h` 里的 **`strcpy`**（字符串拷贝）：

```c
#include <string.h>          // ⚠️ 用 strcpy 要包含这个头文件

struct Student s1;
strcpy(s1.name, "张三");      // ✅ 把 "张三" 拷进 name 里
s1.age = 20;
```

⚠️ 区别记牢：
- **初始化那一刻**可以用大括号带字符串：`struct Student s1 = {"张三", 20, 95.5};` ✅（这是初始化，不是赋值）
- **造完之后再改名字**，必须 `strcpy(s1.name, "新名字");` ✅，不能 `s1.name = "新名字";` ❌

## 六、结构体整体赋值：一张表直接拷给另一张 ⭐⚠️ 高频考点

这是结构体一个很爽、考试也爱考的特性：**同类型的两个结构体变量，可以用 `=` 一次性整体拷贝**，所有成员一起复制过去：

```c
struct Student a = {"张三", 20, 95.5};
struct Student b;

b = a;     // ✅ 把 a 的每一个成员，整体复制给 b
```

执行完 `b = a;` 后，`b.name` 是 "张三"、`b.age` 是 20、`b.score` 是 95.5，相当于一口气写了 strcpy + 两次赋值。

> 翁恺例 9-1 里求最高分学生，就是用 `max = s1;` 这种整体赋值，把当前学生的整张表存进 max。

⚠️ 两个注意点：
1. **只能在同类型之间整体赋值**（都是 `struct Student` 才行）。
2. 数组不能这样！`int x[5]; int y[5]; y = x;` 是错的。但结构体可以整体 `=`——**这正是结构体和数组的一个重要区别，选择题考过。**

## 七、⚠️ 结构体不能用 `==` 比较！（易错点）

既然能用 `=` 整体赋值，新手很自然就会想：那能不能用 `==` 判断两张表是不是一样？

```c
if (a == b) { ... }     // ❌ 编译直接报错！结构体不能用 == 比较
```

**不行。** C 语言没规定整个结构体怎么比较，所以 `==`、`!=` 都不能直接用在结构体上。

想判断两个学生是否信息相同，只能**一个成员一个成员地比**：

```c
if (a.age == b.age && a.score == b.score && strcmp(a.name, b.name) == 0) {
    printf("两个学生信息相同\n");
}
```

（字符串成员的比较要用 `strcmp`，不能用 `==`，这点 M6 讲过。）

> 📌 记死这组对比：结构体**能用 `=` 整体赋值，但不能用 `==` 整体比较**。一个能、一个不能，特别爱拿出来坑人。

## 八、结构体指针：箭头运算符 `->` ⭐⚠️⚠️ 全章最高频考点

这是考研选择题、改错题**出现频率最高**的一个点，务必搞透。

先回忆 M7：指针就是"存地址的变量"。结构体也能有指针——一个指向结构体变量的指针：

```c
struct Student s1 = {"张三", 20, 95.5};
struct Student *p = &s1;     // p 是结构体指针，存着 s1 的地址（指向 s1）
```

- `struct Student *p` —— 造一个指针 `p`，它专门指向 `struct Student` 类型的变量。
- `&s1` —— 取 s1 的地址。
- 连起来：`p` 现在"指着" s1。

### 通过指针访问成员有两种写法

**写法一：`(*p).成员名`**（先解引用，再取成员）

```c
printf("%d\n", (*p).age);    // 先 *p 拿到 s1 本体，再 .age 取成员
```

⚠️ 这里 `*p` 外面的括号**绝对不能省**！因为 `.` 的优先级比 `*` 高，写成 `*p.age` 电脑会先算 `p.age` 再 `*`，意思全错且报错。要靠括号强制"先 `*p`"。

**写法二：`p->成员名`**（箭头运算符，最常用）⭐

```c
printf("%d\n", p->age);      // 和 (*p).age 完全等价，但好写好读
p->age = 21;                 // 通过指针改成员
```

`->` 是一个整体符号（一个减号 `-` 加一个大于号 `>`），专门用来"**通过指针访问它指向的结构体的成员**"。

### ⭐⭐ 核心结论：什么时候用 `.`，什么时候用 `->`

当 `p = &s1` 时，下面三个写法**完全等价**：

```c
s1.age        // 变量本体，用 .
(*p).age      // 指针先解引用再用 .
p->age        // 指针直接用 ->（首选）
```

**一句话口诀，背下来**：
> **变量用点 `.`，指针用箭头 `->`。**

⚠️ 改错题最爱这么坑你：给你一个**指针** `p`，却写成 `p.age`（指针用了点）——错！应该 `p->age`。或者给你一个**变量** `s1`，却写成 `s1->age`（变量用了箭头）——也错！应该 `s1.age`。看到 `.` 和 `->` 先问自己：**这玩意儿是变量还是指针？**

## 九、结构体数组：一张完整的"学生信息表" ⭐⚠️ 大题常客

一个 `struct Student` 变量存一个学生。要存一整个班，就把它们排成**结构体数组**——每个格子都是一张填好的表：

```c
struct Student class[3] = {
    {"张三", 20, 85.0},
    {"李四", 21, 92.0},
    {"王五", 19, 78.0}
};
```

拆解：
- `struct Student class[3]` —— 造一个能装 3 个 `struct Student` 的数组。
- 初始化时**用嵌套大括号**：外层大括号框住整个数组，里层每对大括号是一个学生。

### 怎么访问？先下标，再点成员

```c
printf("%s\n", class[1].name);    // 第2个学生的名字 → 李四
class[0].score = 90.0;            // 改第1个学生的分数
```

读法：`class[1]` 先拿到第 2 张表（下标从 0 数），再 `.name` 取这张表的名字栏。**`class[1].name` = "第 2 个学生的 name 成员"。**

### 配合循环遍历（大题最常见套路）

求全班最高分、求平均分、按分数排序，都是先用 `for` 把整个数组走一遍：

```c
int i;
float maxScore = class[0].score;
int maxIndex = 0;
for (i = 1; i < 3; i++) {
    if (class[i].score > maxScore) {  // 比较每个学生的 score
        maxScore = class[i].score;
        maxIndex = i;
    }
}
printf("最高分是 %s，%.1f 分\n", class[maxIndex].name, maxScore);
```

> 翁恺例 9-2 的"成绩排序"就是对结构体数组做**选择排序**，比较和交换的依据是 `class[j].score`，交换时用前面学的**整体赋值**（`temp = class[i]; class[i] = class[j]; class[j] = temp;`）——一次换一整张表，很方便。

## 十、结构体作函数参数：传值 vs 传指针 ⭐⚠️ 重点

把结构体交给函数处理，有两种传法，**区别是考点**。

### 传法一：传值（传整个结构体）

```c
void print(struct Student s) {     // 形参 s 是实参的一份完整复制
    printf("%s %d\n", s.name, s.age);
}
```

- 调用 `print(s1);` 时，会把 s1 **整张表完整复制一份**给形参 s。
- 优点：写法简单，函数里改 s 不会影响外面的 s1（安全）。
- ⚠️ 缺点：结构体大的话，**复制开销大、慢**；而且函数里的修改传不回去。

### 传法二：传指针（传地址，⭐ 更常用）

```c
void birthday(struct Student *p) {   // 形参是指针，指向外面的真表
    p->age = p->age + 1;             // 用 -> 改，直接改到外面那张表
}
```

调用时传地址：

```c
birthday(&s1);     // 传 s1 的地址
```

- 只传一个地址（4 或 8 字节），**不复制整张表，效率高**。
- 函数里通过 `p->` 修改，**改的是外面真正的 s1**（能把结果带回去）。
- 这正是翁恺例 9-3"修改学生成绩"用的办法：函数拿到数组地址，用指针 `p->` 直接改 main 里的数据。

> 📌 考点对比表：
>
> | | 传值（struct s） | 传指针（struct *p） |
> |---|---|---|
> | 传的东西 | 整张表的复制 | 一个地址 |
> | 开销 | 大（复制全部成员） | 小（只一个地址） |
> | 能否改到外面 | 不能 | 能 |
> | 访问成员 | `s.成员` | `p->成员` |
>
> 结论：**要修改实参、或结构体较大 → 用指针；否则传值也行。** 数组名作参数本身就是地址，所以结构体数组传给函数天然就是"传地址"。

## 十一、嵌套结构体：表里套表（了解）⭐

结构体的成员，本身也可以是**另一个结构体**。比如学生的"住址"信息又能拆成省、市、邮编，就可以单独做一张"地址表"，再塞进学生表里：

```c
struct Address {           // ⚠️ 先定义被嵌套的结构体
    char city[20];
    int  zip;              // 邮编
};

struct Student {           // 再定义包含它的结构体
    char name[20];
    int  age;
    struct Address addr;   // 成员 addr 本身是一个 Address 结构体
};
```

⚠️ 顺序要求：**被嵌套的（Address）必须先定义，再定义用它的（Student）**，否则电脑不认识 `struct Address`。

### 访问里层成员：点一级一级往里点

```c
struct Student s1;
strcpy(s1.addr.city, "杭州");   // s1 的 addr，addr 的 city，从外往里逐级 .
s1.addr.zip = 310000;
```

读法：`s1.addr.city` = "s1 的地址栏里的城市栏"。**从外到内一级一级用 `.`**，几层就点几下。考试见到 `a.b.c` 这种连点，就是嵌套结构体在逐级访问。

## 十二、typedef 起别名：把 `struct Student` 简写成 `Student` ⭐

每次都写 `struct Student` 有点啰嗦。用 **`typedef`** 给类型起个别名，以后就能省掉 `struct`：

```c
typedef struct Student Student;   // 给 "struct Student" 起别名叫 "Student"
Student s;                        // 之后直接写 Student，不用带 struct 了
```

更常见的是定义类型时**一步到位**起别名：

```c
typedef struct {
    char name[20];
    int  age;
} Student;          // 这里的 Student 是别名

Student s1 = {"张三", 20};   // 直接用 Student，干净利落
```

- `typedef` 的作用就是"**给一个已有类型取个新名字**"，不创造新类型，只是换个叫法。
- 好处：代码短、好读。很多课本和真题里直接用 `Student s;` 而不写 `struct`，就是因为前面 typedef 过。
- ⚠️ 别和定义结构体搞混：`typedef` 是**起别名**，`struct {...}` 才是**定义内容**。

## 十二点五、⭐⭐ 考研要求的格式：王道/真题里的结构体长这样（必看）

> 前面讲的是 C 语言"原汁原味"的写法。但你将来刷**王道、真题、408 大题**时，看到的结构体几乎**全是另一副打扮**。这一节专门带你认这套"考研方言"——不是新语法，就是把上面学的 `typedef` + 结构体按**考研约定俗成的格式**写出来。**先认脸，等你学数据结构时天天见。**

### ① 考研标准模板：`typedef struct { ... } 类型名;`

408 的数据结构代码，定义一张表（顺序表、链表、栈……）**基本是固定套路**：

```c
typedef struct {
    int data[100];   // 存数据的数组
    int length;      // 当前表长
} SqList;            // ← 这个 SqList 就是类型名（别名）
```

对照你刚学的第十二节，这其实就是"**定义结构体的同时用 typedef 起别名**"，一模一样的东西，只是：
- 考研里**几乎不写** `struct Xxx x;` 这种带 `struct` 的老式写法。
- 一律 `typedef struct {...} 类型名;`，之后直接 `SqList L;` 声明变量，**干净、省一个 `struct`**。
- 类型名爱用全大写或首字母大写的缩写：`SqList`（顺序表）、`LinkList`（链表）、`SqStack`（顺序栈）、`BiTree`（二叉树）……见名知意。

⚠️ 所以你在王道书上看到 `SqList L;` 别懵——它就是"造一张顺序表的空表 L"，等同于你学的 `struct Student s1;`。

### ② `ElemType`：一个"占位类型"，代表"元素的类型" ⭐⚠️ 考研必懂

王道代码里满屏都是一个你课本上没有的类型——`ElemType`：

```c
typedef struct {
    ElemType data[100];   // ← 这里不写 int，写 ElemType
    int length;
} SqList;
```

`ElemType` 不是 C 自带的类型，而是**作者自己用 typedef 起的"占位别名"**，意思是"**这里的元素到底是什么类型，先不写死，留个空位**"。它通常在代码开头这样定义：

```c
typedef int ElemType;   // 本题里，元素就是 int
```

为什么要这么绕？因为数据结构讲的是**通用算法**——同一套顺序表代码，元素可以是 int、可以是 double、可以是 struct Student。用 `ElemType` 占位，**想换元素类型时只改开头一行** `typedef int ElemType;` → `typedef double ElemType;`，下面所有代码一字不动。

> 📌 一句话记住：**`ElemType` = "元素的类型"的占位符，默认你就当它是 `int`。** 真题大题里看到 `ElemType e;`，脑子里替换成 `int e;` 来读就行。

### ③ 完整对照：同一张"顺序表"，课本写法 vs 考研写法

```c
// 【你课本学的写法】带 struct，元素写死成 int
struct SqList {
    int data[100];
    int length;
};
struct SqList L;          // 声明要带 struct

// 【考研/王道写法】typedef 起别名 + ElemType 占位（标准格式）
typedef int ElemType;     // 先把元素类型定为 int
typedef struct {
    ElemType data[100];
    int length;
} SqList;
SqList L;                 // 声明不用带 struct，干净
```

两段**功能完全一样**，但考研一律用下面那种。**你以后自己写大题，也建议照考研格式来，阅卷老师看着顺眼。**

### ④ 链表节点的考研标准写法（提前认脸，M10 详讲）⭐

链表节点是这套格式最经典的应用，提前给你看一眼，M10 学链表时就不陌生了：

```c
typedef struct LNode {      // ⚠️ 这里结构体保留了名字 LNode
    ElemType data;          // 数据域：存元素
    struct LNode *next;     // 指针域：指向下一个节点（必须用 struct LNode）
} LNode, *LinkList;         // ← 一次起俩别名！
```

最后一行 `} LNode, *LinkList;` 一口气起了**两个**别名，这是考研里极其常见的写法：
- `LNode` —— 代表"一个节点"这个类型。`LNode x;` = 一个节点。
- `*LinkList` —— 代表"指向节点的指针"类型。`LinkList head;` = 一个指向节点的指针（通常当头指针用）。

⚠️ 两个关键细节，现在记个脸熟：
1. 成员里那个 `struct LNode *next;` **必须带 `struct`、必须用 `LNode` 这个名字**——因为此刻 typedef 的别名还没生效（正在定义中），只能用原始的 `struct LNode` 来自我引用。这是新手抄代码最容易抄错的一行。
2. `LNode` 和 `LinkList` **本质是同一种东西的两种叫法**：`LinkList` 等价于 `LNode *`。王道用 `LinkList` 强调"这是条链表/头指针"，用 `LNode *` 强调"这是个节点指针"，**纯粹是为了代码可读性**，编译器眼里完全一样。

> 现在你只要能"看懂这是在定义链表节点"就够了，链表怎么用 M10 会手把手讲。

## 十三、sizeof 与内存：一张表占多大地方（了解）⭐

结构体变量的各个成员，在内存里**按定义顺序一个挨一个连续存放**。整张表占多少字节，可以用 `sizeof` 量：

```c
struct Student {
    char name[20];   // 20 字节
    int  age;        // 4 字节
    float score;     // 4 字节
};
printf("%lu\n", sizeof(struct Student));   // 量整个类型的大小
printf("%lu\n", sizeof(s1));               // 量某个变量也行
```

- 直觉上是 20+4+4 = 28 字节。⚠️ 但实际结果可能比 28 **大一点**，因为编译器会做"**内存对齐**"，在成员之间偷偷补几个空字节让地址更整齐。
- 这点**了解即可**：考研一般不让你手算对齐后的精确字节数，但要知道"**结构体大小 ≥ 各成员大小之和，可能因对齐而变大**"这个结论。

## 十四、新手最常犯的错，照着排查 ⚠️

1. **定义结尾漏分号** —— `struct Student { ... }` 后面忘了 `};`。报错往往很怪、指向下一行。看到结构体定义先数有没有那个分号。
2. **变量用了箭头、指针用了点** —— 变量 `s1` 写成 `s1->age`（错），指针 `p` 写成 `p.age`（错）。口诀：**变量用 `.`，指针用 `->`**。
3. **字符串成员用 `=` 赋值** —— `s1.name = "张三";` ❌。改名字要 `strcpy(s1.name, "张三");`，记得 `#include <string.h>`。
4. **用 `==` 比较两个结构体** —— `if(a==b)` ❌ 编译报错。要逐个成员比，字符串成员用 `strcmp`。
5. **`(*p).age` 漏了括号** —— 写成 `*p.age` ❌。`.` 比 `*` 优先级高，必须括号。（不过实际直接用 `p->age` 最省事。）
6. **占位符配错成员类型** —— `name` 用 `%s`、`age` 用 `%d`、`score` 用 `%f`，配错出乱码。
7. **想一把打印整个结构体** —— `printf("%d", s1)` ❌。必须一个成员一个成员地打。
8. **声明变量忘了写 `struct`** —— `Student s1;` 直接写（没 typedef 时）❌，要 `struct Student s1;`。除非前面 typedef 过别名。

## 十五、本节考点速记 ⭐

1. 结构体 = 把**不同类型**的相关数据打包；数组是**同类型**的一堆——这是两者最大区别。
2. 定义类型：`struct 名 { 成员; ... };`，**结尾分号绝不能漏**。
3. 定义类型只是"画表格模板"，声明变量 `struct Student s1;` 才真正占内存。
4. 初始化按成员顺序填：`struct Student s1 = {"张三", 20, 95.5};`，没填满的成员自动补 0。
5. 访问成员用点：`s1.age`；可打印、可赋值，当普通变量用。不能一把打印整个结构体。
6. **字符串成员赋值用 `strcpy`**，不能用 `=`（需 `#include <string.h>`）。
7. **结构体能用 `=` 整体赋值**（`b = a;` 拷全部成员），但数组不行。
8. **结构体不能用 `==` 比较**，要逐成员比。（能赋值不能比较，高频坑）
9. ⭐⭐ **变量用 `.`，指针用 `->`**；`p->age`、`(*p).age`、`s1.age`（当 `p=&s1`）三者等价。
10. 结构体数组：`class[i].成员`，先下标再点成员，配 `for` 遍历是大题套路；交换整张表用整体赋值。
11. 作函数参数：**传值**复制整张表（慢、改不到外面）；**传指针**只传地址（快、能改外面，用 `p->`）。
12. 嵌套结构体：被嵌套的先定义；访问里层 `s1.addr.city` 逐级点。
13. `typedef struct {...} Student;` 起别名，之后可省掉 `struct` 直接 `Student s;`。
14. `sizeof(struct X)` 量大小；因**内存对齐**，结果 ≥ 各成员之和。
