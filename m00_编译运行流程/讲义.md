# M0 编译运行流程（第1章 §1.2）

> 本模块只有一个目标：彻底搞懂"从写代码到程序跑起来"这条流水线，并把开发环境装好。

---

## 0. 先装好环境：让电脑能编译 C ⭐⚠️ 第一步

学 C 之前，电脑里得有一个**编译器**（把你的代码翻译成程序的工具）。本课统一用 **gcc**（最通用、和考研课本一致）。下面按你的系统选一种装法。

### Windows 用户（推荐 MinGW-w64）

Windows 默认没有 gcc，要自己装。最省心的办法是装 **MSYS2**：

1. 打开官网 `https://www.msys2.org/`，下载安装包，一路下一步装好。
2. 打开"MSYS2 UCRT64"终端，执行下面这行装 gcc：
   ```bash
   pacman -S mingw-w64-ucrt-x86_64-gcc
   ```
3. 把编译器目录加进系统 PATH（这样在任何终端都能用 gcc）：
   - 默认目录一般是 `C:\msys64\ucrt64\bin`
   - 搜索"编辑系统环境变量" → 环境变量 → 在 `Path` 里新增上面这个目录。
4. 重新打开一个 **PowerShell** 或 **CMD**，输入 `gcc --version`，能看到版本号就成功了。

> 💡 也可以用更轻量的 [w64devkit](https://github.com/skeeto/w64devkit/releases)：下载解压，把里面的 `bin` 目录加进 PATH 即可，无需安装。

### macOS 用户

Mac 自带的命令行工具里就有编译器。打开"终端"，执行：
```bash
xcode-select --install
```
弹窗点"安装"，装完输入 `gcc --version` 验证。

> 说明：Mac 上的 `gcc` 实际指向系统自带的 clang，用法完全一样，本课所有命令照用不误。

### Linux 用户（Ubuntu/Debian）

```bash
sudo apt update
sudo apt install build-essential
```
装完 `gcc --version` 验证。

### 编辑器用什么？

写代码用任何纯文本编辑器都行，推荐 **VS Code**（免费、跨平台）。装好后建议再装个"C/C++"插件，有语法高亮和提示。
⚠️ 不要用 Word 写代码，它会塞进看不见的格式字符导致编译失败。

## 1. 一个 C 程序的固定骨架 ⭐

```c
#include <stdio.h>      // 引入标准输入输出库，才能用 printf

int main(void)          // 程序入口，运行时从这里开始
{
    printf("...\n");    // 打印，\n 是换行
    return 0;           // 0 表示正常结束
}
```

记牢三件事：`#include` 在最上面、`main` 是入口、`return 0` 收尾。

## 2. 逐行看懂 demo.c

| 行 | 代码 | 作用 |
|----|------|------|
| `#include <stdio.h>` | 引入库 | 没有它就不能用 printf |
| `int main(void)` | 入口函数 | 程序从这里跑；`void` 表示不收参数 |
| `{ }` | 大括号 | 框住函数体 |
| `printf("xx\n")` | 输出 | `\n` 换行 |
| `int year = 2026;` | 声明变量 | ⚠️ 行尾分号 |
| `printf("%d", year)` | 占位输出 | `%d` 被 year 的值替换 |
| `return 0;` | 返回 | 告诉系统正常结束 |

## 3. 编译运行三步走 ⭐⚠️ 核心

在模块文件夹里，终端执行：

```bash
cd m00_编译运行流程   # ① 进文件夹
gcc demo.c -o demo    # ② 编译，生成可执行文件
./demo                # ③ 运行（Windows 见下方说明）
```

命令拆解：

| 部分 | 含义 |
|------|------|
| `gcc` | 编译器（本课统一用它，和考研课本一致） |
| `demo.c` | 输入：你的源代码 |
| `-o demo` | output，指定输出文件名叫 demo |
| `./demo` | 运行当前目录下的 demo（`./` = 当前文件夹） |

### ⚠️ Windows 运行命令不一样

Windows 上编译出来的是 `.exe` 文件，运行方式也略有不同：

| 平台 | 编译 | 运行 |
|------|------|------|
| **Windows** (PowerShell) | `gcc demo.c -o demo.exe` | `.\demo.exe` |
| **Windows** (CMD) | `gcc demo.c -o demo.exe` | `demo.exe` |
| **Mac / Linux** | `gcc demo.c -o demo` | `./demo` |

> 记忆：Mac/Linux 用 `./名字`，Windows 用 `.\名字.exe`。本课后面命令默认写 Mac/Linux 形式，Windows 用户照着上表换一下即可。

## 4. 编译 vs 链接 ⭐ 高频概念题

你写的 `.c` 是给人看的文本，CPU 不认识。编译器把它变成可执行文件，内部分两步：

```
demo.c  ──编译──▶  目标代码  ──链接──▶  demo（可执行）
 源代码                          ↑
                        把 printf 等库函数接进来
```

⚠️ 考点：**编译**（翻译成机器码）和**链接**（拼接用到的库）是两个不同阶段。平时一条命令全干完，但考研会单独问区别。

## 5. 关键认知 ⚠️

**改了源码必须重新编译**，旧的可执行文件不会自动更新。
源码 `.c` → 编译 → 可执行文件 → 运行，这条因果链要刻在脑子里。
