# C 语言考研课程（练习驱动）

> 面向零基础考研的 C 语言入门讲义 · 参考翁恺《C语言程序设计》、何钦铭/颜晖教材 · 对齐 408 考纲
> 编译器统一用 **gcc**，跨平台（Windows / macOS / Linux 通用）

---

## 怎么用这套课程

每个模块是一个文件夹，里面有三类文件：

| 文件 | 作用 |
|------|------|
| `讲义.md` | 这一模块的知识点 + 考点 + 易错坑 |
| `demo.c` | 写好的示例代码，编译运行看效果 |
| `练习.c` | 带 `TODO` 的题目，自己填空/补全 |

**学习四步法**：读讲义 → 跑 demo → 做练习 → 错题复盘。

> 第一次用先看 **`m00_编译运行流程/讲义.md`**，里面有 Windows / macOS / Linux 三平台的环境（gcc）安装步骤。

## 编译运行（每次都用这个）

在某个模块文件夹里，终端执行：

**macOS / Linux：**
```bash
cd m01_变量与输入输出            # 进入模块
gcc demo.c -o demo && ./demo     # 编译并运行 demo
gcc 练习.c -o lianxi && ./lianxi # 编译并运行你的练习
```

**Windows（PowerShell）：**
```powershell
cd m01_变量与输入输出
gcc demo.c -o demo.exe ; .\demo.exe       # 编译并运行 demo
gcc 练习.c -o lianxi.exe ; .\lianxi.exe   # 编译并运行你的练习
```

> `gcc 源文件.c -o 输出名` 是编译；运行时 macOS/Linux 用 `./输出名`，Windows 用 `.\输出名.exe`。

## 三个"拦路虎"

考研 C 语言失分集中在这三处，到了会放慢节奏、多做题：

- **M5 数据类型陷阱** — 补码、类型转换、`-1` 的存储，选择题高频坑
- **M7 指针** — C 语言命门，理解它后面全顺
- **M10 链表** — 指针实战，大题常考，必须画图
